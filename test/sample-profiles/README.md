# 実データサンプルプロフィール

検証用に保管する**実候補者プロフィール**。会員番号のみ匿名化し、それ以外（年齢・資格・経験・自己PR・希望条件等）は実データを保持する。

## ディレクトリ構成

```
test/sample-profiles/
├── README.md                       # このファイル
├── general/                        # 全件保管（特定会社に紐付かない汎用サンプル）
│   └── profiles_anon_YYYY-MM-DD.csv
└── [会社名]/                        # 会社別検証用（特定パターンを抽出）
    └── salary-pattern-X_*.csv
```

### vs `test/dummy-profiles/`

| ディレクトリ | 種別 | 用途 |
|---|---|---|
| `dummy-profiles/` | **人工データ**（D-DAI-01等） | プロンプトの単体テスト・回帰テスト |
| `sample-profiles/` | **実データ匿名化** | 生成品質の目視検証・新機能の動作確認 |

## 匿名化ルール

- **`member_id`**: 連番に置換（`ANON_0001`, `ANON_C001` 等）
  - `general/` の連番はファイル内の登場順（再現性のため固定）
  - 会社別の `ANON_C001` 等は別系統（general と紐付かない）
- **その他のフィールド**は全て保持:
  - 年齢・性別・居住地・希望勤務地（市区町村レベル）
  - 資格・経験職種・経験年数・就業状況
  - 自己PR（全文）・特記事項・職務経歴サマリー
  - ジョブメドレー側で既に匿名化されている項目（勤務先名等）はそのまま

理由: 自己PRと職務経歴は検証に最も重要な情報のため。会員番号さえ抜けば、第三者から個人特定はできない（プロフィール画面で誰でも見られる情報のため）。

## 命名規則

### 会社別サンプル

`[テストケース名]_anon.csv`

例:
- `salary-pattern-A_500up_anon.csv` - 給与訴求パターンA（500万以上希望）の検証用
- `salary-pattern-B_400up_anon.csv` - 給与訴求パターンB（400万以上希望）の検証用
- `salary-pattern-C_no-condition_anon.csv` - パターンC（年収条件なし、訴求しないケース）の検証用

1ファイル1ケースが原則。複数候補者を混ぜない（テストの粒度を保つ）。

### 汎用サンプル

`profiles_anon_YYYY-MM-DD.csv` - エクスポート日付付き

## 利用方法

### 単一候補者で生成検証

CSVから1行を取り出して `/generate-scout-pro [会社名]` の入力にする。属性判定・給与訴求パターン判定・出力結果を目視確認。

### 一括生成検証

`/csv-scout [会社名] [CSVパス]` で一括生成し、`personalized_text` 列を一覧で見る。

## 注意事項

- 編集禁止: `general/` の汎用サンプルは編集しない（再現性のため）
- 会社別サンプルで desired_area 等を編集する場合は、編集理由をファイル冒頭にコメント行で記録するか、PR description に明記する
- 新しいエクスポートを追加する場合は連番を継続せず、新ファイルとして `profiles_anon_YYYY-MM-DD.csv` で追加する
