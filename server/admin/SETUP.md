# Scout Assistant セットアップガイド（管理者向け）

## 概要

このガイドは、Scout Assistantの初期セットアップと新会社追加の手順をまとめたものです。管理者が実施し、セットアップ完了後にオペレーターへ引き渡します。

---

## 1. Chrome拡張のインストール

### 配布

ビルド済みの `scout-assistant-extension.zip` をオペレーターに配布します。

### インストール手順（各オペレーターのPCで実施）

1. 配布された `scout-assistant-extension.zip` を展開する
2. Chromeのアドレスバーに `chrome://extensions` と入力してEnter
3. 画面右上の **「デベロッパーモード」** のスイッチをONにする
4. 画面左上に表示される **「パッケージ化されていない拡張機能を読み込む」** をクリック
5. 展開したフォルダ内の `extension/dist` フォルダを選択して「フォルダの選択」を押す
6. 拡張機能一覧に「Scout Assistant」が表示されれば完了

> Chromeツールバーにアイコンが表示されない場合は、パズルピースのアイコン（拡張機能メニュー）から「Scout Assistant」をピン留めしてください。

### 拡張機能の更新

新しい `scout-assistant-extension.zip` を配布した場合:

1. 新しいzipを展開して、前回と同じ場所に上書き
2. `chrome://extensions` を開く
3. Scout Assistantの **リロードアイコン**（🔄）をクリック

## 2. API接続の設定

各オペレーターのブラウザで設定します。

1. Chromeツールバーの **Scout Assistant アイコン** をクリック → **「サイドパネルを開く」**
2. サイドパネルの **⚙ 設定** タブを開く
3. 以下を入力:

| 項目 | 入力する内容 |
|------|-------------|
| **APIエンドポイント** | `https://scout-api-1080076995871.asia-northeast1.run.app` |
| **パスワード** | API接続用のパスワード |

4. **「接続テスト」** ボタンを押す
5. 緑色で「接続OK」と表示されれば設定完了

> この設定はブラウザに保存されるので、オペレーターが毎回入力する必要はありません。ポップアップの「API設定」からも同じ設定ができます。

## 3. 管理画面のパスワード設定

管理画面のURLにアクセスし、初回ログインします:

```
https://scout-api-1080076995871.asia-northeast1.run.app/admin/
```

パスワードを変更する場合は、ヘッダー右側の **「パスワード設定」** から変更できます。

## 4. 新会社のセットアップ

### 4-1. AIウィザードで一括生成（推奨）

管理画面のヘッダーにある **「＋ 新規会社」** ボタンで起動します。

#### Step 1: 入力

| 項目 | 説明 |
|------|------|
| **会社ID** | 英小文字・数字・ハイフンのみ（例: `new-visiting-nurse`） |
| **会社情報** | HPの内容、求人情報、特色などをコピペ。テキストが多いほど精度が上がります（500文字以上推奨） |
| **スカウト文の雛形** | 任意。既存テンプレートがあれば貼り付け。空欄の場合はAIが自動生成 |

入力後、**「AIで全設定を生成」** を押す。

#### Step 2: AI生成

AIが既存の全会社設定を参考にしながら、以下を自動生成します（30〜60秒）:

- テンプレート（4種類: パート初回/再送、正社員初回/再送）
- パターン（A〜G: 経歴が少ない候補者向けの定型文）
- プロンプトセクション（会社固有の3セクション: station_features, education, ai_guide）
- バリデーションルール
- 資格修飾テキスト

> 全社共通のプロンプト（トーン、共通ルール、NG表現等）は登録済みのため、会社固有の情報のみが生成されます。

#### Step 3: 確認・保存

生成された各設定がセクションごとに表示されます。内容を確認し、**「保存してセットアップ完了」** を押すと全データがGoogle Sheetsに書き込まれます。

> 求人データはウィザードでは生成されません。次の手順で別途登録してください。

### 4-2. 求人の登録

Chrome拡張から求人を取得・登録します。

1. ジョブメドレーのスカウト送信画面を開く
2. サイドパネルの **⚙ 設定** タブを開く
3. **「求人を取得」** を押す → 求人リストが表示される
4. 登録したい求人にチェックを入れる
5. **「選択した求人をAPIに登録」** を押す

> 管理画面のテーブルビュー → 求人タブから手動で追加することもできます。

### 4-3. 施設情報の取得（任意）

新会社のprofile.md作成に役立つ施設情報を、ジョブメドレーからスクレイピングできます。

1. ジョブメドレーの施設管理ページ（`customers.job-medley.com/customers/facilities`）を開く
2. サイドパネルの **⚙ 設定** タブを開く
3. **「施設一覧を取得」** を押す → 施設のチェックリストが表示される
4. 情報を取得したい施設にチェック
5. **「選択した施設の求人を取得」** を押す
6. 各施設の求人ページを自動巡回し、施設情報・求人詳細を取得
7. 完了後:
   - **「profile.md をダウンロード」** → 整形された会社情報ファイル
   - **「生テキストをダウンロード」** → 取得した元データ（デバッグ用）

> 取得した施設情報は、新会社ウィザードの「会社情報」欄に貼り付けると、AIの生成精度が上がります。

## 5. セットアップ後の確認

### パイプラインテスト

管理画面のフロービューで、設定が正しく動作するか確認します。

1. フロービューに切り替え
2. テストパネルにサンプルの候補者情報を入力
3. **「テスト実行」** を押す
4. 各ステップの通過/失敗を確認:
   - 🟢 緑 = 通過
   - 🔴 赤 = 失敗/除外
   - ⚪ 灰 = 未通過

以下のパターンでテストすることを推奨:

| テストケース | 入力例 |
|-------------|--------|
| AI生成パス | 経歴あり（職務経歴に文章を入力） |
| 型はめパス | 経歴なし（職務経歴を空欄） |
| 除外ケース | 対象外の年齢や資格を入力 |
| 正社員テンプレート | 希望雇用に「正社員」を選択 |

### キャッシュ更新

設定変更後は必ずヘッダーの **「キャッシュ更新」** を押してください。押さないとChrome拡張からの生成に反映されません。

## 6. オペレーターへの引き渡し

セットアップ完了後、オペレーターに以下を伝達してください:

| 項目 | 内容 |
|------|------|
| `scout-assistant-extension.zip` | Chrome拡張のインストール用（インストール済みなら不要） |
| APIエンドポイント | `https://scout-api-1080076995871.asia-northeast1.run.app` |
| パスワード | API接続用パスワード |
| 対象会社 | ドロップダウンで選択する会社名 |
| オペレーター向けマニュアル | `extension/MANUAL.md` |

オペレーターの日常操作は「抽出→生成→送信」のみです。設定変更は不要です。
