# Scout Admin 取扱説明書（管理画面）

## この管理画面について

Scout Admin は、スカウト文の自動生成に使われる設定（テンプレート、パターン、AIのプロンプト、送信条件など）を管理するWebアプリです。

Chrome拡張やAPIから送られるスカウト文は、すべてこの管理画面で設定した内容に基づいて生成されます。設定を変えれば、生成されるスカウト文の内容も変わります。

**主な機能**:
- スカウト文のテンプレート（外枠）の編集
- 送信対象を絞るバリデーションルールの設定
- AIが文章を生成するときのプロンプト（指示文）の調整
- 経歴情報が少ない候補者向けの定型パターン管理
- 対象求人の登録・管理
- 生成ログの閲覧
- 新しい会社の設定をAIで一括生成（既存会社の設定を参考にして生成）

**画面モードは2種類あります**:

| モード | どんなとき使う？ |
|--------|----------------|
| **フロービュー** | 生成の流れを全体像で把握したいとき。テスト実行もここから |
| **テーブルビュー** | テンプレートやパターンを具体的に追加・編集・削除したいとき |

> 初期セットアップ（新会社追加・求人登録等）は管理者向けセットアップガイド（`server/admin/SETUP.md`）を参照してください。

## アクセスとログイン

### URLを開く

ブラウザで以下のURLにアクセスします:

```
https://scout-api-1080076995871.asia-northeast1.run.app/admin/
```

### ログイン

初回アクセス時に「パスワードを入力してください」と表示されます。

1. Chrome拡張のAPI設定と同じ **パスワード** を入力
2. **「ログイン」** を押す

ログインに成功すると管理画面が表示されます。パスワードはブラウザに保存されるので、同じブラウザからは次回以降自動でログインできます。

> 「パスワードが違います」と表示される場合は、管理者にパスワードを確認してください。

> パスワードを変更したい場合は、ログイン後にヘッダー右側の **「パスワード設定」** ボタンから変更できます。

## 画面の見方

ログイン後、画面の一番上にヘッダーバー（緑色）が表示されます。ここで会社の切り替えやビューの切り替えを行います。

### ヘッダーの各ボタン

| 要素 | 説明 |
|------|------|
| **会社ドロップダウン**（左側） | 操作対象の会社を切り替えます。選んだ会社のデータだけが表示されます |
| **＋ 新規会社** | 新しい会社を追加するウィザードを開きます（セットアップガイド参照） |
| **フロー / テーブル** | 画面の表示モードを切り替えます |
| **キャッシュ更新**（右側） | 設定変更後にこのボタンを押すと、変更がスカウト生成に反映されます |
| **パスワード設定**（右側） | ログインパスワードを変更できます |

> **重要**: テンプレートやパターンを編集した後は、必ず **「キャッシュ更新」** を押してください。押さないと、Chrome拡張からのスカウト生成に変更が反映されません。

## フロービュー

スカウト文生成の7ステップを視覚的に表示します。

### パイプライン全体像

```
1. プロフィール受信
      ↓
2. 職種カテゴリ判定（資格から自動分類）
      ↓
3. テンプレート選択（パート/正社員 × 初回/再送）
      ↓
4. バリデーション（年齢・資格・AI条件でフィルタ）
      ↓
   ┌──────────┬──────────┐
   ↓                     ↓
5a. AI生成           5b. 型はめ
（経歴あり）        （経歴なし）
   ↓                     ↓
   └──────────┴──────────┘
      ↓
6. テキスト組み立て（テンプレート＋パーソナライズ文）
      ↓
7. 求人解決（求人IDの割り当て）
```

### ノードの見方

各ステップはカード形式で表示されます。

- **クリック** → 詳細が展開され、設定内容のサマリーが表示されます
- **編集可能なノード**（3, 4, 5a, 5b, 7）は「テーブルで編集」リンク付き
- **テスト結果の色**:
  - 🟢 緑 = このステップを通過
  - 🔴 赤 = このステップで失敗/除外
  - ⚪ 灰 = このステップを通過していない

### パイプラインテスト

画面上部のテストパネルで、候補者プロフィールを入力してパイプラインの動作を確認できます。

**入力項目**:

| 項目 | 説明 |
|------|------|
| **資格** | 複数選択可（看護師、准看護師 等）。選択するとタグ表示され、×で解除 |
| **希望雇用** | 複数選択可（パート、正社員 等） |
| **年齢** | 数値入力（例: 35） |
| **経験年数** | 数値入力（例: 5） |
| **経歴タイプ** | 訪問看護、病棟 等 |
| **就業状態** | ドロップダウンから選択（就業中、離職中 等） |
| **職務経歴（空=型はめ）** | テキスト入力。空欄にすると型はめパスで処理されます |
| **自己PR** | テキスト入力 |
| **AI生成をスキップ（モック）** | チェックを入れるとAI呼び出しをスキップ（高速テスト用） |

**手順**:

1. テスト用のプロフィール情報を入力
2. **「テスト実行」** を押す
3. 各ステップの通過/失敗がノードの色で可視化されます
4. AI生成またはパターン型はめの結果テキストが表示されます

## テーブルビュー

6つのタブでデータを管理します。

### 共通操作

| 操作 | 方法 |
|------|------|
| **新規追加** | ツールバーの **「＋ 追加」** ボタン |
| **編集** | 行の **「編集」** ボタン → 編集ダイアログ → **「保存」** |
| **削除** | 行の **「削除」** ボタン → 確認ダイアログ → OK |
| **ロック/ロック解除** | ツールバーの **「🔓 ロック解除」** / **「🔒 ロック」** ボタン。ロック中は他社データや全社共通データの編集・削除が無効になります |

長いテキストは先頭部分のみ表示されます。**「もっと見る」** で全文を確認できます。

---

### テンプレート

スカウト文の外枠（挨拶・会社紹介・締め等）を管理します。

**4種類**:
- パート初回 / パート再送
- 正社員初回 / 正社員再送

テーブルの列:

| 列名 | 説明 |
|------|------|
| **会社** | 会社ID（例: `ark-visiting-nurse`） |
| **職種カテゴリ** | 対象職種（例: `nurse`） |
| **テンプレート種別** | パート_初回 / パート_再送 / 正社員_初回 / 正社員_再送 |
| **テンプレート本文** | スカウト文の外枠テキスト |
| **操作** | 「編集」「削除」ボタン |

**重要**: テンプレート本文内の `{パーソナライズ文}` がAI生成文や型はめ文の挿入位置です。この記述を必ず含めてください。

---

### バリデーション

スカウト対象外の条件（年齢制限・資格ルール・AI判定条件）を管理します。

テーブルの列:

| 列名 | 説明 |
|------|------|
| **会社** | 会社ID |
| **年齢下限** | スカウト対象の最低年齢（例: 20） |
| **年齢上限** | スカウト対象の最高年齢（例: 55） |
| **資格ルール（JSON）** | 求人ごとの必須資格・除外資格（下記参照） |
| **操作** | 「編集」「削除」ボタン |

#### 編集ダイアログの設定項目

**ルールベース（チェックボックス）**:

| ルール | 説明 |
|--------|------|
| **資格不一致NG** | 求人の必須資格を持たない候補者を除外 |
| **臨床経験なしNG** | 臨床経験がない候補者を除外 |
| **スカウト済みNG** | 過去にスカウト送信済みの候補者を除外 |

**スカウト上限設定**（スカウト済みNGを有効にした場合に表示）:

| 項目 | 説明 |
|------|------|
| **最大送信回数** | 同じ候補者へのスカウト上限回数 |
| **再送間隔（日）** | 再送までの最低日数 |

**資格ルール**（求人ごと）:

- **必須資格**: この資格のいずれかを持つ候補者のみ対象
- **除外資格**: この資格を持つ候補者を除外

**AI判定条件（自由記述）**: テキストで条件を書くとAIが判定します（例:「ブランク10年以上の場合は除外」）

---

### AI生成設定

AI生成パス（経歴あり候補者）で使われるプロンプトの構成パーツを管理します。複数のセクションが表示順に結合されて、AIへの指示文（system prompt）になります。

テーブルの列:

| 列名 | 説明 |
|------|------|
| **会社** | 会社IDまたは **全社共通**（青色リンク表示） |
| **セクション種別** | このセクションの種類（後述） |
| **職種カテゴリ** | 対象職種（空欄＝全職種共通） |
| **表示順** | 結合される順番（1が最初） |
| **内容** | プロンプト本文 |
| **操作** | 「編集」「削除」ボタン |

> **全社共通** と表示されている行は、全ての会社で共有されるセクションです。会社固有の設定と組み合わせて使われます。

**セクション種別**:

| 種別 | 説明 | 補足 |
|------|------|------|
| `role_definition` | AIの役割定義 | 通常は全社共通 |
| `station_features` | 会社の特色（AI生成の参考情報） | 会社固有 |
| `education` | 教育体制（AI生成の参考情報） | 会社固有 |
| `tone_and_manner` | 文体・トーンのルール | 通常は全社共通 |
| `common_rules` | 共通ルール（文字数等） | 通常は全社共通 |
| `ai_guide` | AI生成ガイド（生成の細かいルール） | 会社固有 |
| `ng_expressions` | NG表現リスト | 通常は全社共通 |

**職種カテゴリ列**: 空欄の場合は全職種で使われます。特定の職種（例: `nurse`）を指定すると、その職種の候補者にのみ適用されます。

**プロンプトプレビュー**: **「全文プレビュー」** ボタンで、全セクションがどう組み立てられてAIに渡されるか確認できます。

---

### 型はめ設定

経歴が少ない候補者（自己PR・職歴なし）に使う定型文を管理します。候補者の経験年数×年齢で自動的にパターンが選ばれます。

#### 型の一覧ガイド

タブ上部に **▼ 型の一覧ガイド** が表示されます（クリックで開閉）。各パターンの対象がカード形式で確認できます:

| パターン | 表示名 | 対象 |
|---------|--------|------|
| **A** | 豊富な経験への期待 | 経験10年+ / 40代〜×経験6年+ |
| **B1** | 確かな経験×特色 | 経験6〜9年 |
| **B2** | 経験×特色 | 経験3〜5年 |
| **C** | 経験とのフィット | 40代〜 × 経験1〜2年 |
| **D** | 経験ある前提で評価 | 40代〜 × 経験未入力 |
| **E** | ポテンシャル+教育体制 | 20〜30代 × 経験1〜2年 |
| **F** | 教育体制+成長環境 | 20〜30代 × 経験未入力 |
| **G** | 教育体制メイン | 在学中 |

テーブルの列:

| 列名 | 説明 |
|------|------|
| **会社** | 会社ID |
| **職種カテゴリ** | 対象職種 |
| **パターン型** | A〜G（色付きバッジで表示） |
| **就業状態** | 就業中/離職中 等のバリエーション |
| **パターン本文** | 型はめ文のテキスト |
| **特色バリエーション** | 自己PR等に応じた差し替えテキスト |
| **表示名** | パターンの表示名 |
| **対象条件** | このパターンの対象説明 |
| **マッチングルール** | 適用条件（経験年数・年齢のJSON） |
| **資格の組み合わせ** | 資格修飾用の資格パターン |
| **差し替え文** | 資格修飾用の差し替えテキスト |
| **操作** | 「編集」「削除」ボタン |

#### AIで一括生成

パターンのテキストをAIで一括生成できます。

1. ツールバーの **「AIで一括生成」** を押す
2. **会社情報** テキストエリアに会社の特色・教育体制等を入力（profile.mdの内容を貼り付け）
3. **「生成する」** を押す → AIがパターンA〜Gのテキストを生成
4. プレビューを確認
5. **「保存する」** を押す → 既存パターンが上書きされます

### 求人

ジョブメドレーの求人情報を管理します。

テーブルの列:

| 列名 | 説明 |
|------|------|
| **会社** | 会社ID |
| **職種カテゴリ** | 対象職種 |
| **求人ID** | ジョブメドレーの求人ID（例: `1550716`） |
| **求人名** | 求人の正式名称 |
| **表示ラベル** | 短い表示名（例: `看護師 パート`） |
| **雇用形態** | パート / 正社員 |
| **有効** | TRUE（使用中）/ FALSE（無効） |
| **操作** | 「編集」「削除」ボタン |

**求人の追加方法**:
- **Chrome拡張から**: 設定タブの「求人抽出・登録」でジョブメドレーから自動取得して登録（推奨）
- **管理画面から**: 「＋ 追加」で手動入力

---

### 生成ログ

スカウト文の生成履歴を閲覧できます。**読み取り専用**（追加・編集・削除不可）。

テーブルの列:

| 列名 | 説明 |
|------|------|
| **会社** | 会社ID |
| **会員番号** | 候補者の会員番号 |
| **生成ステータス** | 生成の成否 |
| **パス** | AI生成 / 型はめ のどちらで生成されたか |
| **生成テキスト** | 生成されたパーソナライズ文のプレビュー |

生成の監査や問題調査に使用します。

## よくある操作手順

### テンプレートの文言を変更する

1. テーブルビューで **テンプレート** タブを選択
2. 対象行の **「編集」** を押す
3. **body** フィールドでテキストを修正
4. **「保存」** → **「キャッシュ更新」**

### バリデーションルールを追加・変更する

1. テーブルビューで **バリデーション** タブを選択
2. 対象行の **「編集」** を押す
3. 年齢範囲やルールを変更
4. **「保存」** → **「キャッシュ更新」**

### パターンをAIで一括生成する

1. テーブルビューで **型はめ設定** タブを選択
2. **「AIで一括生成」** を押す
3. 会社情報を入力 → **「生成する」**
4. プレビュー確認 → **「保存する」** → **「キャッシュ更新」**

### パイプラインの動作を確認する

1. フロービューに切り替え
2. テストパネルに候補者情報を入力
3. **「テスト実行」** を押す
4. 各ステップの通過/失敗を確認

### 求人を追加する

**推奨**: Chrome拡張の⚙ 設定タブ →「求人を取得」→「選択した求人をAPIに登録」

**手動**: テーブルビューの **求人** タブ → **「＋ 追加」** でジョブメドレーの求人IDを入力

---

## キャッシュ更新

管理画面での設定変更は Google Sheets に書き込まれますが、APIサーバーはメモリ上のキャッシュを使用しています。

**設定変更後は必ずヘッダーの「キャッシュ更新」を押してください。** これにより、APIサーバーがGoogle Sheetsから最新データを再読み込みします。

## トラブルシューティング

| 症状 | 対処法 |
|------|--------|
| パスワードを忘れた | 管理者に確認してください |
| 設定変更が生成結果に反映されない | ヘッダーの **「キャッシュ更新」** を押してください |
| AI一括生成が失敗する | 会社情報テキストが短すぎないか確認してください（500文字以上推奨） |
| テスト実行でエラーが出る | APIサーバーが起動しているか確認してください |
| 「パスワードが違います」と表示される | 正しいパスワードを管理者に確認してください |
| テーブルに他社のデータが表示される | ヘッダーの会社ドロップダウンを確認してください |
| 編集ボタンが効かない | 🔒ロックがONになっていないか確認してください |
