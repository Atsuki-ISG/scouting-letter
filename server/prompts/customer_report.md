# 顧客向けレポート生成プロンプト

`POST /api/v1/admin/export_report` から呼ばれる Gemini 用プロンプト。実装上の文字列は `server/api/routes_admin.py` の `_CUSTOMER_REPORT_PROMPT` に inline で持つ（`improve_template.md` と同じスタイル）。

## 目的

スカウト送信データの分析結果を、顧客（クライアント）に渡せる業務日本語の所感に変換する。
内部運用の情報（パターンコード・生成パス・テンプレートVer 等）は事前にフィルタしたうえで入力に与える。

## 出力仕様

必ず以下の JSON 単体で返すこと。前後に余計な解説や ```json フェンスを含めない。

```json
{
  "situation": "今期の送信状況を2〜4文で。数値は地の文に埋め込む。",
  "findings": "目立った傾向を2〜4文で。どのセグメントが反応良い/悪いかと、その数値根拠を明記する。",
  "next_actions": "次サイクルで取り組むべきことを2〜3文で。施策ではなく観点として書く。"
}
```

## 禁止事項（AIっぽさの排除）

以下の表現・構造は使わない:

- 「と考えられます」「〜が示唆されます」「〜と推察されます」「〜と見受けられます」
- 「AI分析」「データ分析の結果」「興味深いことに」
- 「## 発見したパターン」などの装飾見出し（JSON フィールド内では本文のみ）
- 箇条書きの多用（JSON の各フィールドは原則**段落で**書く）
- 英数字の前後に半角スペースを入れる癖（読み物として自然な日本語）
- 絵文字・記号（⭐️ 📈 💡 など）

## 推奨されるトーン

- 「ございました」「しております」「いたしました」系の事務的な丁寧語
- 「〜通の送信中〜件の返信があり、返信率は〜%でした」のように数値を地の文に埋め込む
- 施策の**当否**を断定せず、事実ベースで述べる（例: 「40代の返信率が他年齢層より高い水準でした」）

## 入力データの形式

プロンプトには以下を埋め込む:

- 期間・会社名・KPI（総送信数/返信数/返信率）
- 顧客向けに絞り込んだクロス集計（地域、年齢層、経験区分、希望雇用形態、就業状況、テンプレート種別のみ）
- （任意）ディレクターからの directive（「今月は◯◯を強調したい」等）
