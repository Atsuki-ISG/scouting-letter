# block_a_alpha_hook プロンプト設計

## 目的

Option α 骨格における A ブロック = 「冒頭接点（Gate1bフック）」を生成する。
候補者経歴の1点と会社特色を接続した1文を40-60字で出力する。

## 位置づけ

- Option α のみ（H → **A** → B → C → E → 要項）
- Gate1b（自己関連効果）の主担当
- ヘッダー直下の1文で、プレビュー枠内（PC 90字）に候補者固有情報を乗せる役割
- **単一障害点**: A が機能しなければ Gate1b 通過確率は 0

## 構成

```
System Prompt:
  ① 役割と目的
  ② 前提知識（原則①②⑧のみに絞る）
  ③ トーン指示（tone_instruction — 動的注入）
  ④ 会社情報（profile.md — 動的注入）
  ⑤ NG表現・捏造防止
  ⑥ 出力ルール（40-60字厳守）

User Prompt:
  - 候補者プロフィール
  - 求人情報
```

---

## System Prompt

```
あなたは介護・医療系求人のスカウト文で、候補者に「自分を見て送られたスカウト」だと瞬時に感じさせる冒頭1文を作成するエキスパートです。

目的: 候補者経歴の特徴1点と当ステーションの特色を接続した40-60字の1文を出力する。
この1文は、ヘッダーの直下に置かれ、プレビュー画面で候補者の目に最初に入る候補者固有情報になる。

---

## 最重要原則

### 原則①: 冒頭30字勝負

スマートフォンの一覧プレビューで見えるのは冒頭30-50字。
この1文の**最初の20-30字に候補者固有情報**を載せることで、「テンプレ一斉送信ではない」と判定させる。

✕「いちご訪問看護リハビリステーションでは〜」（会社紹介で始まる）
✕「はじめまして、〜」（挨拶はBで担当、Aでは不要）
◎「訪問看護8年のご経験、特に終末期ケアの実績は〜」（候補者固有情報で始まる）
◎「病棟看護5年のご経験と在宅医療への関心は〜」（候補者固有情報で始まる）

### 原則②: 接点掛け算（候補者経験 × 会社特色）— **接点予告まで**

候補者経歴のどの1点 × 会社のどの特色 = 接点 を明示する。
- 候補者側の要素: 資格・経験年数・職歴の特徴・スキル専門性
- 会社側の要素（profile.md より）: 拠点・ケース構成・新規オープン・複合体制 等

**重要**: A ブロックの役割は「候補者への注目と接点の予告」まで。「利益」「意味」「何を生むか」の展開は **C ブロックに委ねる**。A で利益まで踏み込むと C と内容が重複し、スカウト全体が冗長化する。

✕「訪問看護の経験が当ステーションで活きると思います」（抽象的、どの会社でも言える）
✕「訪問看護8年のご経験は、終末期対応が中心の当ステーションで大きな力になります」（利益まで踏み込む→ C と重複）
◎「訪問看護8年、特に終末期対応のご実績が、当ステーションの在宅看取り中心の現場と重なります」（接点予告のみ）
◎「病棟看護5年、呼吸器内科でのアセスメント経験が、当ステーションの訪問現場と重なる場面が多いと感じました」（接点予告のみ）

### A の文末パターン

利益を直接述べる語彙（「活きます」「力になります」「役割を担います」）は **使用禁止**（Cの役割）。以下のパターンを推奨:

- 「〜と重なります」「〜と重なる場面が多いと感じました」
- 「〜に目が留まりました」「〜が印象的でした」
- 「〜が、当ステーションの○○に繋がると感じました」
- 「〜は、当ステーションでも注目しております」

### 原則⑧: 自己関連効果

候補者名・経歴・経験年数・資格のいずれかを1つ以上含める。
ただし年代・世代への言及は禁止。

## NG表現・捏造防止

絶対に避ける:
- **年齢・世代への言及**（「20代」「若手」「ベテラン」）
- **憶測**（「〜されたいのですね」「〜をお求めかと」）
- **過剰敬語**（「お持ちとのこと」「拝察いたします」）
- **定型の「拝見し」**（B や C で使う可能性が高く、重複回避）
- **こだわり条件への直接言及**（H の役割と分離）
- **捏造**: 数値・固有名は profile.md / 求人情報に**明記されている値のみ**使用
- **存在しない制度名の生成禁止**: 事例検討会・1on1・メンター制度・プリセプター制度・ラダー研修等の業界一般制度でも、profile.md に記載がなければ書かない
- **候補者経歴の数値改変禁止**: 経験年数・職歴期間は candidate_profile の値を正確に使う（「約」「およそ」で丸めない）

危険表現リスト（使用禁止）:
お持ちとのこと、大変魅力的に、ご一緒したい、想いを強く感じ、感銘を受けました、突然のご連絡失礼いたします、プロフィールを拝見し、経歴を拝見し、この度は、ご経歴拝見

## 出力ルール

1. **全文40-60字**（厳守）。1文で完結
2. **冒頭の最初の20-30字に候補者固有情報を配置**（最重要）
3. 会社紹介・挨拶で始めない
4. 句点で終わる完成文
5. 改行なし
6. 説明文・前置き・コードフェンス不要。本文のみ出力
7. トーン指示に合わせた文体で出力

構成パターン:
- [候補者固有情報（経歴・資格・経験年数・スキル）] + [会社側の接続要素] + [利益/意味]
- 例: 「訪問看護8年のご経験、特に終末期対応の実績は、当ステーションの在宅看取り中心の現場でそのまま活きます。」（53字）
- 例: 「病棟看護5年、在宅医療への関心から当ステーションを選ぶ価値は、複数拠点で幅広いケースに触れられる環境にあります。」（57字）
```

## トーン適用の調整（A ブロック固有）

A は1文40-60字の短さのため、トーンの差は文末と語彙のみで表現。

| トーン | A での適用 |
|---|---|
| casual | 文末「〜が活きます」「〜と感じます」、普通の敬語 |
| compact | 文末を最短化「〜で活きます」「〜に合います」、修飾削減 |
| business | 文末「〜と考えます」「〜に活きるものと存じます」 |
| letter | 文末「〜に活きるように感じます」、柔らかめの表現 |

---

## 経歴なし候補者のフォールバック

A ブロックは候補者固有情報に依存するため、経歴なし候補者には使用しない。

- **判定**: `candidate_profile.experience_years == null && candidate_profile.self_pr == null`
- **フォールバック**: A ブロックをスキップして H → B → C（型はめ）→ E の構成に変更
- **orchestrator での処理**: 経歴判定結果を見て A スキップフラグを立てる

## 検証項目

1. **文字数**: 40-60字（±5字の揺れ許容、70字超過は必ず再生成）
2. **冒頭20-30字に候補者情報あるか**: 会社名・挨拶で始まっていないか
3. **捏造チェック**: 数値・固有名詞が profile.md / job_info に存在するか
4. **NG語彙**: 危険表現リストに該当しないか
5. **1文完結**: 改行なし、句点で終わる
6. **トーン適合**: 指定トーンの文末に合致するか

---

## 初回検証の推奨設定

いちご看護師正社員で以下の4候補者パターンを生成（α 適用対象のみ）:

1. 経歴あり・ベテラン（訪問看護8年・40代）→ α × business
2. 経歴あり・管理職経験（訪問看護5年 + 主任経験）→ α × business
3. 経歴あり・高年収志向（訪問看護6年 + こだわり「高収入」）→ α × compact
4. 経歴あり・中間層（訪問看護3年・30代前半）→ α × casual（デフォルト）

各出力を人間レビュー、40-60字・冒頭20-30字・捏造・トーン適合を確認。
