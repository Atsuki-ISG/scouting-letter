# ジョブメドレー スカウト送信支援 Chrome拡張機能 — 要件定義・工数見積書

**作成日**: 2026年3月
**バージョン**: 1.0

---

## 1. プロジェクト概要

### 1.1 目的

医療・介護系求人プラットフォーム「ジョブメドレー」において、スカウトメッセージの送信業務を効率化するChrome拡張機能を開発する。候補者プロフィールの一括抽出、AIで生成したスカウト文のインポート・送信アシスト、返信データの収集・管理を一体化し、スカウト業務の工数を大幅に削減する。

### 1.2 対象プラットフォーム

- **対象サイト**: ジョブメドレー（job-medley.com）
- **動作環境**: Google Chrome（Manifest V3準拠）
- **導入形態**: ローカルインストール（Chrome Webストア非公開）

### 1.3 技術スタック

| 項目 | 技術 |
|------|------|
| 拡張機能仕様 | Chrome Extension Manifest V3 |
| 開発言語 | TypeScript |
| ビルドシステム | Vite（HTML/CSS）+ esbuild（Content Script / Service Worker） |
| UIフレームワーク | なし（Vanilla TypeScript + カスタムCSS） |
| 外部ライブラリ | なし（CSV/YAMLシリアライザは自前実装） |

## 2. 機能要件

### Phase 1: プロフィール一括抽出

候補者検索結果ページから、候補者プロフィール情報を自動的に一括抽出しCSV出力する。

| # | 機能 | 詳細 |
|---|------|------|
| 1-1 | 候補者カード自動操作 | 検索結果の候補者カードを順次クリックし、スカウトボタン押下でoverlayパネルを表示 |
| 1-2 | overlay表示待機 | MutationObserver＋ポーリングの二重監視で、非同期読み込みされるoverlayの表示完了を検知（タイムアウト: 10秒） |
| 1-3 | プロフィールデータ抽出 | overlay内のDT/DD定義リストから15項目を日本語ラベルベースで取得（会員番号、性別、年齢、居住地、保有資格、経験年数、就業状況、希望職種・勤務地・雇用形態・入職時期、自己PR、特記事項、職務経歴） |
| 1-4 | タブ切替・職務経歴取得 | 「プロフィール」→「職務経歴」タブを自動切替し、経歴テキストを取得 |
| 1-5 | レートリミット | 抽出間隔800ms以上を確保し、サーバー負荷を抑制 |
| 1-6 | 進捗表示 | プログレスバー＋処理済み件数をリアルタイム表示 |
| 1-7 | 中断機能 | 抽出処理をいつでも中断可能 |
| 1-8 | CSV出力 | 抽出データをRFC 4180準拠のCSVファイルとしてダウンロード（UTF-8 BOM付き、Excel互換） |
| 1-9 | データ永続化 | 抽出済みプロフィールをchrome.storage.localに保存し、パネル再開時に復元 |

### Phase 2: スカウト文インポート・送信アシスト

AIで生成したスカウト文をインポートし、ジョブメドレーのフォームへの自動入力・送信を支援する。

| # | 機能 | 詳細 |
|---|------|------|
| 2-1 | CSVファイルインポート | ファイルピッカーからCSVを読み込み、候補者リストとして展開 |
| 2-2 | テキスト貼り付けインポート | テキストエリアにCSVを直接ペーストして取り込み（Markdownコードフェンス自動除去対応） |
| 2-3 | 候補者リスト表示 | インポートした候補者をカード形式で一覧表示（会員番号、ラベル、パーソナライズ文プレビュー） |
| 2-4 | ステータス管理 | 各候補者の送信状態を管理（ready → sent / skipped） |
| 2-5 | インライン編集 | カード内でスカウト文をその場で編集可能。修正前後のテキストと修正理由を`FixRecord`として記録 |
| 2-6 | 本文コピー | スカウト文全文をクリップボードにコピー |
| 2-7 | 本文自動セット | overlay表示 → 求人自動選択 → テキスト自動入力を一括実行 |
| 2-8 | 求人自動選択 | React + Downshiftオートコンプリートに対応した自動求人選択（キーワード入力 → ドロップダウン待機 → オプション選択） |
| 2-9 | React対応フォーム入力 | React管理下のフォームに対し、ネイティブsetterおよびexecCommandを用いて値をセット |
| 2-10 | overlay自動オープン | 候補者の会員番号から該当カードを特定し、スカウトボタンクリック → overlay表示まで自動化 |
| 2-11 | 連続送信モード | 候補者リストを順次処理し、overlayの表示→入力→ユーザー確認→次の候補者へと自動遷移 |
| 2-12 | スキップ機能 | 連続送信中に特定の候補者をスキップ可能 |
| 2-13 | 送信進捗表示 | プログレスバーで送信済み/全体の進捗を表示 |
| 2-14 | overlay同期 | ページ上で開いているoverlayの会員番号を検知し、候補者リストの該当カードをハイライト・自動スクロール |
| 2-15 | 修正履歴エクスポート | 記録されたFixRecordをMarkdown形式（修正前後のdiffブロック付き）でダウンロード |
| 2-16 | データ永続化 | 候補者リスト・修正記録をchrome.storage.localに保存 |

### Phase 3: 返信・やりとり管理

送信済みスカウトに対する候補者の返信データを収集・管理し、改善分析に活用する。

| # | 機能 | 詳細 |
|---|------|------|
| 3-1 | 単体メッセージ抽出 | メッセージ画面で表示中の会話スレッドを抽出（送信者判別、日付解析、メッセージ種別除去） |
| 3-2 | 一括メッセージ抽出 | サイドバーの無限スクロールを自動読み込みし、全会話スレッドを順次クリック・抽出 |
| 3-3 | 一括抽出の進捗表示 | 処理中の件数・ステータスをリアルタイム表示、中断ボタン |
| 3-4 | 手動入力パーサー | テキストエリアに手入力されたやりとりテキストを解析（企業/求職者プレフィックス判定、日付行対応、複数行メッセージ対応） |
| 3-5 | 返信記録の自動生成 | 手動入力時、候補者リストとマッチングしてスカウトメタデータ（テンプレート種別、パーソナライズ文）と返信を紐付け |
| 3-6 | 会話スレッド管理 | スレッド一覧表示（会員番号、開始日、メッセージ数、返信有無バッジ）、個別削除 |
| 3-7 | YAML出力（個別） | 会話スレッドごとにYAMLファイルをダウンロード |
| 3-8 | YAML出力（一括） | 全会話スレッドをマルチドキュメントYAMLとして一括ダウンロード |
| 3-9 | 返信記録YAML出力 | スカウト→返信の紐付けデータをYAMLとしてエクスポート |
| 3-10 | データ永続化 | 会話スレッド・返信記録をchrome.storage.localに保存（同一会員番号は上書き更新） |

## 3. 非機能要件

| 項目 | 要件 |
|------|------|
| 対応ブラウザ | Google Chrome（Manifest V3） |
| パフォーマンス | 候補者抽出時800ms以上の間隔を確保。overlay表示待機タイムアウト10秒。バッチ処理は500ms間隔。 |
| セキュリティ | Content Scriptの適用範囲をjob-medley.comドメインに限定。外部通信なし。 |
| データ保存 | chrome.storage.local使用。データはブラウザローカルに保持。 |
| 保守性 | DOMセレクタを専用ファイルに集約し、サイト側のDOM構造変更時の修正箇所を局所化 |
| 拡張性 | 会社・求人の追加は設定ファイル（constants.ts）への追記のみで対応可能 |

## 5. 前提条件・制約事項

1. **DOM依存**: 本拡張機能はジョブメドレーのHTML DOM構造に依存しています。サイト側のUI改修によりDOM構造が変更された場合、セレクタの修正対応が必要となります。
2. **公式API不使用**: ジョブメドレーは本用途向けの公式APIを提供していないため、ブラウザ上のDOM操作によるスクレイピング方式を採用しています。
3. **Chrome Webストア非公開**: ローカルインストール（開発者モード）での利用を想定しています。Webストアへの公開は本見積もりに含まれません。
4. **AI生成は含まない**: スカウト文の生成自体は外部AI（Claude、ChatGPT等）で行う前提です。本拡張機能はその入出力のハブとして機能します。
5. **React操作の技術的制約**: 対象サイトがReact + Downshiftを使用しているため、フォーム自動入力にはブラウザネイティブAPIの直接操作が必要です。サイト側のフレームワーク更新により動作に影響が出る可能性があります。
6. **単一ブラウザ対応**: Google Chromeのみを対象とし、Firefox等他ブラウザへの対応は含みません。

## 6. リスク・注意事項

### 6.1 技術リスク

| リスク | 影響度 | 対策 |
|--------|:------:|------|
| ジョブメドレーのDOM構造変更 | 高 | セレクタを専用ファイルに集約し、変更時の修正箇所を局所化済み |
| React/Downshiftバージョンアップによるフォーム入力不具合 | 高 | ネイティブsetter方式 + execCommand方式の二重対策済み。フォールバック処理あり |
| Chrome拡張API（Manifest V3）の仕様変更 | 中 | Googleの非推奨アナウンスに従い都度対応が必要 |
| Content Scriptの動的インポート制限 | 低 | IIFE形式バンドルで回避済み |

### 6.2 運用リスク

- **サイト規約**: スクレイピング・自動操作に関するジョブメドレーの利用規約を事前にご確認ください。
- **アカウント制限**: 短時間での大量操作はアカウント制限の対象となる可能性があります。レートリミット（800ms間隔）を設けていますが、運用時の操作ペースにご留意ください。

### 6.3 保守契約のご提案

本拡張機能はジョブメドレーのDOM構造に強く依存するため、サイト側の更新に伴う修正対応が定期的に発生する可能性があります。安定した運用のため、以下の保守契約をご検討ください。

| プラン | 内容 | 月額（税別） |
|--------|------|:-----------:|
| ライト | セレクタ修正・軽微なバグ対応（月2時間まで） | 15,000円 |
| スタンダード | 上記 + 機能改修相談・優先対応（月4時間まで） | 30,000円 |
