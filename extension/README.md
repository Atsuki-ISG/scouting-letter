# Job Medley Scout Assistant - 取扱説明書

## 全体フロー

```
Step 1: プロフィール抽出  →  CSV出力
Step 2: AIでスカウト文生成  →  CSV出力
Step 3: CSVインポート  →  送信アシスト
Step 4: メッセージページ  →  やりとり取得・分析
```

---

## 事前準備

### 拡張機能のインストール
1. `npm run build` でビルド
2. Chrome → `chrome://extensions` → デベロッパーモードON
3. 「パッケージ化されていない拡張機能を読み込む」→ `extension/dist/` を選択

### サイドパネルを開く
- 拡張機能アイコンをクリック →「サイドパネルを開く」
- またはジョブメドレーのページでアイコンをクリック

### 会社の選択
- サイドパネル上部のドロップダウンで会社を選択
- 現在対応: `ark-visiting-nurse`

---

## Step 1: プロフィール抽出

**場所**: サイドパネル →「プロフィール抽出」タブ

### 手順
1. ジョブメドレーの**候補者検索ページ**を開く
2. 抽出件数を入力（1〜50、デフォルト5）
3. 「抽出開始」をクリック
4. 自動で候補者カードを1件ずつクリック → プロフィール抽出 → 閉じる を繰り返す
5. 進捗バーで進行状況を確認
6. 完了後「CSVダウンロード」で保存

### 出力CSV（profiles_YYYY-MM-DD.csv）
| 列名 | 内容 |
|------|------|
| member_id | 会員番号 |
| gender | 性別 |
| age | 年齢 |
| area | 居住地 |
| qualifications | 保有資格 |
| experience_type | 経験職種 |
| experience_years | 経験年数 |
| employment_status | 現在の就業状況 |
| desired_job | 希望職種 |
| desired_area | 希望勤務地 |
| desired_employment_type | 希望雇用形態 |
| desired_start | 希望入職時期 |
| self_pr | 自己PR |
| special_conditions | 特記事項 |
| work_history_summary | 職務経歴概要 |

### 途中停止
- 「停止」ボタンで中断可能（抽出済みデータは保持される）

---

## Step 2: AIでスカウト文を一括生成

**場所**: ChatGPT / Claude.ai などのチャットAI

### 手順
1. `extension/prompts/[会社名]/` にあるプロンプトをコピー
   - `scout-batch.md` → パート初回
   - `scout-batch-resend.md` → パート再送
   - `scout-batch-seishain.md` → 正社員初回
   - `scout-batch-seishain-resend.md` → 正社員再送
2. AIにプロンプトを貼り付け
3. Step 1で取得したCSVを貼り付け（またはファイルアップロード）
4. AIが生成したCSVをコピー → テキストファイルとして `.csv` で保存

### AIが出力するCSV
| 列名 | 内容 |
|------|------|
| member_id | 会員番号 |
| template_type | テンプレート種別 |
| personalized_text | パーソナライズ文 |
| full_scout_text | スカウト文全文 |

### 別の方法: Claude Code で生成
```
/generate-scout [会社名]
```
→ スクリーンショットから1件ずつ生成（より高品質）

## Step 3: 送信アシスト

**場所**: サイドパネル →「送信アシスト」タブ

### 手順
1. 「CSVインポート」→ Step 2で作成したCSVを選択
2. 候補者一覧が表示される
3. 各候補者に対して:
   - **プレビュー** をクリック → 文面を確認・編集
   - **コピー** → スカウト文をクリップボードにコピー
   - **本文にセット** → ジョブメドレーのスカウト画面を自動で開き、テキストエリアに自動入力
   - ジョブメドレー上で内容を確認し、手動で送信
   - **送信済み** をクリックしてステータス更新
   - 不要な候補者は **スキップ**
4. 全件送信後、修正があれば「修正履歴エクスポート」で保存

### ステータス
- 🔵 未送信（ready）
- ✅ 送信済み（sent）
- ➖ スキップ（skipped）

### 修正した場合
- 文面を編集すると修正記録が自動保存される
- 「修正履歴エクスポート」→ Markdownファイルで出力
- Claude Code で `/integrate-feedback` を使って学習に反映可能

---

## Step 4: やりとり取得

**場所**: サイドパネル →「返信・やりとり」タブ

### スカウト送信後、返信があった場合にやりとりを記録する機能

### 自動取得（単体）
1. ジョブメドレーの**メッセージページ**を開く
2. 左サイドバーで会話を選択（右側にメッセージが表示された状態にする）
3. 「やりとりを取得」をクリック
4. 自動で企業/求職者のメッセージ・日付を抽出して保存

### 自動取得（一括）
1. メッセージページを開く
2. 「一括取得」をクリック
3. サイドバーの全会話が順番にクリックされ自動抽出される
4. 進捗: `取得中... 5/34件 (03122692)` と表示
5. 「中断」で途中停止可能

### 手動入力
1. 「手動入力」をクリックしてフォームを表示
2. 会員番号を入力
3. やりとり内容をテキストで入力:
   ```
   2026-03-01
   企業: はじめまして。スカウトのご連絡です...
   求職者: ご連絡ありがとうございます...
   企業: ご返信ありがとうございます...
   ```
4. 「保存」をクリック

### エクスポート
- **YAML DL**: 個別のやりとりをYAMLファイルでダウンロード
- **全件YAMLエクスポート**: 全やりとりを一括ダウンロード
- **返信スカウト記録エクスポート**: 返信があったスカウトの分析用データ

### YAML出力例
```yaml
company: "ark-visiting-nurse"
member_id: "03114222"
started: "2026-02-26"
messages:
  - role: "company"
    date: "2026-02-26"
    text: "はじめまして..."
  - role: "candidate"
    date: "2026-02-26"
    text: "ご連絡ありがとうございます..."
```

## ナレッジの蓄積・改善

### 修正をナレッジに反映
スカウト文を修正した場合:

**Claude Code内で生成した場合:**
```
/integrate-feedback [会社名] [会員番号] [修正理由] [修正前後]
```

**チャットAIで生成した場合:**
1. AIが出力した「learnings.md追加用」セクションをコピー
2. Claude Code で `/add-learning` を実行
3. コピーした内容を貼り付け

### 良い例の保存
特に効果的だったスカウト文を蓄積:
```
/save-example [会社名] [タイトル] [本文]
```

## トラブルシューティング

| 症状 | 対処法 |
|------|--------|
| Content Scriptと通信できません | メッセージページをリロード（F5） |
| 抽出に失敗しました | 会話を選択してから再度「やりとりを取得」 |
| 拡張機能更新後に動かない | chrome://extensions でリロード → ページもリロード |
| CSVが読めない | UTF-8エンコーディングで保存されているか確認 |
| 「本文にセット」が動かない | ジョブメドレーの候補者検索ページで実行しているか確認 |
| companyが空になる | サイドパネル上部のドロップダウンで会社を選択 |

## ディレクトリ構成

```
extension/
├── dist/                  # ビルド出力（Chromeはここを読み込む）
├── src/
│   ├── background/        # Service Worker
│   ├── content/           # Content Script（ページ操作）
│   ├── sidepanel/         # サイドパネルUI
│   ├── popup/             # ポップアップUI
│   └── shared/            # 型定義・storage・定数
├── prompts/               # バッチ生成用プロンプト
├── scripts/               # ビルドスクリプト
└── icons/                 # 拡張機能アイコン
```
