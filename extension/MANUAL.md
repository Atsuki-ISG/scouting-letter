# Scout Assistant 取扱説明書（オペレーター向け）

## この拡張機能について

Scout Assistant は、ジョブメドレーでのスカウト送信作業をサポートするChrome拡張機能です。

```
📋 プロフィール抽出 → 🤖 スカウト文の一括生成 → 📤 確認しながら連続送信
```

**主な機能**:
- ジョブメドレーの候補者一覧から、プロフィール情報を自動で読み取ります（最大50件）
- 読み取った情報をもとに、AIが候補者ごとのスカウト文を自動生成します
- 生成されたスカウト文を1件ずつ確認しながら、連続で送信できます

**画面構成**: サイドパネル（ブラウザの右側に表示）に5つのタブがあります。

| タブ | 役割 |
|------|------|
| 📋 抽出 | 候補者のプロフィールを読み取る |
| 🤖 API生成 | スカウト文を一括生成する |
| 📤 送信 | スカウト文を確認・編集・送信する |
| 💬 返信 | 候補者とのやりとりを取得・保存する |
| ⚙ 設定 | APIの接続先やパスワードを設定する |

> 初期設定（インストール・API接続・会社登録等）は管理者が行います。手順は管理者向けセットアップガイドを参照してください。

---

## サイドパネルを開く

1. Chromeツールバーの **Scout Assistant アイコン** をクリック
2. ポップアップが表示されるので、**「サイドパネルを開く」** ボタンを押す
3. ブラウザの右側にサイドパネルが開きます

> サイドパネルは一度開くと、別のタブに移動しても表示されたままです。

## 会社を選ぶ

サイドパネル上部の **「会社」** ドロップダウンで、スカウトを送る会社を選択してください。

---

## 基本操作：抽出→生成→送信

### Step 1: プロフィール抽出（📋 抽出タブ）

**前提**: ジョブメドレーの候補者検索ページを開いた状態で操作してください。

| 項目 | 説明 |
|------|------|
| **開始会員番号** | 途中から抽出したい場合に入力（空欄＝先頭から） |
| **抽出件数** | 1〜50件（デフォルト5件） |

**手順**:

1. 件数を設定
2. **「抽出開始」** を押す
3. 拡張機能が候補者カードを順番にクリックし、プロフィールを自動で読み取ります
4. 進捗バーで「進捗: X/Y」が表示されます
5. 完了後、候補者一覧が表示されます

**途中停止**: **「停止」** ボタンで中断できます。抽出済み分は保持されます。

**CSVダウンロード**: 抽出完了後に **「CSVダウンロード」** ボタンが表示されます。外部ツールでスカウト文を生成する場合に使用します。

---

### Step 2: API生成（🤖 API生成タブ）

抽出したプロフィールからスカウト文をAIで一括生成します。

**画面表示**: 「抽出済み: N件」で対象件数を確認できます。

**手順**:

1. **「一括生成」** を押す
2. **生成設定モーダル** が表示されます。必要に応じて設定を変更してください:

| 設定項目 | 説明 |
|-----------|------|
| **会社** | サイドパネル上部と連動（変更可） |
| **雇用形態** | 自動判定 / パート / 正社員 から選択 |
| **職種カテゴリ** | 全て / 看護師 / PT / ST / OT / 医療事務 から選択 |
| **送信タイプ** | 初回 / 再送 から選択 |

> 前回の設定が保存されています。変更がなければそのまま生成できます。

3. モーダル内の **「一括生成」** を押す
4. 「生成中... X/Y」の進捗バーが表示されます
5. 完了後、結果サマリーが表示されます:
   - **合計**: 処理した総数
   - **AI生成**: 経歴情報からAIが文章を生成した件数
   - **型はめ**: 定型パターンで生成した件数
   - **除外**: バリデーションで対象外になった件数
6. 生成結果は自動で **📤 送信** タブの候補者リストにセットされます

---

### Step 3: 送信（📤 送信タブ）

生成されたスカウト文を確認し、ジョブメドレーから送信します。

#### 候補者リストの見方

各候補者の行には以下が表示されます:

| 表示 | 意味 |
|------|------|
| 🔵 | 未送信（送信待ち） |
| ✅ | 送信済み |
| ➖ | スキップ |
| テンプレートバッジ | パート/正社員、初回/再送の種別 |
| ⚠ 警告バッジ | バリデーション警告（送信可能だが注意が必要） |
| ❌ エラーバッジ | バリデーションエラー（対象外候補者） |

#### 対象求人の自動選択

**「求人自動選択」** がONの場合、候補者の職種カテゴリと雇用形態に応じて適切な求人が自動選択されます。

自動選択の仕組み:
- ジョブメドレーの求人ドロップダウンを検索キーワードで開く
- ドロップダウン内の求人テキストを、職種カテゴリ（看護師、理学療法士 等）＋雇用形態（パート、正職員 等）のキーワードで照合
- 両方一致する求人が優先、なければ職種カテゴリのみで選択、1件のみの場合はそのまま選択

うまく選択されない場合は **「求人自動選択」** をOFFにして、手動で求人を選んでください。

#### 手動送信（1件ずつ）

各候補者の行にあるボタンで操作します:

| ボタン | 動作 |
|--------|------|
| **コピー** | スカウト文全文をクリップボードにコピー |
| **本文セット** | ジョブメドレーのスカウト画面にテキストを自動入力 |
| **求人+本文** | 求人選択＋テキスト入力を同時に実行 |
| **送信済** | ステータスを「送信済み」に変更 |
| **スキップ** | この候補者をスキップ |

**「求人+本文」の流れ**:
1. ボタンを押すとジョブメドレーのスカウトフォームが開きます
2. 求人が自動選択され、テキストエリアにスカウト文が自動入力されます
3. 確認ポップアップが表示されます:
   - 候補者情報（資格、経験、希望雇用、エリア、自己PR、職歴）
   - 会員番号、対象求人、パーソナライズ文、全文プレビュー
4. チェックリスト（会員番号・求人・パーソナライズ文）を確認してチェック
5. **OK（送信する）** → ジョブメドレーの送信ボタンをクリックして送信
6. **NG（スキップ）** → この候補者をスキップ

#### 連続送信アシスト

複数の候補者を順番に処理する場合に使います。

1. **「連続送信アシスト」** トグルをONにする → **「開始」** ボタンが表示される
2. **「開始」** を押す
3. 最初の未送信候補者から順番に確認ポップアップが表示されます
4. 各候補者ごとに:
   - **OK** → 送信して次の候補者へ
   - **NG** → スキップして次の候補者へ
5. 全件処理すると完了通知が表示されます

**途中停止**: 連続送信中は確認ポップアップ内に **「連続送信を停止」** ボタンが表示されます。サイドパネルの **「停止」** ボタンでも中断できます。

**スキップ**: サイドパネルの **「スキップ」** ボタンで現在の候補者を飛ばして次へ進みます。

#### 文面の編集

候補者リストのパーソナライズ文プレビューをクリックすると、インライン編集モードになります。

- テキストを修正して保存できます
- 修正理由を入力するフィールドもあります（任意）
- 修正すると修正記録が自動保存されます
- **「修正履歴エクスポート」** ボタンで修正内容をMarkdown形式でエクスポートできます

確認ポップアップ内でも直接テキストを編集できます。

#### ドライランモード

**「ドライランモード」** をONにすると、実際には送信せずに送信フローをシミュレートします。初回利用時の練習や動作確認に使用してください。

#### デバッグログ

**「デバッグログ」** をONにすると、各ステップの詳細ログが表示されます。問題発生時のトラブルシューティングに使用します。

## CSVインポートによる送信（代替フロー）

外部ツール（Claude.ai、ChatGPT等）で生成したスカウト文を送信する場合に使用します。

### インポート方法

**方法1: CSVファイル選択**

1. 📤 送信タブの **「CSVファイル」** ボタンを押す
2. ファイル選択ダイアログでCSVファイルを選択

**方法2: テキスト貼り付け**

1. **「貼り付け」** ボタンを押す → テキストエリアが表示される
2. CSVテキストを貼り付け（コードブロック ` ```csv ``` ` 付きでもOK）
3. **「インポート」** ボタンを押す

### CSVの必須列

| 列名 | 説明 |
|------|------|
| `member_id` | 会員番号 |
| `template_type` | テンプレート種別（例: `part_first`, `seishain_first`） |
| `personalized_text` | パーソナライズ文（AI生成部分） |
| `full_scout_text` | スカウト文全文（テンプレート＋パーソナライズ文） |

**クリア**: **「クリア」** ボタンで候補者リストを全て削除できます。

## やりとり取得（💬 返信タブ）

候補者との返信やりとりを取得・保存し、分析用にエクスポートします。

### 自動取得（1件）

1. ジョブメドレーのメッセージページで対象の会話を開く
2. **「やりとりを取得」** を押す
3. 現在表示中の会話が自動で読み取られます

### 自動取得（一括）

1. ジョブメドレーのメッセージページを開く
2. **「一括取得」** を押す
3. サイドバーの会話リストを順番に自動取得します
4. 「取得中... X/Y件」で進捗が表示されます

### 手動入力

1. **「手動入力」** を押す → 入力フォームが表示される
2. **会員番号** を入力
3. **やりとり内容** を以下のフォーマットで入力:

```
2026-03-01
企業: はじめまして...
求職者: ご連絡ありがとうございます...
企業: ご返信ありがとうございます...
```

4. **「保存」** を押す

### エクスポート

| ボタン | 出力内容 |
|--------|---------|
| **全件YAMLエクスポート** | 全会話をYAMLファイルとしてダウンロード |
| **返信スカウト記録エクスポート** | 分析用の構造化データとしてエクスポート |

## 設定（⚙ 設定タブ）

通常のオペレーション時は設定変更不要です。管理者がセットアップ済みの状態で使用します。

### API設定

| 項目 | 説明 |
|------|------|
| **APIエンドポイント** | Cloud RunサーバーのURL |
| **パスワード** | API認証用パスワード |
| **接続テスト** | 接続が正常か確認（緑=OK、赤=NG） |

### GAS連携

| 項目 | 説明 |
|------|------|
| **有効化** | GAS（Google Apps Script）連携のON/OFF |
| **GAS Web App URL** | GASデプロイURL |
| **接続テスト** | GASへの接続確認 |

---

## トラブルシューティング

| 症状 | 対処法 |
|------|--------|
| 「Content Scriptと通信できません」エラー | ジョブメドレーのページをリロードしてください |
| API接続テスト失敗 | エンドポイントURLとパスワードが正しいか管理者に確認してください |
| 一括生成ボタンが押せない | 先に📋 抽出タブでプロフィールを抽出してください |
| 生成結果が全て「除外」 | 管理者にバリデーション設定を確認してもらってください |
| 求人自動選択が失敗する | 「求人自動選択」をOFFにして手動で求人を選択してください |
| 連続送信が途中で止まる | ジョブメドレーのページがアクティブか確認してください |
| 本文セットしてもテキストが入力されない | ページをリロードしてから再試行してください |
| 抽出中にエラーが出る | 候補者検索結果ページを表示した状態で実行してください |
| 拡張機能が動作しない | `chrome://extensions` で拡張機能が有効か確認してください |

## 用語集

| 用語 | 説明 |
|------|------|
| **パーソナライズ文** | 候補者の経歴に合わせてAIが生成する個別メッセージ部分 |
| **テンプレート** | スカウト文の外枠（挨拶・会社紹介・締め等）。パーソナライズ文が挿入される |
| **型はめ** | 経歴情報が少ない候補者向けに、定型パターンから機械的に文章を生成する方式 |
| **AI生成** | 経歴情報がある候補者向けに、AIが個別にパーソナライズ文を生成する方式 |
| **バリデーション** | 候補者が送信対象かどうかを判定するルール（年齢・資格等） |
| **求人自動選択** | 職種カテゴリと雇用形態に応じて、適切な求人を自動的に選択する機能 |
| **ドライラン** | 実際には送信せず、送信フローの動作だけを確認するテストモード |
| **GAS連携** | Google Apps Scriptを使った送信記録の連携（オプション機能） |
| **職種カテゴリ** | 資格から自動分類される職種（看護師、PT、ST、OT、医療事務） |
