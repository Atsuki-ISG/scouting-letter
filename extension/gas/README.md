# GAS連携セットアップガイド

スカウト送信ログをGoogleスプレッドシートに自動記録する機能。
送信済みマークを押したタイミングで、バックグラウンドでスプレッドシートに1行追記される。

## 全体の流れ

```
Chrome拡張（送信済みマーク）
  ↓ fetch POST（fire-and-forget）
GAS Web App（doPost）
  ↓ appendRow
Googleスプレッドシート「送信ログ」シート
```

- 拡張側では送信フローをブロックしない（失敗してもconsole.warnだけ）
- GAS無効時やURL未設定時はリクエスト自体を送らない

---

## 1. スプレッドシートを作成

1. [Google Sheets](https://sheets.google.com) で新規スプレッドシートを作成
2. スプレッドシートのURLからIDをコピー
   - `https://docs.google.com/spreadsheets/d/【ここがID】/edit`
3. シートは自動作成されるので、事前準備は不要

### 記録される列

| 列 | 内容 | 例 |
|----|------|-----|
| timestamp | 送信日時（ISO 8601） | 2026-03-06T10:30:00.000Z |
| member_id | 会員番号 | 03122692 |
| company | 会社名 | ark-visiting-nurse |
| job_offer_id | 求人ID | 1550716 |
| job_offer_label | 求人名 | 看護師 パート |
| template_type | テンプレート種別 | part_first |
| personalized_text | パーソナライズ文 | 10年以上の病棟ご経験... |

---

## 2. GASプロジェクトを作成

1. [Google Apps Script](https://script.google.com) で新規プロジェクト作成
2. デフォルトの `Code.gs` の内容を全削除
3. `extension/gas/Code.gs` の内容をコピー&ペースト
4. **14行目** の `SPREADSHEET_ID` を、手順1で取得したIDに書き換え：
   ```javascript
   const SPREADSHEET_ID = 'あなたのスプレッドシートID';
   ```
5. 保存（Ctrl+S）

---

## 3. Web Appとしてデプロイ

1. GASエディタ右上の「デプロイ」→「新しいデプロイ」
2. 種類: **ウェブアプリ**
3. 設定:
   - **説明**: Scout Assistant（任意）
   - **次のユーザーとして実行**: **自分**
   - **アクセスできるユーザー**: **全員**
4. 「デプロイ」をクリック
5. 初回はGoogleアカウントの認証が求められる → 許可
6. 表示された **ウェブアプリURL** をコピー
   - `https://script.google.com/macros/s/xxxxx.../exec` の形式

> **注意**: デプロイURL末尾の `/exec` まで含めてコピーすること

---

## 4. Chrome拡張に設定

1. Chrome拡張のアイコンをクリック → ポップアップが開く
2. 下部の「GAS連携」セクション:
   - **有効化** にチェック
   - **GAS Web App URL** に手順3のURLを貼り付け
3. **「接続テスト」** ボタンをクリック
   - 「接続成功」と表示されれば OK
   - 失敗時はURLの確認・GASのデプロイ設定を再確認

---

## 5. 動作確認

1. サイドパネルでCSVインポート → 候補者をセット
2. 任意の候補者で「送信済み」ボタンをクリック
3. スプレッドシートを開き、「送信ログ」シートにデータが追記されていることを確認

---

## トラブルシューティング

### 接続テストが失敗する
- URLが `/exec` で終わっているか確認
- GASのデプロイで「アクセスできるユーザー」が「全員」になっているか確認
- GASを修正した場合は**新しいデプロイ**が必要（「デプロイを管理」→「新しいバージョン」）

### データが記録されない
- ポップアップで「有効化」チェックが入っているか確認
- Chrome DevTools → Console に `[Scout Assistant] GAS log failed:` のエラーが出ていないか確認
- GASの「実行数」メニューでエラーログを確認

### GASのコードを変更した場合
GASコードを編集しただけでは反映されない。必ず：
1. 「デプロイ」→「デプロイを管理」
2. 右上の鉛筆アイコン → バージョンを「新バージョン」に変更
3. 「デプロイ」をクリック

URLは変わらないので、拡張側の再設定は不要。

---

## 拡張ポイント（将来）

現在の `Code.gs` は送信ログの記録のみ。以下の拡張が可能：

- **返信記録**: `doPost` に `action: 'logReply'` ハンドラを追加
- **テンプレート管理**: `doGet` でスプレッドシートからテンプレートを返す
- **分析データ取得**: `doGet` で送信数・返信率の集計を返す
