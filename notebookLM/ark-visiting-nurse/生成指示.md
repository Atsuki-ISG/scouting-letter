# NotebookLM スカウト文生成指示

## このノートブックの使い方

アップロード済みソース:
- **recipes.md** — 型はめパターン・共通ルール・AI生成ガイド（最重要）
- **templates.md** — テンプレート本体・行動指針14項目・フィルタリング条件
- **profile.md** — アーク訪問看護ステーションの会社情報
- **examples.md** — 過去の良いパーソナライズ文の例

---

## 処理パイプライン

CSVデータを受け取ってから出力するまでの全手順。

### ステップ1: フィルタリング（除外判定）

以下の順で除外する。除外した候補者はCSV出力に含めず、末尾に一覧表示する。

1. **送信済み除外**: `scout_sent_date` 列に値がある → 除外（理由: 「スカウト送信済み」）
2. **資格チェック**: 看護師または准看護師の資格を保有していない → 除外
3. **非臨床職チェック**: 経験が保健師・事務職・生活支援員等の非臨床職のみで、看護師としての臨床経験が一切ない → 除外
   - ただし経験未入力で就業中の場合は臨床経験ありと推定し、対象とする

### ステップ2: テンプレート判定（候補者ごと）

`desired_employment_type`（希望雇用形態）に基づき判定:

| 条件 | template_type |
|------|--------------|
| 「正職員」を含む（「パート・バイト」も同時に含んでいても） | `正社員_初回` or `正社員_再送` |
| 「パート・バイト」のみ（正職員を一切含まない） | `パート_初回` or `パート_再送` |

初回/再送はプロンプトで指定する。

### ステップ3: 生成パスの判定（候補者ごと）

候補者の `work_history_summary`（職務経歴）と `experience_type`（経験職種）を確認:

| 条件 | 生成パス |
|------|---------|
| いずれかに実質的な情報あり（「未入力」以外） | **AI生成パス**（ステップ4Aへ） |
| どちらも情報なし | **型はめパス**（ステップ4Bへ） |

### ステップ4A: AI生成パス（経歴あり）

recipes.md の「AI生成パス用ガイド」に従う:

1. 候補者の経歴・自己PRから最も強い接点を**1つ**選ぶ
2. recipes.md の「経歴別の接点対応表」から対応するステーション特色を選ぶ
3. 接点を約100文字（再送は120〜150文字）で表現
4. examples.md の良い例を参考にする

**経験年数ルール**: 4年以上の候補者にはパーソナライズ文中に具体的な経験年数を含める

**NGパターン**:
- 運転免許など弱い接点で文を構成する
- 会社情報を羅列するだけで接点になっていない
- 地理的要素のみに頼る

### ステップ4B: 型はめパス（経歴なし）

recipes.md の「型はめパターン」に従う:

**1. 資格修飾の確認**: 複数資格（保健師、ケアマネ等）があれば「資格修飾」セクションの文を使用 → マトリクス不要

**2. マトリクスで型を選択**:

| | 経験10年+ | 経験6〜9年 | 経験3〜5年 | 経験1〜2年 | 経験なし/未入力 |
|---|---|---|---|---|---|
| **40代〜** | 型A | 型A | 型B2 | 型C | 型D |
| **20〜30代** | 型A | 型B1 | 型B2 | 型E | 型F |
| **在学中** | — | — | — | — | 型G |

**3. 変数差し替え**:
- `{N}` → 経験年数
- `{特色}` → バリエーションからローテーション（候補者ごとに異なる特色を選び、同じ文の連続を防ぐ）

**4. 就業状況切り替え**: 型D・型Fは `employment_status`（就業中/離職中）で文面を切り替え

**5. 自己PRによる特色差し替え（該当時のみ）**:
- `self_pr` に実質的な内容あり → 特色バリエーションを自己PRに合ったものに固定
- `self_pr` が抽象的（「頑張ります」等）→ 差し替えせずそのまま

### ステップ5: テンプレートへの組み込み

templates.md の該当テンプレート（パート初回/再送/正社員初回/再送）の `{ここに生成した文章を挿入}` にパーソナライズ文を挿入して全文を完成させる。

### ステップ6: 出力

**CSVのみを出力。説明文・見出し・Markdown記法は一切不要。**

ヘッダー:
```
member_id,template_type,personalized_text,full_scout_text
```

CSVルール:
- カンマや改行を含む値は必ずダブルクォート(`"`)で囲む
- `full_scout_text` は改行を含むため必ずダブルクォートで囲む
- ダブルクォート内のダブルクォートは `""` でエスケープ

除外した候補者がいる場合、CSV出力の後に:
```
除外: [N]件
- [member_id]: [除外理由]
```

## 共通ルール（全ステップで適用）

recipes.md「共通ルール」の要約:

- **書き出し**: テンプレート冒頭に「ご経歴を拝見し」があるため「拝見し」以外で開始
- **文字数**: 初回 約100文字 / 再送 120〜150文字
- **年齢・世代への言及禁止**: 「〇代」「若手」「ベテラン」一切不可
- **経験年数の記載ルール**:
  - 1〜2年: 書かない
  - 3年: 30代前半以下なら書く、30代後半以上なら書かない
  - 4年以上: 書く
- **地理情報**: テンプレートに記載済みのため言及しない。居住地は広域表現（「札幌市内」）
- **上から目線の排除**: 「フォローします」「安心してください」「基礎を固めてこられた」禁止
- **送り手感情主体の排除**: 「ご一緒したい」「想いを強く感じ」→ 相手へのオファー主体で
- **憶測禁止**: 「〜されたいのですね」「長く続けているはず」
- **不要な前置き禁止**: 「現在離職中とのこと」「新たな環境でとお考えの〜」
- **堅苦しい敬語禁止**: 「拝察いたします」「敬意を表します」
- **ですます調で統一**: 体言止めや「〜とのこと、」で文を切らない
- **「感銘を受ける」使用制限**: 経歴そのものに使用禁止。代替:「心強く拝見」「注目しました」
- **こだわり条件**: 資格・経験で接点が作れるなら言及不要

## 検証チェックリスト

生成結果を確認する際のポイント:

- [ ] フィルタリング: 資格なし・非臨床職のみの候補者が正しく除外されているか
- [ ] テンプレート判定: 正職員希望の候補者に正社員テンプレートが使われているか
- [ ] 生成パス判定: 経歴ありはAI生成、経歴なしは型はめになっているか
- [ ] 型選択: 経験年数×年齢のマトリクスで正しい型（A〜G）が選ばれているか
- [ ] 経験年数の記載ルール: 1〜2年は非表示、3年は年齢で判断、4年+は表示
- [ ] 年齢・世代への直接的な言及がないか
- [ ] 地理情報（〇〇区など）が含まれていないか
- [ ] 上から目線の表現がないか（「フォローします」「基礎を固めてこられた」等）
- [ ] CSV形式が正しいか（ダブルクォート、改行のエスケープ）
- [ ] パーソナライズ文が約100文字か（再送は120〜150文字）
- [ ] 特色バリエーションがローテーションされているか（同じ文の連続がないか）
- [ ] 型D/型Fで就業中/離職中の文面切り替えが正しいか
