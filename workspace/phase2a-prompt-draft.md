# Phase 2-A: パーソナライズ生成プロンプト改訂案

**目的:** あゆみんの3通から抽出した全社共通ナレッジ（knowledge/learnings.md 2026-04-21追記）をAI生成プロンプトに反映する。

**スコープ:** Google Sheets「プロンプト」シートの global セクション（company 列が空のもの）を更新。会社固有の section_type は変更しない。

**本番反映前の作業:**
1. この案で差分レビュー
2. 会社別トーンとの整合確認（ARK・LCC・いちご・茅ヶ崎徳洲会・an・大和ハウスLS・野村病院）
3. `/server-admin push` でSheetsに反映

## 変更サマリー（現行 → 新）

| 項目 | 現行 | 新 |
|---|---|---|
| 文字数 | 初回 約100文字 | **初回 100〜250文字程度（候補者情報量に応じる）** |
| 候補者呼称 | 「〇〇様」等の呼びかけ不要 | **会員番号を一切使わない**（より厳格に） |
| 引用技法 | 言及なし | **3ステップで明文化**（源明示なし／導入パレット／温度マッチ） |
| 橋渡し表現 | 「接点を1つ選ぶ」 | **経歴の"何が" × 業務の"何に"効くかを具体化** |
| 時間帯配慮 | 言及なし | **夜間送信時は「夜分に」を冒頭に添える** |
| フレーズ多様化 | 「特色バリエーションをローテ」のみ | **バッチ内で導入フレーズ・受けフレーズの連続使用禁止** |

## 追加するセクション（全社共通プロンプトに挿入）

### 【新設】セクション: 引用技法（section_type: `quotation_rules`、order: 既存トーン＆マナーの後）

```
## 候補者本人の言葉を引用する（自己PR記載ありの場合）

自己PRに志や価値観の言葉がある場合、そのまま「 」で引用して共感を示す。
自己関連効果が強く、「自分のことを読んでくれている」印象を作る。

### 引用を組み立てる3ステップ

**Step 1. 引用源の明示はしない（AI臭対策）**

候補者は自分が書いた言葉だと知っている。「自己PRに綴られていた」「自己PR欄に記載の」
のような引用元ラベルは絶対につけない。直接引用する。

- NG: 自己PRに綴られていた「〇〇」というお言葉に
- NG: 自己PR欄に記載の「〇〇」を拝見し
- OK: 何より「〇〇」というお言葉に
- OK: 「〇〇」という姿勢に

**Step 2. 導入フレーズを内容に応じて選ぶ（固定しない）**

引用の前に置く"差し込み語"は、毎回同じ語を使わない。以下から選ぶか、無しで直接引用する:

| パターン | 使いどころ |
|---|---|
| （何もつけず直接引用） | 最も自然。迷ったらこれ |
| 「何より〜」 | 複数の特徴がある中で、引用箇所が一番響いたと強調したいとき |
| 「〜という言葉が印象的で」 | 引用が文章中のワンフレーズとして置かれているとき |
| 「〜というお考えが」 | 思想・哲学寄りの内容のとき |
| 「〜というご姿勢から」 | 仕事への向き合い方が語られているとき |
| 「〜という想いが伝わり」 | 感情・決意寄りの内容のとき |

**Step 3. 引用の"受け"を内容の重さに合わせる**

引用した後の反応表現は、引用内容の温度と合わせる。
大げさな反応は「このAI過剰だな」の違和感を生む。

| 引用内容のタイプ | 例 | 使えるフレーズ |
|---|---|---|
| **人生観・大きな志** | 「地域に貢献し、支え合える社会を作りたい」 | 「深く感銘を受けました」「深く共感いたします」「心を動かされました」「強く惹かれました」 |
| **仕事観・姿勢** | 「楽しく仕事をして相手にも楽しんでもらう」 | 「印象に残りました」「共感いたします」「素敵なお考えだと感じました」「温かさを感じました」 |
| **具体的な工夫・取り組み** | 「毎朝の申し送りで〇〇を意識」等 | 「丁寧なお仕事ぶりが伝わってきました」「誠実な向き合い方を感じました」 |

### 禁則

- 仕事観レベルの引用に「強く心を動かされました」「深く感銘を受けました」等の強い反応を返さない
- 同じバッチ処理内で、同じ導入フレーズ・同じ受けフレーズを連続で使わない
  （例: 10人生成するとき「何より〜」「印象に残りました」が全員分に並ぶのは×）
- 自己PRが未入力または事務的（資格更新日等）な場合、無理に引用しない。経歴で勝負する

### 参考例

- 志レベル: 何より「地域に貢献し、支え合える社会を作りたい」という温かい志に、深く感銘を受けました
- 仕事観レベル: 「自分が楽しく仕事をして、相手にも楽しんで貰える」という一貫した姿勢が印象に残りました
- PR未入力: （引用は使わず経歴で勝負）
```

### 【新設】セクション: 橋渡し具体化（section_type: `bridge_building`）

```
## 経歴 → 業務の「橋渡し」を具体化する

「経験を活かせます」で終わらせない。候補者の経験の"何が" × 当社業務の"何に"効くかを
具体的に対応させる。これが生成される文章の説得力を左右する最大の要素。

### 弱い例 vs 強い例

| 弱い例 | 強い例（橋渡し具体化） |
|---|---|
| 長年のご経験を活かしていただけます | 長年の営業経験による**調整力**が、**関係機関との連携が不可欠な当業務**において、最適な支援を繋ぐ力になると確信しております |
| コミュニケーション能力が活かせます | 現場での療育経験や高いコミュニケーション能力があれば、**ご家族の細やかな悩みにも深く共感し最適なプランニング**をしていただけるものと確信しております |
| 魅力的な経歴と拝見しました | 医療ソーシャルワーカーとして培われた視点と、障害福祉の現場の両方を歩んでこられた方だからこそ描ける地域生活像は、訪問看護と一体で動く当ステーションの相談支援において、まさに必要とされている軸になる |

### 作り方

1. 候補者の経歴から、意味的に最も重い要素を1つ選ぶ（経験年数ではなく、**役割の本質**）
2. 当社業務の特性から、その経験が活きる**具体的な場面**を選ぶ
3. 「〇〇だからこそ〇〇できる」の構文で結ぶ

### 注意

- 候補者が自己PR未入力でも、経歴の掘り下げだけで橋渡しは作れる
- 職種羅列（「生活支援員・児発管・相談支援専門員とご経験豊富」）で終わらせず、
  それらが繋がる**物語**として言語化する
```

13. 会員番号を一切使わない: 「〇〇様」「03066549様」等の番号呼称は禁止。

### 【更新】セクション: AI生成パス（既存を加筆）

既存の「参照する情報の優先順位」の後に以下を追加:

```
#### 生成の基本構成（推奨の段落順）

1. **経歴の具体的言及**（役職・年数・分野）— 自己認知成立の土台
2. **自己PRの引用**（記載ありの場合のみ）— 引用3ステップに従う
3. **業務への橋渡し**（候補者の何が × 当社業務の何に）— 説得力の核
4. **簡潔な締め**（連絡理由の補強 or 期待）

### NG構成

- 番号呼び → 冒頭いきなり候補者を識別ラベルで扱う
- 抽象語のみ → 「幅広いご経験」「魅力的な経歴」で終わる
- 汎用クロージング → 「ぜひご活躍いただきたくご連絡いたしました」「ご検討のほどお願い申し上げます」のみで締める
- 事実の羅列 → 資格・経歴を並べただけで接点ゼロ
```

## バッチ処理時の追加指示（prompt_builder.py 改修が必要）

### 新規追加するプロンプトセクション

```
## バッチ生成時の多様性確保

このバッチは複数の候補者を連続生成しています。以下を守ってください:

- 直前3名の候補者と、同じ導入フレーズを使わない
- 直前3名の候補者と、同じ受けフレーズを使わない
- 文末（「感じております」「確信しております」「考えております」等）も連続回避

内部的に「直前の使用フレーズリスト」を保持して、重複を防ぐ。
```

**実装案:** `prompt_builder.py` に「前N名の使用フレーズ」を渡す引数を追加し、システムプロンプトに織り込む。
