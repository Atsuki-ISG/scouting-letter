# スカウト集合知メタ分析（rate-scout 11件 + 自社良い例）

**作成日:** 2026-04-28
**素材:** `training/scout-examples.md`（11件） / `companies/ark-visiting-nurse/history/examples.md`（1件）
**目的:** 集まった他社・他業界スカウトから「自社が見落としている原則」を抽出し、テンプレ改善の入力にする。
**位置づけ:** これは下書き。会社別 templates.md / recipes.md / `knowledge/learnings.md` への反映は次フェーズ（`/review-template` 経由）。

---

## サマリ

- **S評価 3件 / A 3件 / B 4件 / C 1件 / D 0件 / 自社事例なし**。データ偏り: 他社・エンジニア4、他社・医療介護4、人材紹介1、人事向け1、その他1。
- **評価を最も強く分けているのは「冒頭1〜3行のパーソナライズ密度」**。S評価の#11は経歴を3層で積み上げ、S #2は現職会社名を名指し。逆にB以下は全件「ご経歴を拝見し」レベルの定型で止まっている。
- **数字訴求の質が次の分かれ目**。社会的証明（年齢×年数×UP額の4軸 / 入社後改善率TOP5 / OpenWork上位1%）+ 業界平均との相対比較が、強いスカウトの共通武器。
- 自社（ark）の良い例は1件のみで素材が薄い。`/save-example` の運用が回っていない兆候。

## 1. 高評価の型（S/Aから抽出）

### 1.1 冒頭パーソナライズの3パターン

| パターン | 出典 | 中身 | 自社転用イメージ |
|---|---|---|---|
| **3層積み上げ型** | #11 (S) | 「年間160名の採用を牽引／SNS発信を武器／コンセプト〜動画編集まで一貫」 | 「○病棟○年→在宅看取り→新人教育」の3層で書き起こす |
| **現職名指し型** | #2 (S) | 「○○病院様で日々お仕事に励まれながら」 | profile→現職名が拾える時のみ。雑だと逆効果 |
| **掛け合わせ構文** | #11 (S) | 「○○として△△する一方で、◇◇を武器に××してこられた」 | 「在宅看取りを担う一方で、新人指導でチーム質を引き上げてこられた」 |

### 1.2 数字訴求の効きどころ

- **年齢×年数×UP額の4軸**（#1, #8）— 「30歳/Java6年→+600万」は自己関連効果で読み手が自分を投影する
- **転職理由TOP5×入社後改善率**（#7） — 切り口は秀逸（C評価は装飾と宛名で台無しだが学びは強）
- **絶対値＋業界平均との相対値**（#8） — 「定着率91%」より「業界平均◯% vs 当社91%」
- **権威の検証可能性**（#8, #11） — メディアURL、固有名詞＋金額（「元メルカリ日本統括」「199億円」）

### 1.3 CTAハードル下げの鉄板表現

- 「**まだ転職検討中の段階でも問題ありません**」（#9） — 潜在層を取りこぼさない
- 「**聞くだけOK／顔出しなしOK／スマホだけOK**」3点セット（#10）
- 「**相互理解の場**」フレーミング（#6） — 上下関係を作らない
- **面談フローの分単位開示**（#6） — 「自己紹介2-3分→質問20-30分→会社説明10-20分」で不安を先回り

### 1.4 構造系の鉄板

- **業務の箇条書き5項目**（#3） — 訪問看護なら「バイタル管理／服薬管理／家族対応／多職種連携／看取り対応」
- **条件一括提示**（#4, #5, #10） — 駅／残業／給与／賞与／休日／社宅／祝い金
- **悩み共感→施設→具体エピソード→CTA の4段構成**（#10）

### 1.5 エピソード挿入（#10の切り札）

- 結婚式サプライズ（タキシード&ドレスで息子夫婦が居室訪問）
- 「**看護師を目指したときに、やりたかった看護**」黄金フレーズ
- 訪問看護版の候補：「最期の誕生日をご自宅でご家族と囲んだ」「○年通った利用者様を最期まで」等の実話

### 1.6 再送特化の作法（#8）

- 「**書類選考免除**」のような再送だけの特典提示
- 前回との差分を1点に絞る（情報追加でなく、特典追加）
- 冒頭で「再度ご連絡」「前回内容を再掲」と接続を明示

## 2. 低評価・反面教師パターン

### 2.1 致命傷リスト

| 問題 | 出典 | 何が起きるか |
|---|---|---|
| 装飾記号・絵文字過多（🌟✨🌷📤 / ●〇★◇▼◎■☆） | #4, #5, #7, #8, #10 | 専門職向けではチープ印象、データ訴求すら殺す |
| **会員番号を本文宛名に流用**（「[会員番号]様」） | #6, #7, #10 | 差し込み機能の素が見え、親しみのつもりが機械感を増幅 |
| 根拠なし「ぴったりだと感じた」「ご活躍頂けると思い」 | #1, #4, #5, #6, #10 | A6違反の典型。情報量を盛っても冒頭1行で冷める |
| **冒頭タイトルの誤字**（#6 「栄養を資格を活かしお客さを」） | #6 | 一瞬で信頼喪失。テンプレ更新時の必須プレビューゲート要 |
| スクロール必須の長文 | #1, #5, #8 | 中盤で離脱。情報濃度に関係なく長さは罪 |
| 「未経験OK」過剰アピール | #4 | 経験者には特別感が消えて逆効果 |
| 「匿名情報を元に送付」と「ぜひ一度ご案内したい」の併記 | #5 | 自己矛盾。トーン不一致を自ら露呈 |

### 2.2 自社運用への警告

- アーク等で**会員番号宛名フォールバック**を使っている箇所があれば即廃止候補。差し込めない時は「無宛名」が正解。
- 装飾記号・絵文字の本文使用は、医療介護専門職向けでは1〜2個まで（保育・栄養士の一部例外を除く）。

## 3. 自社（医療介護）に効きそうな転用候補（優先度付き）

### A. すぐ実装したい（コスト低 × 効果高）

1. **「まだ転職検討中の段階でも問題ありません」一文** — CTAハードル下げ。各社テンプレ末尾に挿入候補
2. **「聞くだけOK／顔出しなしOK／スマホだけOK」3点セット** — オンライン面談訴求の即補強材
3. **業務の箇条書き5項目** — 訪問看護版「バイタル／服薬／家族対応／多職種連携／看取り」を recipes.md に組み込み
4. **条件一括提示の粒度を揃える** — 「月給／訪問手当／オンコール手当／直行直帰」を全社で同レベルで列挙
5. **会員番号宛名フォールバックの撲滅チェック** — テンプレ・パターン横断で grep して残存があれば削除

### B. 試す価値あり（コスト中 × 効果中〜高）

6. **掛け合わせ構文の導入** — 「○○として△△する一方で、◇◇を武器に××してこられた」を冒頭フックの選択肢に追加
7. **面談フロー分単位開示** — 「見学の流れ：施設案内15分→先輩看護師との対話20分→質疑10分」を病棟経験者向け再送に
8. **エピソード型テンプレートの追加**（最重要候補） — 訪問看護向けに「実話1エピソード＋黄金フレーズ」型を作る。会社プロフィールに実話エピソードを集める運用と一体
9. **再送特典の追加** — 「今回特別に見学のご案内」等、再送だけの+αを設計
10. **業界平均との相対値訴求** — 「訪問看護業界離職率15% vs 当社◯%」のような形で全社プロフィールに数字を整備

### C. 中長期に検討（コスト高 × 効果不確実）

11. **転職理由TOP5×入社後改善率データの自社版** — 返信ログから集計可能。武器化に値するが調査設計が必要
12. **権威の検証可能化** — 認定ページURL、訪問件数実績、満足度調査の方法明示
13. **「看護師を目指したときに、やりたかった看護」黄金フレーズ** — 強力だが多用すると陳腐化。現状未採用の会社限定で試す

## 4. データ取得の課題

- **自社スカウトの rate-scout 評価がゼロ件**。他社評価ばかりで自社の現在地と直接比較できない。
- `companies/*/history/examples.md` は ark 1件のみ。`/save-example` の運用が回っていない。
- 返信率データ（`/analyze-replies`）との突合は別タスクとして必要。質的パターン（本ドキュメント）と量的データ（返信率）が両輪。

### 推奨アクション

- ディレクター業務に「**毎週送信文を3通 rate-scout でセルフ評価する**」を組み込み、自社サンプルを蓄積
- 返信が来たスカウトを優先的に `save-example` で保存する運用ルール化

## 5. 自社送信データ分析（YML 177件・2026-04-28 追加）

7社の `companies/*/history/conversations/*.yml` に shu された paired conversation（初回スカウト + 候補者返信）177件を分析。

**返信センチメント分布:**

| カテゴリ | 件数 | 内訳 |
|---|---|---|
| 興味あり | 135 | うち深い対話103・1往復32 |
| 辞退(丁寧) | 14 | 理由を添えて辞退 |
| 辞退 | 12 | 短文の定型辞退 |
| 質問 | 8 | 条件確認 |
| その他 | 8 | |

注: これは「返信した中での内訳」。送信全体に対する返信率ではない（強い selection bias）。

### 5.1 最初の比較は selection bias で結論不可（撤回）

最初に強ポジ103件 vs 明確辞退12件で textual 5指標を比較したが、**両グループとも「返信した」時点で同質**で、循環論的な比較になっていた。
真に必要な比較は「**返信あり vs 完全無視（沈黙）**」。これを試行したが:

- send_data の 全文有 初回 = 238件
- YML 返信ありを member_id で join できたのは **10件のみ**
- 残り228件は「沈黙」扱いだが、古い送信は全文列が無いため除外され、新しい送信は YML 同期がまだ及んでいない

→ **textual な「効くスカウト」分析は現状のデータインフラでは成立しない**。
→ 結論「本文品質では返信率は決まらない」も「決まる」も、現データからは**言えない**。

**必要な整備**: ①過去送信の全文遡及投入 OR ②これからの送信を全文ありで蓄積し3-6ヶ月後に再分析

### 5.2 辞退26件の理由分類（テンプレ責任 vs フィルタ責任）

| 理由 | 件数 | 責任所在 |
|---|---|---|
| **車免許なし**（訪問看護） | 3 | フィルタ |
| **専門領域ミスマッチ**（精神科未経験 / 小児志向 等） | 3 | フィルタ |
| **通勤距離・エリア** | 6 | フィルタ |
| **雇用形態ミスマッチ**（副業希望に正社員 等） | 3 | フィルタ |
| **既に決まった/在職継続** | 3 | 不可避 |
| **家庭事情**（妊娠・親介護・体調） | 3 | 不可避 |
| **JM定型ボタン辞退**（lcc/ichigo に集中） | 5 | 文面未到達？ |

**約58%（15/26）がフィルタゲートで防げた辞退**。テンプレ改善より候補者属性フィルタの整備が優先度高。

### 5.3 候補者が「効いた」と明示している要素

返信に動機を書いてくれた候補者から抽出:

| 効いた要素 | 候補者の言及例 |
|---|---|
| 個人を見てる感 | 「経歴を細かく見ていただき」「個人個人をしっかり見てくださっているスカウト内容」 |
| 温度ある言い回し | 「ご丁寧で温かいご連絡」「『諦めきれず』という一文に思わずクスッと」 |
| 経歴への具体言及 | 「これまでの介護現場での経験を評価していただき、大変光栄です」 |
| 候補者不安の先回り | 「自家用車を所有していないが訪問方法を聞きたい」「PC不得意だが記録業務は」「ブランク3年でも」と質問が来る = 返信障壁が下がっている |

特に注目: chigasaki 49歳の辞退(丁寧)で **「普段は辞退ボタンですぐ返信しているが、大変ご丁寧なスカウト内容で文章送信したい気持ちになった」** という返信。  
→ **辞退時でも丁寧な反応を引き出すスカウトは将来の縁につながる**。短期返信率には現れない長期価値。

### 5.4 集計上の selection bias 問題

YML 強ポジの中に **「ジョブメドレーの求人広告を見て応募」という冒頭の応募が5件以上混入**。これはスカウト由来ではなく求人ページ経由の応募が paired として記録されたもの。
→ 集計時には role:candidate の冒頭で「求人広告を見て応募」を含む行を除外する補正が必要（バックフィル改善案件）。

### 5.5 自社送信データから導かれるアクション

#### 最優先：候補者属性フィルタの強化（テンプレより先に手を打つべき）

| ゲート | 状況 | 対象 |
|---|---|---|
| 自動車免許の保有チェック | **3件の即辞退**を生んでいる | 訪問看護 全社 |
| 居住地と訪問範囲の物理距離 | 6件以上が距離理由で辞退 | 訪問看護 全社 |
| 雇用形態希望のマッチング | 副業/短時間希望に正社員紹介で3件辞退 | 全社（既存ロジックの精度確認要） |
| 専門領域マッチング | 精神科病院に精神科未経験者を送付 | 野村病院、特化型施設 |

→ 現在の `validation` シートに上記がどこまで実装されているか、別タスクで確認・補強すべき。

#### 次優先：本文の「個人を見てる感」強化（rate-scout学びと整合）

- **rate-scout #11 の3層積み上げ構文**（○○として△△する一方で◇◇を武器に××）を訪問看護版で導入
- **温度ある言い回しの一文**（「諦めきれず」のような）をテンプレ後半に1箇所許容
- **候補者不安の先回りQ&A**を本文に組み込み（「車をお持ちでなくても訪問方法をご相談可能です」等）

#### lcc/ichigo の定型ボタン辞退対策

- lcc 5件中3件、ichigo 2件中2件が「誠に申し訳ありませんが今回は辞退」の完全コピペ。文面が候補者に届く前に判断されている可能性
- → ボタンを押すまでの **冒頭1画面** の刷新が必要。現テンプレの最初の3行を診断する価値

   - 例: 「整形外科の急性期から回復期まで、10年以上にわたり幅広い病棟」
   - 汎用「看護師として10年以上ご活躍」より深い返信を引き出す

2. **候補者の経歴 → 自社の具体機能・ポジション への橋渡し**
   - chigasaki の最強パターン: 「精神科経験 → 身体拘束最小化チーム」「回復期リハビリ → 地域包括ケア病棟」

3. **取得資格・研修の固有名詞引用**
   - lcc「がんリハビリ研修」「救急相談電話の研修」
   - chigasaki「心電図検定」
   - ichigo「バルーン挿入・ルート確保・採血・マーゲンチューブ挿入」（手技名）

4. **希望勤務地と自社所在地のマッチを明示的に橋渡し**
   - ichigo「希望勤務地（杉並区）→ 自社所在地（練馬区）→ 通勤しやすさ」
   - エリアミスマッチで離脱しがちな候補者を引き止める効果

5. **不安・障壁の先回り**
   - nomura「精神科病院は一般科と比べてブランクがあっても復職しやすい」（候補者は離職中）
   - an「パート募集だが将来正社員登用パスあり」（雇用形態柔軟性）

6. **テンプレ感のない人間味ある一文**
   - an の「諦めきれず」フック → 候補者「思わずクスッとしてしまいました」と返信
   - 不器用さ・カジュアル語（「ピッタリ」「素敵だなと思います」）が温度を出す

7. **担当者の固有名（差出人名）を本文中に明示**
   - lcc「水田」、ark「黒木」、an「梶山」、chigasaki「菅」
   - 候補者の信頼性担保に貢献

### 注意すべき副作用（同時に観察された警告サイン）

- **AI臭を見抜く候補者あり**: lcc 55歳「ジョブメドレーさんAIで書いているようなので、ご教示いただけましたら」と直接指摘
  → 具体度が高すぎても疑われる。担当者名や人間味ある表現で人格を出す
- **経歴・資格の機械的列挙だけ** はぎこちなく、リスペクト感より差し込み感が出る（daiwa #4/#7）
  → 列挙＋抽象スキルへの昇華の併用が安全

### 候補者属性が多様な会社（daiwa）の特殊事情

- 応募者が看護師でなく介護・事務・販売・営業など多岐にわたる
- 「経歴 → 抽象スキル → 当社業務」の **3段橋渡し** が必須（無いと「なぜ自分に？」で離脱）
- daiwa の textマシン化は他社より一段難しい

### 距離・物理立地の影響（nomura）

- 興味あり5件中、深い対話まで進んだ例ゼロ（全1往復）
- 距離理由の見送り・辞退が多い（北九州市から下関等）
- → 本文側で送迎・寮・社宅情報を明示する戦略を検討、または山口県内候補者へのフィルタ運用

## 6. 全体まとめ

3つの素材（rate-scout 11件 / 自社良い例1件 / YML 177件）から **言えること** と **言えないこと** を分離:

### 言えること（データ根拠あり）

1. **辞退26件のうち58%（15件）はフィルタゲートで防げた候補者ミスマッチ**。車免許なし・専門領域・通勤距離・雇用形態の4ゲートが穴。これは辞退者本人が理由を書いてくれているので解釈可能で、selection bias の影響を受けない。
2. **rate-scout 11件の他社学び**は質的に valuable（個人例として参照可能）。例: 3層積み上げ構文、業界平均比較、面談フロー開示、エピソード型。
3. **辞退時でも「丁寧」と感じさせるスカウトは長期的な縁につながる証言あり**（chigasaki 49歳の例）。短期返信率に出ない長期価値。

### 言えないこと（データ不足）

1. **「テンプレ品質を上げれば返信率が上がるか」** — 沈黙サンプルとの textual 比較ができないため不明。
2. **rate-scout の他社学びの転用効果**（数値）— 導入後の A/B 比較が必要。
3. **会社別のテンプレ品質差**が返信率に影響しているか — 同上。

### データ整備の優先課題

| 課題 | 優先度 |
|---|---|
| 過去送信の全文を遡及で send_data に投入（または別シートで蓄積） | 高 |
| YML→send_data の結合運用（member_id + sent_at で join） | 高 |
| 求人広告経由応募の除外フラグ（candidate冒頭で識別） | 中 |
| 自社スカウトの rate-scout セルフ評価運用 | 中 |

### 次のアクション候補

データインフラが整うまで textual 効果は測れないので、**フィルタゲート整備を先に進める**のが筋。具体的には:

1. validation シートで4ゲート（車免許・距離・雇用形態・専門領域）の現状を棚卸し
2. 漏れているゲートを admin/拡張側に実装
3. 整備後に「フィルタゲート整備前後で辞退率が下がったか」を月次集計タブで確認（Phase 2 で構築済みのインフラが使える）
