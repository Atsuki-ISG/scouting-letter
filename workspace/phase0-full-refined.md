# Phase 0 サンプル 全文改定版（2026-04-25）

**対応する課題**: 現状サンプルは「伝わる」が「パッと見」が重い。文章量過多・体言止め・主語不明・もりもり感・押しが弱い箇所の解消。

---

## 設計原則

### 5ブロック構成の見直し
- **opening**: 候補者の歩みを描写 → 属性別修飾句 → 「〜お迎えしたい／ご検討いただけたら」
- **bridge**: 会社側の現状・進行中の動き（具体数字はここで触れる）と、候補者の力がどう活きるか
- **facility_intro**: 制度3つに絞り文章で（箇条書きは AI臭／事務的のため採用しない）
- **job_framing**: 希望条件に書く材料ある時のみ。なければ省略
- **closing_cta**: 短く軽く

### 属性別の opening 修飾句
| タイプ | 修飾句 | 対象 |
|---|---|---|
| ベテラン × 専門資格 | 〜ご経験と〜ご視点 | S1 |
| 姿勢・価値観がPR明記 | 〜を大切にされている方 | S2, S4 |
| 若手・sparse | 構文外し「ご検討いただけたら」 | S3 |
| 多職歴・ブランク | 〜なじむ／しっくりくる系 | S4 |

### 禁止表現
- 体言止め（「〜歩み。」「〜キャリア。」「〜ご経歴。」）
- ダッシュ（——）の多用
- 連用形で終わる主語不明文（「〜こられました。」単体）
- 「思い、ご連絡」の読点（「思いご連絡」に統一）
- もりもり文（経歴動詞列挙4つ以上＋追加属性＋会社説明を1文に詰める等）
- 数字の唐突な投入（「172床増床を控える当院で」のような事務的な組織語を opening に置かない）
