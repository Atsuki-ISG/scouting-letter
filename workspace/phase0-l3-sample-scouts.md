## 全体所見

### 良い点

1. **役割分離は機能**: 4サンプル全てで opening→bridge の冒頭構造被りが発生していない
2. **捏造防止は機能**: 事例検討会・1on1・プリセプター等のNG制度名は1件も出現せず
3. **トーン切替は機能**: letter トーン（Sample 2）は明確に casual と異なる温度感
4. **寄り添い・良いイメージ**: 特に Sample 2（育児中）でプロンプト方針が鮮明に反映

### 改善候補

| # | 指摘 | 対応案 |
|---|---|---|
| A | リッチPRのカギ括弧引用が若干テンプレ臭（Sample 1） | プロンプトに「候補者PR を地の文に溶かす、直接引用は最小限」追加 |
| B | 推測踏み込みリスク（Sample 2「お子様のいるスタッフ在籍」、Sample 3「ケアマネ視点を活かすポジション」） | 「推測事実を断定的に書かない」指示を強化 |
| C | 給与情報が facility_intro に混入（Sample 3） | facility_intro の記述に「給与・勤務時間は要項に任せる」を明記（既に job_framing にはあるが、facility_intro にも適用） |
| D | sparse情報ケースは L3 が無理をしている（Sample 4） | スパース判定時は自動で L2 モードに切替する分岐を pipeline に追加 |
| E | 文字数が最大724字（sample 1-3）、要項含めると1200字超 | 運用結果で読了率を見て、各ブロックの上限を5-10%絞る余地 |

### 振り分けルールの微調整ポイント

- **rule 6 の盲点**: 「訪問看護未経験＋他科経験あり」は nursing_years == 0 になるため、現rule `nursing_years == null AND total_years >= 1` ではマッチせず default 行きに。Sample 1 の 41歳HCU13年がそうで、rule 10（40代以上）で拾えたが、30代HCUベテランは default 落ち。rule 6 を `nursing_years in (null, 0) AND total_years >= 1` 相当に広げる検討要
