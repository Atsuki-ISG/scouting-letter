# Phase 0 L3パイプライン シミュレーション v2（配慮表現の調整後）

**実施**: 2026-04-24 / プロンプト最終化後の再生成
**変更点**:
- 「汲み取りましたよ」系メタメッセージを禁止（「〜ではないでしょうか」「分断させずに」「〜にありがたい」等）
- 感情修飾語（「無理のない」「抱え込まずに」等）は1スカウト内1-2箇所まで
- **役割・活かしポイントの具体的な提示は歓迎**（「こう活きる」「こう貢献いただける」はOK）
- 配慮は「どの制度・数字を選んで書くか」に宿らせる

## v1 → v2 改善の全体像

### ✅ 改善されたこと

1. **「汲み取りましたよ」系の撤去**
   - Sample 2: 「お子様のいるスタッフも在籍」「育児で時間の使い方を覚えた方ほど」を削除
   - Sample 3: 「分断させずに」「希少な立ち位置」を削除
   - Sample 4: 「〜な意向を持つ方に」等の仮説提示を削除

2. **装飾語の整理**
   - 「無理のない」「抱え込まずに」は1箇所までに抑制（Sample 1 の facility_intro 1箇所のみ）
   - 「ご自身のペースで」「生活リズムを保ちながら」等の感情修飾語の連発を排除

3. **役割・活かしポイントの具体化は維持**
   - Sample 1「チーム全体の支えとして働いていただけます」「運営設計にも活かしていただけます」
   - Sample 2「臨床判断と支援調整の両面を担っていただけます」
   - Sample 3「ケース構成の設計や情報共有の仕組みづくりにもかかわっていただける」

4. **事実列挙が主軸に**
   - 制度・数字を **選んで並べる** ことで配慮を示す設計が実現

### ⚠️ 継続課題（v2 で解決していない）

1. **文字数バランス**: 一部ブロックで 200字超過。プロンプトの文字数厳守化が引き続き必要
2. **sparse情報時の独立ブロック破綻**: Sample 4 で job_framing が事実上融合。L2自動切替が望ましい
3. **振り分けルールの盲点**: 実データ4名で 3名がミス振り分け（rule 6/9-10の隙間・blank検出・alias不足）
