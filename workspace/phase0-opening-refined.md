# Opening 部 修正版（2026-04-25）

## 設計原則

1. **属性別の修飾句**
   - ベテラン × 専門資格 → 「〜ご経験と〜ご視点を拝見しました」
   - 姿勢・価値観がPR明記 → 「〜を大切にされている方」
   - 若手・sparse → 「お迎えしたい」構文を外し「ご検討いただけたら」
   - 多職歴・ブランク → 「なじむ／しっくりくる」系でフィット感

2. **禁止表現**
   - 体言止め（「〜歩み。」「〜キャリア。」「〜ご経歴。」）
   - ダッシュ（——）の多用
   - 連用形で終わる主語不明文（「〜こられました。」単体では誰が不明）

3. **文型**
   - 前半: 候補者の歩みを描写→「〜を拝見しました」で書き手側から締める
   - 後半: 属性別修飾句 →「〜と感じ／思いご連絡いたしました」

4. **読点処理**
   - 「思い、ご連絡」→ 「思いご連絡」

5. **要素過多を避ける**
   - 経歴の動詞列挙は2つまで
   - 追加属性（資格・姿勢）は前半の「〜を拝見しました」までに畳む

## 次のステップ

opening が固まったら、同じ原則で他ブロックを調整：
- **bridge**: 2文以内に圧縮。opening の「お迎えしたい」と重複しないよう、会社側の体制・今進めている動きを描写
- **facility_intro**: 制度列挙を3つに絞り、文章のまま（箇条書きは採用しない → AI臭・事務的印象のため）
- **job_framing**: 希望条件に書く材料ある時だけ書く。なければ省略
- **closing_cta**: 軽く短く

4サンプル全文を新トーンで書き起こし → 社内再レビュー → プロンプト / routing.py 修正 → 全テスト再pass → 実データ再生成、の順序。
