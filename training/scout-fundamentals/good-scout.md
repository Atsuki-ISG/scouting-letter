# 良いスカウトとは

オペレーター向け教育資料。私たちのスカウトが「なぜ価値があるのか」を理解し、品質を自分で判断できるようになることが目的。

---

## なぜスカウトの質が重要なのか

### 数字で見る現実

- テンプレート一斉送信の返信率: **1〜5%**
- パーソナライズされたスカウトの返信率: **10〜25%**
- スカウト経由の応募は通常応募と比べて内定率が**約1.7倍**

同じ100通でも、質の違いで返信が5件か25件かに分かれる。

### 私たちの仕事の価値

お客様（クライアント）が採用代行に求めていることは「送信数」ではない。**「自分たちではできない質のスカウトを送ってくれること」** だ。

- 候補者のプロフィールを読み込み、経験と会社の接点を見つけてくれる
- テンプレートの一斉送信ではない「あなたに送りました」感がある
- 返信率・応募率が自分たちで送るより高い

**「ここまでのことは自分たちではできない。頼んで良かった」** — これがお客様に思ってもらうべきこと。そのために、一通一通の質が問われる。

## 相手を知る: 転職希望者の心理

スカウトは「送る側」の都合で書くものではない。**受け取る人が何を考え、何に悩み、何を求めているか**を知ることが、良いスカウトの出発点。

### 転職を考えている人の頭の中

転職希望者が抱える不安のトップ3:

| 不安 | 割合 |
|------|------|
| 自分に合う職場に転職できるか | **58.5%** |
| 希望に合う求人が見つかるか | 47.7% |
| 自分の経歴で転職できるか | 34.2% |

つまり求職者の最大の関心は**「自分に合うか」**。会社がどんなに素晴らしくても、「自分に合う」と感じなければ動かない。だからパーソナライズが効く。「あなたの経験がここで活きる」と伝えることで、この不安に直接応えられる。

### 介護・看護職が転職する本当の理由

**看護師の退職理由:**

| 順位 | 理由 | 割合 |
|-----|------|------|
| 1位 | 結婚・出産・育児 | 24.8% |
| 2位 | 残業が多い・休暇が取れない | 18.1% |
| 3位 | 人間関係 | 12.4% |
| 4位 | 他の施設に興味 | 8.5% |
| 5位 | 健康問題 | 8.5% |

**介護職の退職理由:**

| 順位 | 理由 | 割合 |
|-----|------|------|
| 1位 | 職場の人間関係 | **34.3%** |
| 2位 | 施設の理念・運営への不満 | 26.3% |
| 3位 | 他に良い仕事があった | 19.9% |
| 4位 | 収入が少ない | 16.6% |

**ここから読み取れること:**
- 介護職は**人間関係が圧倒的な退職理由**。「チームの雰囲気」「上司の姿勢」を伝えることが刺さる
- 看護師は**ライフステージの変化**が多い。「日勤のみ」「時短」「子育て支援」が響く
- どちらも**「今の職場が嫌」で動いている**ことが多い。新しい職場の良さを具体的に伝える必要がある

### 介護職特有の「辞めたいけど辞められない」心理

- 利用者への情、後任への引継ぎの負い目、人手不足の職場への罪悪感
- でも67%が次の転職先も介護施設を選ぶ — **業界への愛着はあるが、環境を変えたい**

→ スカウトでは「介護の仕事を否定せず、もっと良い環境がある」というトーンが効く

### 求職者がスカウトで見ているポイント

2,000名調査で判明した、返信するかどうかの判断基準:

| 判断要素 | 割合 |
|---------|------|
| 年収・待遇 | **35.9%** |
| 自分のスキルと合うか | **30.4%** |
| 勤務条件 | 25.5% |

**→ 「待遇」と「自分に合うか」で66%**。この2つに応えるスカウトが返信される。

### 求職者が「良い」と感じるスカウトの条件

| 要素 | 好反応の割合 |
|------|------------|
| プロフィールをしっかり読んでいると感じる | **60.5%** |
| 特別感がある | 51.1% |
| なぜスカウトしたか理由が具体的 | 46.9% |

反対に、**71.3%の求職者が「短くて簡潔な文章」を好む**。長く丁寧に書くより、短く的確に書く方が読まれる。

### 求職者が嫌がるスカウト

- 希望に合わない求人（最多の不満）
- 大量に届くテンプレートメール
- 同じ案件が何度も届く
- 会社の情報ばかりで自分への言及がない

→ **「数撃てば当たる」の真逆がプロの仕事。**

## 冒頭の勝負: 最初の数十文字で決まる

### ジョブメドレーの一覧画面で何が見えるか

求職者がスカウトを受け取ると、一覧画面に**事業所名 + 本文の冒頭部分**がプレビュー表示される。ここで興味を引けなければ開かれない。

**モバイルの表示制約:**
- プレビューで見えるのは**冒頭30〜50文字程度**
- 全メールの55%以上がスマホで開封される
- スクロールせずに見える範囲で勝負が決まる

### 冒頭でやるべきこと

**最初の1文で「なぜあなたに送ったか」を伝える。** これがテンプレートとの決定的な差。

| NG（テンプレ感） | OK（パーソナライズ） |
|----------------|------------------|
| 「突然のご連絡失礼いたします。○○の採用担当です。」 | 「○○さんの訪問看護でのリハビリ連携のご経験を拝見し、ご連絡しました。」 |
| 「当ステーションの求人にご興味はございませんか。」 | 「認知症ケア5年のご経験は、当ステーションで大きな力になると考えております。」 |
| 「プロフィールを拝見し、ご連絡いたしました。」 | 「急性期でのご経験は、医療依存度の高い在宅ケアでこそ活きると考えています。」 |

上の段は誰にでも送れる。下の段は「この人のプロフィールを読んだ」と伝わる。一覧画面でこの違いが見えたとき、どちらを開くかは明らか。

### 冒頭で避けるべきこと

- **挨拶から始めない**: 「はじめまして」「突然のご連絡〜」は文字数の無駄。プレビューの貴重な30文字を挨拶で消費しない
- **会社紹介から始めない**: 「当ステーションは〜」で始まるスカウトは一覧で会社情報の羅列に見える
- **全員に当てはまることを書かない**: 「看護師の資格をお持ちの方へ」は全員に送れる

## 読んだ人を動かすテクニック

### 構成の基本: 特徴ではなく「あなたにとっての利益」で語る

会社の特徴をそのまま書いても人は動かない。**「それが相手にとって何を意味するか」**まで踏み込む。

| レベル | 書き方 | 例 |
|--------|--------|-----|
| 特徴（Feature） | 会社の事実をそのまま書く | 「電子カルテ導入済みです」 |
| 利点（Advantage） | その特徴の一般的なメリット | 「記録業務が効率化されています」 |
| **利益（Benefit）** | **相手にとっての具体的な価値** | **「残業が月平均5時間以内で、プライベートとの両立が可能です」** |

スカウト文では常に**Benefit（相手にとっての利益）**で語る。「うちはこうです」ではなく「だからあなたにとってこういう良いことがあります」。

### 心理テクニック: なぜ人はスカウトに返信するのか

#### ① 返報性 — 先に価値を与える

人は何かをしてもらうと「お返しをしなければ」と感じる。

**スカウトでの使い方:** プロフィールを丁寧に読み、具体的に評価する。先に相手を認める行為が「読んでくれたから返信しよう」という気持ちを生む。

#### ② 希少性 — 「あなただから」の特別感

手に入りにくいものに高い価値を感じる心理。

**スカウトでの使い方:**
- 「○○の経験をお持ちの方にのみお送りしています」
- 「今回の募集は1名限定です」

事実ベースで。嘘はNG。

#### ③ 社会的証明 — 他の人も選んでいる安心感

人は他の人がやっていることを正しいと判断する傾向がある。

**スカウトでの使い方:**
- 「直近1年で看護師○名が入職しています」
- 「前職で訪問看護に携わっていた方が多数活躍中です」

具体的な数字が信頼性を高める。

#### ④ 損失回避 — 今のままでいいのか、と気づかせる

人は利益を得ることより、損失を避けることに約2倍強く反応する。

**スカウトでの使い方:**
- 「○○のスキルを、もっと活かせる環境があります」

煽りすぎは逆効果。さりげなく「今のままではもったいない」と気づかせる程度に。

### CTA（行動を促す締め）: ハードルを下げる

スカウトの目的は「返信してもらうこと」。最後の一文でハードルを下げる。

| ハードル | 例 | 効果 |
|---------|-----|------|
| **最低** | 「ご興味があればお気軽にご返信ください」 | 返信の負担がほぼゼロ |
| **低** | 「まずは見学だけでも歓迎です」 | コミットメントが小さい |
| **中** | 「オンラインで15分ほどお話しできればと思います」 | 時間が具体的で想像しやすい |
| **高（避ける）** | 「ぜひご応募ください」 | 重い。面接の準備が必要と感じる |

**ポイント:**
- CTAは**1つだけ**。複数並べると迷って行動しない
- 「今すぐ転職でなくても」「情報収集だけでも」は転職意欲が低い層にも効く
- 半端な数字が効く: 「15分」より「14分」の方がリアルに感じる

## 良いスカウトの3条件

### 1. 「あなたを見ました」が伝わる

求職者は何十通ものスカウトを受け取っている。その中で「自分のプロフィールをちゃんと読んでくれた」と思えるスカウトだけが目に留まる。

**ダメな例（誰にでも送れる）:**
> 当ステーションは札幌市西区にあり、認知症の方や神経難病の方など幅広い利用者様がいらっしゃいます。クリニック併設で研修制度も充実しています。

→ 会社の情報を並べただけ。候補者の「何」を見たのかが伝わらない。

**良い例（あなただから送った）:**
> 認知症ケアのご経験は、認知症の利用者様が多い当ステーションにおいて、大きな力になると考えております。

→ 「認知症ケアの経験」という候補者固有の情報に触れ、それが会社でどう活きるかを伝えている。

### 2. 接点が明確

パーソナライズの核心は**「候補者の強み × 会社の特色 = 接点」**。この掛け算が1つ、明確に伝わること。

```
候補者の強み  ×  会社の特色  ＝  接点
```

| 候補者の情報 | 会社の特色 | 接点の例 |
|------------|----------|---------|
| 認知症ケア5年 | 認知症の利用者が多い | 「認知症ケア5年のご経験は、当ステーションで大きな力に」 |
| 急性期病棟の経験 | 医療依存度の高い利用者が多い | 「急性期でのご経験は、医療依存度の高い在宅ケアで活かしていただける」 |
| 看護師+ケアマネ | チーム連携を重視 | 「看護とケアマネの両方の視点をお持ちのことで、多職種連携の場で力を発揮いただける」 |

**接点がないパターン（NG）:**
- 会社情報の羅列（「当社は〇〇で、〇〇もあり、〇〇です」）
- 地理情報のみ（「お近くにお住まいとのこと」だけ）
- 弱い接点（運転免許のみなど）

### 3. 自然で対等

求職者は「選んでもらう側」ではなく「選ぶ側」でもある。上から目線でも、へりくだりすぎでもない、対等な温度感が大事。

| NG（上から目線） | OK（対等） |
|----------------|----------|
| 「フォローしますので安心してください」 | 「チーム全体でサポートし合える環境です」 |
| 「基礎を固めてこられた」 | 「培われた経験を活かしていただける」 |
| 「初めての方でもサポート」 | 「現場で培われたスキルが当ステーションで力になる」 |

| NG（過剰敬語） | OK（自然な丁寧さ） |
|---------------|-----------------|
| 「拝察いたします」 | 「心強く拝見しました」 |
| 「敬意を表します」 | 「注目しました」 |
| 「感銘を受けました」（経歴に対して） | 「心強く拝見しました」 |

## パーソナライズの考え方

### プロフィールのどこを見るか（優先順位）

1. **経験・スキル（最優先）**: 臨床経験、施設形態、得意分野
2. **保有資格（複数ある場合）**: 看護師+保健師 など
3. **自己PR**: 本人が書いた想いや強み
4. **こだわり条件（参考程度）**: 経験で接点が作れるなら不要
5. **地理情報（最も弱い）**: これだけでは接点として不十分

### パーソナライズの深さ

| レベル | やり方 | 求職者の受け取り方 |
|--------|--------|-----------------|
| Lv.0 | 「プロフィールを拝見し」のみ | テンプレートだな |
| Lv.1 | 資格名を書くだけ | 一応見たのか |
| **Lv.2** | **経験 × 会社特色 → 適性を論理展開** | **自分のために書いてくれた** |
| **Lv.3** | **過去の経験 → 将来像まで接続** | **キャリアを考えてくれている** |

私たちが目指すのは**Lv.2〜3**。これがテンプレート一斉送信との決定的な差であり、お客様が自分たちではできないと感じるポイント。

### 転職理由から逆算してパーソナライズする

求職者の退職理由がわからなくても、統計から推測できる。

| よくある退職理由 | スカウトで響く訴求 |
|---------------|-----------------|
| 人間関係（介護34%） | 「チームで声を掛け合いながら〜」「少人数でアットホーム」ではなく**具体的なエピソード** |
| ライフステージ変化（看護25%） | 「日勤のみ」「時短勤務可」「子育て中のスタッフも在籍」 |
| 残業・休暇（看護18%） | 「残業月平均○時間」「有給消化率○%」— **数字で語る** |
| 収入（介護17%） | 「月給○万円〜」「賞与○ヶ月」— **曖昧にしない** |

### 情報が少ない候補者への対応

プロフィール情報が少ない = 経験が浅い、ではない。

- **就業中なら経験がある前提**で書く
- **40代以上 × 情報少ない → 教育体制を前面に出さない**。年齢から相応のキャリアが推測されるため「初めての方でもサポート」は上から目線になる
- 代わりに「これまでの経験やスキルを活かしていただける」という評価ベースで書く

## 絶対に守るルール

### やったら一発NG

| ルール | 理由 |
|--------|------|
| **年齢・世代への言及禁止** | 「〇代」「若手」「ベテラン」は不快感を与える |
| **他社・他職種の教育制度を混ぜない** | LCC看護師に「eラーニング」（医療事務の制度）など。信頼が崩壊する |
| **会社名を間違えない** | 他社の名前が混入したら致命的 |
| **居住地を詳細に書かない** | 「〇〇区」→「都内」「札幌市内」等の広域表現に |
| **憶測を書かない** | 「〜されたいのですね」「長く続けているはず」は事実だけ書く |

### トーンの基本

| やること | やらないこと |
|---------|------------|
| ですます調で統一 | 「拝察いたします」のような過剰敬語 |
| 同僚に話すような自然な丁寧さ | 「ご一緒したい」など送り手の感情が主語 |
| 「〜とのことで、」と自然につなぐ | 「〜とのこと、」で文を切る（体言止め） |
| 接点から直接書き始める | 「現在離職中とのこと」のような不要な前置き |

## 初回と再送の違い

| | 初回 | 再送 |
|---|------|------|
| 文字数 | 約100文字 | 120〜150文字 |
| 角度 | 経験・スキルとの接点 | **初回と異なる角度**（姿勢、別の経験、自己PR等） |
| 熱量 | 標準 | 「諦めきれず」の冒頭に合った熱量（ただし押し付けはNG） |

再送で初回と同じ接点・同じ表現を使い回すのはNG。

| 初回で使った接点 | 再送で使う別角度 |
|--------------|----------------|
| 専門的な経験・資格 | 姿勢・自己PRの想い |
| 訪問看護の実務経験 | チームワーク面 |
| 主な診療科の経験 | 経験の幅広さ |
| 経験年数の豊富さ | 在宅・地域への可能性 |

## よくある修正パターン

実際の運用で繰り返し見つかった問題。同じミスを防ぐために目を通してください。

### ① 情報が少ない = 経験が浅い、と決めつける

**修正前:** 「初めての方でもサポート体制が充実しています」（40代・就業中・情報少）

**問題:** 就業中の40代は相応の経験がある可能性が高い。教育体制訴求は上から目線。

**修正後:** 「現在も臨床の現場で培われている経験やスキルは、当ステーションで活かしていただけると考えております。」

### ② 対象職種の資格にわざわざ言及する

**修正前:** 「看護師の資格をお持ちとのこと、当ステーションでは〜」

**問題:** 看護師求人に送っている以上、看護師資格は持っている前提。冗長。

**修正後:** 資格への言及を削除し、経験や教育体制から直接書き始める。

### ③ 他社・他職種の教育制度を混ぜる

**修正前:** LCC看護師向けに「eラーニングで学べる環境が〜」

**問題:** eラーニングはLCC医療事務の制度。看護師は同行訪問が正しい。

**修正後:** 「東京都訪問看護教育ステーション事業を受託しており、入社後約1ヶ月の同行訪問など学びの環境が整っています。」

### ④ 会社情報の羅列だけで接点がない

**修正前:** 「当ステーションは〇〇に拠点を構えており、訪問範囲には〇〇も含まれております。」

**問題:** 会社情報はテンプレートに書いてある。パーソナライズ文は候補者との接点に集中すべき。

### ⑤ 送り手の感情が前に出すぎる

**修正前:** 「ぜひご一緒したいという想いを強く感じ、ご連絡いたしました。」

**問題:** 相手にとってのメリットが伝わらない。

**修正後:** 「〇〇のご経験は、当ステーションで大きな力になると考えております。」

## セルフチェックリスト

送信前に確認してください。

### 必須（1つでもNGなら送らない）

- [ ] パーソナライズ文が入っている（テンプレートだけになっていない）
- [ ] 会社名が正しい
- [ ] 教育制度が正しい会社・職種のものになっている
- [ ] 年齢・世代への言及がない
- [ ] 居住地が詳細すぎない

### 品質（良いスカウトにするために）

- [ ] 冒頭30文字以内に「なぜあなたに送ったか」が入っている
- [ ] 候補者の経験と会社の特色の「接点」が明確に伝わる
- [ ] 会社の特徴ではなく、相手にとっての利益（Benefit）で語っている
- [ ] 会社情報の羅列だけになっていない
- [ ] 上から目線・過剰敬語がない
- [ ] テンプレート前後の文と表現が重複していない
- [ ] CTAが具体的で、ハードルが低い
- [ ] 再送の場合、初回と異なる角度で書かれている

## まとめ: 私たちのスカウトが選ばれる理由

1. **相手の心理を理解している** — 転職者の不安や動機を知り、それに応えるスカウトを書ける
2. **冒頭で惹きつけている** — 一覧画面の30文字で「あなたに送った」と伝わる
3. **経験と会社の接点を見つけている** — 候補者が「自分のために書かれた」と感じる
4. **特徴ではなく利益で語っている** — 「うちはこうです」ではなく「だからあなたにこういう良いことがある」
5. **行動を促している** — 低いハードルのCTAで返信しやすくしている
6. **会社ごとの正確な情報** — 教育制度や職場の特徴を間違えない

この6つを毎通維持することが、お客様に「頼んで良かった」と思ってもらえる最大の武器になる。
