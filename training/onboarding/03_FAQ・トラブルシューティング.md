# FAQ・トラブルシューティング

スカウト送信業務でよくある質問と、問題が起きた場合の対処法をまとめています。

---

## よくある質問

### 基本的な操作について

**Q: 1回の作業で何件くらい送ればいいですか？**
A: 1バッチ10〜20件が目安です。慣れてきたら30件程度まで増やせますが、50件を超える大量抽出はエラーリスクが上がるため推奨しません。

**Q: 抽出中に他の作業をしてもいいですか？**
A: ジョブメドレーのタブは操作しないでください。別のブラウザウィンドウやアプリでの作業はOKです。

**Q: 途中で中断して、後から再開できますか？**
A: はい。抽出・生成済みのデータはブラウザに保存されます。サイドパネルを閉じても、再度開けばデータは残っています。ただしChromeを完全に閉じるとデータが消える場合があります。作業途中でChromeを閉じる場合は、できればその前に送信まで完了させてください。

**Q: 同じ候補者に間違って2回スカウトを送ってしまう心配はありますか？**
A: ジョブメドレー側で二重送信はブロックされるので、同一人物に2回送られることはありません。以前送った候補者に再度送りたい場合は、生成時の送信種別で「再送」を選択してください。

**Q: 「連続送信アシスト」を使わず、1件ずつ手動で送ることはできますか？**
A: はい。送信タブの候補者リストから個別に選んで送信することもできます。ただし連続送信のほうが効率的です。

**Q: スカウト文を送信前に修正したいのですが**
A: 連続送信を一度「停止」してから、ジョブメドレーのテキストエリアを直接編集して手動で送信してください。もしくは「NG - スキップ」で飛ばし、後で個別に対応してください。

### 生成について

**Q: 「型はめ」と「AI生成」の違いは何ですか？**
A: 職務経歴が記載されている候補者にはAIが個別の文章を生成します（AI生成）。経歴の記載がない候補者には、経験年数と年齢に応じた定型パターンの文章が使われます（型はめ）。最終的にどちらもテンプレートと組み合わせて完成形になります。

**Q: 雇用形態の「自動判定」は何を見ていますか？**
A: 候補者がジョブメドレーで設定している「希望雇用形態」を見て、パートか正社員かを自動判定します。通常は「自動判定」のまま問題ありません。

**Q: 全員が「除外」になってしまいました**
A: 以下を確認してください：
1. 会社選択が正しいか
2. 職種が候補者の資格と合っているか（例: 看護師資格の人に「理学療法士」を選んでいないか）
3. 送信種別（初回/再送）が正しいか
4. ジョブメドレーで適切にフィルタリングして候補者を絞り込んでいるか

**Q: AI生成の文章が変なことがあります**
A: AIは完璧ではないため、稀に不自然な文章を生成することがあります。確認ポップアップで内容をチェックし、問題がある場合は「NG - スキップ」してください。頻繁に発生する場合は管理者に報告してください。改善に役立てます。

### 送信について

**Q: 「OK - 送信する」ボタンが灰色で押せません**
A: 3つのチェックボックス（会員番号・対象求人・パーソナライズ文）のすべてにチェックを入れる必要があります。

**Q: 赤い警告で「スカウト対象外の候補者です」と出ました**
A: 年齢や資格が求人の条件に合わない候補者です。必ず「NG - スキップ」を押してください。この候補者にスカウトを送らないでください。

**Q: 求人が自動で選ばれない場合は？**
A: 「求人選択に失敗しました」と表示されたら、ジョブメドレーのドロップダウンから手動で正しい求人を選んで「再開」ボタンを押してください。頻繁に発生する場合は管理者に連絡。

**Q: 送信した後にスカウト文の間違いに気付きました**
A: 送信後の取り消しはできません。重大な誤りがある場合は管理者に報告してください。今後の対策として、確認時のチェックをより丁寧に行うようにしましょう。

**Q: スキップした候補者に後から送りたい**
A: 送信タブの候補者リストでその候補者を見つけて、個別に送信操作ができます。

---

### 設定・アカウントについて

**Q: ジョブメドレーのログインが切れた場合は？**
A: ジョブメドレーに再ログインして、ページをリロードしてください。サイドパネルのデータは保持されます。

**Q: 拡張機能のアップデート方法は？**
A: 管理者から新しいZIPが来たら、`chrome://extensions` で既存を削除 → 新しいdistフォルダを読み込み → API設定を再入力してください。

**Q: API設定（URLやパスワード）を忘れてしまいました**
A: 管理者に連絡して再共有してもらってください。セキュリティ上、メモ帳等への保存は推奨しません。

## トラブルシューティング

### 拡張機能が動かない

| 症状 | 対処法 |
|------|--------|
| サイドパネルが開かない | ① ジョブメドレーにログインしているか確認 ② ページをリロード（F5 or Cmd+R） ③ `chrome://extensions` で拡張が有効（ON）か確認 |
| 拡張機能が「エラー」表示 | `chrome://extensions` で一度削除し、再度 dist フォルダを読み込む |
| ポップアップが表示されない | パズルマーク🧩から Scout Assistant をクリック。ピン留めしていると直接アイコンクリック |
| Chrome起動時に「デベロッパーモードの拡張機能を無効にする」警告 | 「×」で閉じる（「無効にする」を押さないこと） |

### 抽出（📋）のトラブル

| 症状 | 原因 | 対処法 |
|------|------|--------|
| 抽出が途中で止まる | プロフィール画面の読み込み遅延 | 少し待つ。回復しなければページリロードして再抽出 |
| 「会員番号がリストに見つかりません」 | 指定した番号がページ上にない | 会員番号を確認するか空欄にして先頭から |
| 抽出件数が指定より少ない | ページの候補者数が指定数未満 | 正常。次ページに進んで追加抽出 |
| 抽出データが全部「不明」になる | ジョブメドレーのHTML構造変更 | 管理者に連絡（拡張の更新が必要な可能性） |

### 生成（🤖）のトラブル

| 症状 | 原因 | 対処法 |
|------|------|--------|
| 「タイムアウト」エラー | サーバーの応答が遅い | 数分待ってから再試行。件数を5-10件に減らす |
| 「APIキーが無効です」 | パスワードが間違っている | ⚙設定タブまたはポップアップでパスワードを再確認・再入力 |
| 「API枠超過（レート制限）」 | 短時間にリクエストが多すぎ | 2-3分待ってから再試行 |
| 「サーバーが一時的に利用できません」 | サーバーがダウンしている | 管理者に連絡 |
| 生成ボタンが押せない（灰色） | 抽出データが0件 | 先に📋抽出タブで候補者を抽出 |
| 全員「除外」になる | 会社・職種の選択ミス or フィルタリング不足 | 設定を確認。ジョブメドレーで適切にフィルタ |

### 送信（📤）のトラブル

| 症状 | 原因 | 対処法 |
|------|------|--------|
| 「求人選択に失敗しました」 | 求人キーワードが一致しない | 手動でドロップダウンから選択 →「再開」 |
| スカウト文がテキスト欄に入力されない | ジョブメドレーのUI変更 | 管理者に連絡 |
| 送信完了の確認画面が消えない | ジョブメドレーの読み込みが遅い | 最大5分間自動で待機する。それでも消えなければ手動で閉じる |
| 連続送信が途中で止まる | ネットワーク断やページエラー | ページリロード後、送信タブで未送信分を再開 |
| 候補者全員に「対象外」警告 | パーソナライズ文が未生成 | 🤖API生成タブに戻って再生成 |

### 接続のトラブル

| 症状 | 対処法 |
|------|--------|
| 接続テスト失敗 | ① URLをコピペで再入力（手入力ミス排除） ② URL末尾の余分な/を削除 ③ パスワードをコピペで再入力 ④ WiFi接続を確認 ⑤ 管理者に連絡 |
| 接続は成功するが生成でエラー | パスワードは合っているが会社設定に問題がある可能性。管理者に連絡 |
| 「ネットワークエラー」 | インターネット接続を確認。WiFiを再接続して再試行 |

## リカバリー手順（よくあるシナリオ）

### シナリオ1: 抽出の途中でChromeがフリーズした

1. Chromeを強制終了（Mac: Cmd+Q、Windows: Alt+F4）
2. Chromeを再起動
3. ジョブメドレーにログイン
4. サイドパネルを開く
5. 📋 抽出タブで最初からやり直し（途中からの再開はできない）

### シナリオ2: 生成中にエラーが出て中断した

1. 🤖 API生成タブを確認
2. 一部は生成成功していれば、そのまま📤送信タブで送信可能
3. 全件エラーの場合は、数分待ってから「一括生成」を再実行

### シナリオ3: 送信中にジョブメドレーのセッションが切れた

1. 連続送信を「停止」
2. ジョブメドレーに再ログイン
3. ページをリロード
4. サイドパネルを開き直す
5. 📤 送信タブで、未送信の候補者から送信を再開

### シナリオ4: 間違った会社でスカウト文を生成してしまった

1. まだ送信していなければ問題なし
2. 🤖 API生成タブに戻る
3. 正しい会社を選択して再生成すれば上書きされる

## 管理者への連絡が必要なケース

以下の場合は自力での解決が難しいため、管理者に連絡してください：

- 接続テストが何度やっても失敗する
- 「サーバーが一時的に利用できません」が長時間続く
- 抽出データが全部「不明」になる（ジョブメドレーの仕様変更の可能性）
- スカウト文の入力が動作しない（ジョブメドレーの画面変更の可能性）
- 特定の会社でだけエラーが出る
- AI生成の文章の品質が悪い（同じような問題が繰り返し起きる場合）

**連絡時に伝えること：**
1. 何をしようとしていたか（どのタブで何のボタンを押したか）
2. 表示されたエラーメッセージ（スクリーンショット推奨）
3. 何件中何件目で問題が起きたか
4. 再試行したか、その結果はどうだったか

## 用語集

| 用語 | 説明 |
|------|------|
| スカウト | 求職者に送る採用のお誘いメッセージ |
| パーソナライズ文 | 候補者一人ひとりに合わせてAIが生成する文章の部分 |
| テンプレート | 挨拶・会社紹介など定型の文章部分 |
| 型はめ | 経歴情報がない候補者に対して、定型パターンから文章を選ぶ方式 |
| バリデーション | 候補者が求人の条件（年齢・資格等）に合うかのチェック |
| 除外 | バリデーションの結果、スカウト対象外と判定された候補者 |
| ドライラン | 実際には送信せず、動作だけテストするモード |
| 求人自動選択 | 候補者の職種に合わせて自動で求人を選ぶ機能 |
| サイドパネル | Chrome画面の右側に表示される拡張機能の操作画面 |
| 会員番号 | ジョブメドレーでの候補者のID番号 |
| 再送 | 以前スカウトを送った候補者に再度送ること |
| 初回 | その候補者に初めてスカウトを送ること |
