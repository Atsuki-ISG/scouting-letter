# スカウト送信の業務フロー操作ガイド

この資料では、Chrome拡張機能を使ったスカウト送信の実際の操作手順を、画面の流れに沿って説明します。

---

## 全体の流れ

スカウト送信は以下の3ステップで行います。毎回この順番で進めます。

```
ステップ1: 候補者プロフィールの抽出（📋 抽出タブ）
    ↓
ステップ2: スカウト文のAI一括生成（🤖 API生成タブ）
    ↓
ステップ3: 1件ずつ確認して送信（📤 送信タブ）
```

所要時間の目安：10件あたり抽出5分 + 生成2分 + 送信10分 ＝ 計15-20分程度

---

## 事前準備（毎回の作業開始時）

1. Chromeでジョブメドレー（https://job-medley.com）にログイン
2. スカウトを送りたい候補者の一覧ページに移動
3. 拡張機能アイコンをクリック → ポップアップで**会社**が正しいことを確認
4. **「サイドパネルを開く」** をクリック
5. サイドパネルが画面右側に表示される

### ジョブメドレーでの候補者絞り込み（推奨）

抽出前に、ジョブメドレーの検索フィルターで候補者を絞り込んでおくと効率的です。

- 対象エリア
- 保有資格
- 希望雇用形態
- 最終ログイン日（直近のほうが返信率が高い傾向）

絞り込みをしないと、対象外の候補者も抽出してしまい、生成時に「除外」が多くなります。

---

## ステップ1: 候補者プロフィールの抽出（📋 抽出タブ）

### 操作手順

1. サイドパネルで **「📋 抽出」** タブを選択
2. **抽出件数** を入力（推奨: 10〜20件）
   - 最大50件まで指定可能
   - 初めてのうちは5件程度で試すのがおすすめ
3. **開始会員番号**（任意）
   - 通常は空欄でOK（ページの先頭から抽出）
   - 特定の候補者から始めたい場合のみ入力
4. **「抽出開始」** ボタンをクリック

### 抽出中の画面

- 進捗バーが表示される（「3/10 抽出中...」のように進行状況が見える）
- ジョブメドレーの画面上で、候補者のプロフィール画面が自動で開閉される
- 1件あたり数秒〜十数秒かかる

### 重要な注意点

- **抽出中はジョブメドレーのタブを操作しないでください**
  - マウスクリック、スクロール、タブ切り替えをしないこと
  - 自動で画面を操作しているため、別の操作が入ると失敗します
- **他のChromeウィンドウやアプリでの作業はOK**
  - ジョブメドレーのタブさえ触らなければ、他のことをしていて大丈夫です

### 抽出完了後

- 抽出されたプロフィール一覧が表示される
- 各候補者の **会員番号、性別、年齢、エリア** が確認できる
- 必要に応じて **「CSVダウンロード」** でデータを保存できる（通常は不要）

### 抽出される情報の内容

拡張機能はジョブメドレーのプロフィール画面から以下の情報を自動で読み取ります：

| 情報 | 説明 | 使われ方 |
|------|------|---------|
| 会員番号 | 候補者の一意のID | 送信先の特定 |
| 性別・年齢 | 基本属性 | バリデーション（年齢制限チェック） |
| エリア | 居住地域 | バリデーション |
| 保有資格 | 看護師、理学療法士等 | テンプレート選択・バリデーション |
| 経験 | 業種・年数 | パーソナライズ文の生成 |
| 希望雇用形態 | パート or 正社員 | テンプレート選択 |
| 希望勤務エリア | 通勤可能エリア | パーソナライズ文の生成 |
| 自己PR | 候補者が書いた自由記述 | パーソナライズ文の生成 |
| 職務経歴 | 過去の勤務先・業務内容 | AI生成の重要な材料 |
| スカウト送信日 | 過去にスカウトを送った日 | 再送判定 |

## ステップ2: スカウト文のAI一括生成（🤖 API生成タブ）

### 操作手順

1. **「🤖 API生成」** タブを選択
2. 画面上部に **「抽出済み: X件」** と表示されていることを確認
3. **「一括生成」** ボタンをクリック
4. 生成設定のモーダル（小窓）が表示される

### 生成設定の入力

モーダルに表示される各項目を設定します：

| 項目 | 説明 | 通常の選択 |
|------|------|-----------|
| **会社** | スカウトを送る会社名 | 業務指示で指定された会社（ヘッダーと連動） |
| **雇用形態** | パート or 正社員テンプレートの選択 | **「自動判定」**（候補者の希望に合わせて自動で判断） |
| **職種** | 対象職種 | 業務指示で指定された職種（例: 看護師） |
| **送信種別** | 初めて送るか、以前送った人への再送か | **「初回」**（再送の場合のみ「再送」を選択） |

5. 設定を確認して **「生成開始」** をクリック

### 生成中の画面

- 「生成中... 3/10」のように進捗が表示される
- 10件で約1〜2分程度
- この間は待つだけでOK（他の作業をしていても大丈夫）

### 生成結果の読み方

生成が完了すると、以下のようなサマリーが表示されます：

```
合計: 10 / AI生成: 6 / 型はめ: 2 / 除外: 2
```

| 表示 | 意味 | 補足 |
|------|------|------|
| **AI生成** | AIが個別に文章を作った件数 | 職務経歴がある候補者が対象 |
| **型はめ** | 定型パターンで文章を作った件数 | 職務経歴がない候補者が対象 |
| **除外** | スカウト対象外として除外された件数 | 年齢や資格が条件に合わない |

**「除外」が多い場合**は、以下を確認してください：
- 会社の選択が間違っていないか
- 職種の選択が候補者の資格と合っているか
- ジョブメドレーの検索フィルターで適切に絞り込んでいるか

### AI生成と型はめの違い

**AI生成（経歴あり）:**
- 候補者の職務経歴・自己PRを読んで、一人ひとり異なるパーソナライズ文を作る
- 例: 「訪問看護の5年のご経験を拝見しました。特に在宅ターミナルケアの経験は当施設でも活かしていただけます。」

**型はめ（経歴なし）:**
- 経験年数と年齢に応じて、あらかじめ用意されたパターンの中から最適なものを選ぶ
- 例: 「看護師としてのご経験を活かし、訪問看護の分野でキャリアを広げませんか。」

どちらも最終的にはテンプレート（挨拶文や会社紹介の定型部分）と組み合わせて、完成したスカウト文になります。

## ステップ3: スカウト文の確認と送信（📤 送信タブ）

### 操作手順

1. **「📤 送信」** タブを選択
2. 生成された候補者リストが表示される
3. **求人**のドロップダウンが正しいことを確認
4. **「連続送信アシスト」** にチェックを入れる
5. **「開始」** ボタンをクリック
6. 1件ずつ確認ポップアップが表示されるので、確認して送信

### 連続送信の流れ

「開始」を押すと、以下のサイクルが候補者の数だけ繰り返されます：

```
確認ポップアップ表示
  ↓
内容を確認
  ↓
「OK - 送信する」または「NG - スキップ」
  ↓
OKの場合 → 自動で送信される
  ↓
次の候補者のポップアップが表示される
```

### 確認ポップアップの詳細な見方

確認ポップアップでは以下の項目を上から順に確認してください。

#### 1. 警告エリア（最上部）

- **赤色の警告「スカウト対象外の候補者です」**: この候補者は対象外です。必ず **「NG - スキップ」** を押してください
- **黄色の警告（バリデーション警告）**: 注意が必要ですが送信は可能です。内容を確認して判断してください
  - 例: 「年齢が上限に近い」「希望エリアが遠い」等

#### 2. プロフィール概要テーブル

候補者の基本情報が表形式で表示されます：

| 項目 | 確認ポイント |
|------|-------------|
| 資格 | この資格に合った求人が選ばれているか |
| 経験 | パーソナライズ文と経験内容が合っているか |
| 希望雇用 | テンプレート（パート/正社員）が合っているか |
| エリア | 通勤可能な範囲か |
| 自己PR | パーソナライズ文に反映されているか |
| 職歴 | パーソナライズ文と矛盾がないか |

#### 3. チェックボックス（3つすべてにチェックが必要）

内容を確認したら、以下の3つにチェックを入れます：

- ☐ **会員番号**: 正しい候補者のプロフィールが表示されているか
- ☐ **対象求人**: 正しい求人が選択されているか
- ☐ **パーソナライズ文**: 文章が適切か（後述の確認ポイント参照）

**3つすべてにチェックを入れると、「OK - 送信する」ボタンが有効になります。**

#### 4. スカウト文全文プレビュー

「全文を表示」をクリックすると、実際に候補者に届くスカウト文の全文が見られます。
- テンプレート部分（定型の挨拶・会社紹介）＋ パーソナライズ文（個別生成部分）が結合された完成形
- 送信前に一度は全文を確認することを推奨

#### 5. 操作ボタン

| ボタン | 色 | 動作 |
|--------|-----|------|
| **OK - 送信する** | 青色 | このスカウト文を送信し、次の候補者へ進む |
| **NG - スキップ** | 灰色 | この候補者をスキップし、次の候補者へ進む |
| **連続送信を停止** | 赤色 | 連続送信自体を中止する |

### パーソナライズ文の確認ポイント

以下の点をチェックしてください：

| チェック項目 | NGの例 |
|-------------|--------|
| 候補者の経歴に合っているか | 病院勤務なのに「訪問看護のご経験」と書いている |
| 存在しない資格に言及していないか | 看護師なのに「理学療法士としてのご経験」と書いている |
| 日本語が不自然でないか | 文章が途中で切れている、同じことを繰り返している |
| 会社の情報が正しいか | 別の会社の特徴が混ざっている |

問題がある場合は **「NG - スキップ」** してください。

### 「OK - 送信する」を押した後に起きること

1. ジョブメドレー上で候補者のスカウト画面が自動で開く
2. 求人が自動で選択される（「求人自動選択」がONの場合）
3. スカウト文がテキスト入力欄に自動で入力される
4. 送信ボタンが自動でクリックされる
5. ジョブメドレーの「スカウトを送信しました」確認画面が表示される
6. 確認画面が自動で閉じられる
7. 次の候補者の確認ポップアップが表示される

**この間（約5〜10秒）はブラウザを操作しないでください。**

### 求人の自動選択について

- 「求人自動選択」のチェックがONの場合、候補者の職種に合わせて自動で求人を選択する
- 通常はONのままで問題なし

**自動選択がうまくいかない場合:**
1. 「求人選択に失敗しました。手動で選択して再開を押してください」と表示される
2. ジョブメドレーのスカウト画面のドロップダウンから、正しい求人を手動で選択
3. サイドパネルの「再開」ボタンをクリックして続行

### 全件送信完了後

すべての候補者の確認・送信が完了すると、「送信完了」のメッセージが表示されます。
- 送信結果（送信済み/スキップ）の一覧が確認できる
- 次のバッチに進む場合は、📋 抽出タブに戻って新しい候補者を抽出

---

## 求人の自動選択がうまくいかない場合の「手動モード」

「求人自動選択」のチェックをOFFにすると手動モードになります。

手動モードでは：
1. 確認ポップアップで「OK」を押した後、ジョブメドレーの画面が開く
2. 求人は自動選択されない → 手動でドロップダウンから選択する
3. スカウト文は自動入力される
4. 求人を選んだら、ジョブメドレーの送信ボタンを手動でクリック

頻繁に自動選択が失敗する場合は管理者に連絡してください。

## 1日の業務の流れ（例）

```
09:00 ジョブメドレーにログイン、サイドパネルを開く
09:05 📋 抽出: 20件の候補者プロフィールを抽出
09:15 🤖 API生成: スカウト文を一括生成
09:20 📤 送信: 1件ずつ確認しながら送信（約20件で30分）
09:50 次のバッチ: 📋 抽出に戻って次の20件を抽出
...繰り返し...
```

1バッチ15〜20件がおすすめです。件数が多いと抽出エラーのリスクが上がります。

## 送信タブのその他の機能

### CSVインポート

API生成を使わず、外部で生成したスカウト文をCSVから読み込むこともできます。

1. 送信タブの **「CSVファイル」** ボタンをクリック
2. CSVファイルを選択
3. CSVには `member_id`, `template_type`, `personalized_text`, `full_scout_text` 列が必要

通常の業務では使いません。管理者から指示があった場合のみ使用してください。

### ドライランモード（テスト用）

送信タブの下部にある **「ドライランモード」** にチェックを入れると、実際には送信されないテストモードになります。

- フォームへの入力は行われるが、送信ボタンはクリックされない
- 画面上に赤いバナーで「テストモード」と表示される
- 初めて操作する際の練習に使えます

**本番送信時は必ずドライランモードをOFFにしてください。**
