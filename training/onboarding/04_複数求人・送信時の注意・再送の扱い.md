# 複数求人がある場合・送信時の注意点・再送の扱い

---

## 1. 複数の求人がある会社の扱い

### なぜ複数求人があるのか

1社で複数の求人を出しているケースがあります。主な理由：

- **職種が異なる**: 看護師、理学療法士、作業療法士、言語聴覚士、医療事務など
- **雇用形態が異なる**: パート募集と正社員募集
- **施設が異なる**: 同じ会社でも施設ごとに別の求人

例えば「いちご訪問看護」は3つの施設（いちご介護訪問看護ステーション、富士見台サテライト、いちごリハビリステーション）があり、それぞれに複数の職種の求人があります。

### 求人の自動選択の仕組み

システムは候補者の情報をもとに、最適な求人を自動で選びます。

**選択の優先順位：**
1. 候補者の **資格** → 職種カテゴリ（看護師、理学療法士等）を判定
2. 候補者の **希望雇用形態** → パートか正社員かを判定
3. その職種 × 雇用形態に合う求人を選ぶ

例：
- 看護師資格 ＋ パート希望 → 「看護師 パート」の求人
- 理学療法士資格 ＋ 正社員希望 → 「理学療法士 正社員」の求人

### オペレーターが気をつけること

#### 生成時（🤖 API生成タブ）

- **職種の選択を間違えないこと**: 生成設定モーダルで選んだ職種に基づいてテンプレートと求人が決まる
- 例えば「看護師」を選んで生成すると、理学療法士の候補者は「除外」される
- **複数職種を送りたい場合**: 職種ごとにバッチを分ける
  - 1回目: 職種「看護師」で生成・送信
  - 2回目: 職種「理学療法士」で生成・送信

#### 送信時（📤 送信タブ）

- **確認ポップアップで求人名を必ず確認**する
  - 「対象求人」のチェックボックスは、正しい求人が選ばれていることを確認してからチェック
  - 看護師の候補者に理学療法士の求人が当たっていないか
  - パート希望の人に正社員の求人が当たっていないか

#### 求人自動選択が失敗するケース

以下のケースでは自動選択がうまくいかないことがあります：

| ケース | 起きること | 対処法 |
|--------|----------|--------|
| 該当する求人がない | 「求人選択に失敗しました」と表示 | 手動で正しい求人を選び「再開」 |
| 複数の求人が候補になる | 意図しない方が選ばれる可能性 | 確認ポップアップで求人名をしっかり確認 |
| 新しく追加された求人 | 拡張機能のキャッシュにまだない | ページをリロードして再試行。または⚙設定タブで求人を再取得 |

#### 「いちご訪問看護」の注意点

いちご訪問看護は3施設あり、求人数が多いです。

- 職種に応じて自動的に施設が振り分けられる
- 看護師 → いちご介護訪問看護ステーション
- リハビリ系（理学療法士・作業療法士・言語聴覚士）→ いちごリハビリステーション
- 確認ポップアップで施設名を含む求人名が表示されるので、正しい施設か確認してください

## 2. スカウト送信時に気をつけること

### 送信前の確認チェックリスト

確認ポップアップで「OK - 送信する」を押す前に、以下を確認してください。

#### 必須チェック項目

| # | チェック内容 | 具体的に見ること |
|---|-------------|----------------|
| 1 | **警告がないか** | ポップアップ上部に赤色や黄色の警告が出ていないか。赤は必ずスキップ |
| 2 | **求人が合っているか** | 候補者の資格・希望と、選ばれている求人の職種・雇用形態が一致しているか |
| 3 | **パーソナライズ文が自然か** | 日本語が不自然でないか、意味が通るか |
| 4 | **事実と一致しているか** | 持っていない資格や経験に言及していないか |

#### バリデーション警告の種類と対応

システムが自動チェックして表示する警告には以下があります：

| 警告の内容 | 深刻度 | 対応 |
|-----------|--------|------|
| **「スカウト対象外の候補者です」** | 高（赤色） | **必ずスキップ**。パーソナライズ文が生成されていない |
| **「年齢が範囲外です」** | 中（黄色） | 原則スキップ。管理者の指示がある場合のみ送信 |
| **「必要な資格がありません」** | 中（黄色） | スキップ。その求人に応募できない候補者 |
| **「カテゴリ対象外」** | 中（黄色） | スキップ。希望職種が募集と合わない |
| **「雇用形態の不一致」** | 低（黄色） | テンプレートが正社員だが候補者がパート希望、など。内容を確認して判断 |

#### パーソナライズ文でよくある問題

AIが生成した文章で、特に注意すべきパターン：

| 問題パターン | 例 | 対処 |
|-------------|-----|------|
| **資格の取り違え** | 看護師なのに「理学療法士としてのご経験」 | スキップ |
| **経験の捏造** | プロフィールにない経験に言及 | スキップ |
| **文章が途中で切れている** | 「当施設では...」で終わる | スキップ |
| **同じ内容の繰り返し** | 同じフレーズが2回出てくる | スキップ |
| **別の会社の情報が混ざっている** | 存在しない施設名やサービスに言及 | スキップ |
| **不自然な敬語** | 過剰に回りくどい表現 | 軽微なら送信OK、ひどければスキップ |

**迷ったらスキップ（NG）してください。** 送信後の取り消しはできません。

### 送信中の注意点

- **送信中はブラウザを操作しない**: 「OK」を押してから次のポップアップが出るまで（5〜10秒）は待つ
- **連続送信中に離席する場合**: 必ず「停止」してから離席する。ポップアップは操作待ちで止まるので放置は問題ないが、ジョブメドレーのセッションが切れる可能性がある
- **送信数のペース**: 1時間に30〜50件が目安。あまりに速いペースで送ると、ジョブメドレー側で制限がかかる可能性がある

### スキップした候補者について

- スキップした候補者は送信リスト上で「スキップ」ステータスになる
- 後から個別に送信することも可能
- ただしAI生成の問題でスキップした場合、同じ文章が使われるので注意
- 新しい文章で送りたい場合は、再度🤖API生成タブで生成し直す

## 3. 再送（再スカウト）の注意点

### 再送とは

過去にスカウトを送ったが返信がなかった候補者に、改めてスカウトを送ることです。

### 再送のルール

システムには以下の自動制限があります：

| ルール | 内容 | 意味 |
|--------|------|------|
| **送信間隔** | 前回のスカウトから最低7日間空ける | 7日以内に送ろうとすると自動で除外される |
| **送信回数上限** | 同一候補者への送信は最大2回まで | 3回目以降は自動で除外される |

これらは自動的にチェックされるので、オペレーター側で計算する必要はありません。条件に引っかかる候補者は生成時に「除外」されます。

### 再送時の操作上の違い

通常の初回送信との違い：

| 項目 | 初回 | 再送 |
|------|------|------|
| 生成設定の「送信種別」 | 「初回」を選択 | **「再送」を選択** |
| 使用されるテンプレート | 初回用テンプレート | 再送用テンプレート（文面が異なる） |
| パーソナライズ文の長さ | 約100文字 | 約120〜150文字（初回より少し長め） |
| 候補者リストの表示 | 「初回」バッジ | **「再送」バッジが表示される** |

### 再送で最も重要なこと

**「送信種別」を「再送」に切り替えること。**

初回のまま再送すると：
- 初回用テンプレート（初めましての挨拶）が使われてしまう
- 以前スカウトを送った相手に「初めまして」と送ることになり不自然
- 再送用テンプレートでは「改めてのご連絡」など、2回目であることを踏まえた文面になっている

### 再送のタイミング

- 初回スカウト送信後、**7日以上経過してから**再送する
- 7日未満だとシステムが自動で除外するため、生成しても「除外」になる
- 一般的に、初回から1〜2週間後が再送の目安

### 再送で除外される候補者

再送で生成を実行した際に「除外」になるケースと原因：

| 除外理由の表示 | 意味 | 対応 |
|---------------|------|------|
| 「スカウト送信済み(2回)」 | すでに2回スカウトを送っている | これ以上は送れない。新しい候補者を探す |
| 「スカウト送信から3日(7日未満)」 | 前回の送信から7日経っていない | 7日以上経ってから再試行 |
| その他の除外（年齢、資格等） | 初回と同じバリデーション | 初回と同じ対処 |

### 再送と初回を混ぜないこと

**1回のバッチでは初回か再送のどちらか一方に統一してください。**

- 初回と再送を同じバッチで混ぜると、テンプレートの使い分けが正しくできない
- 手順：
  1. ジョブメドレーで「スカウト未送信」の候補者を絞り込み → 初回として生成・送信
  2. ジョブメドレーで「スカウト送信済み・未返信」の候補者を絞り込み → 再送として生成・送信

### 再送の業務フロー

```
① ジョブメドレーで「スカウト済み・未返信」の候補者を絞り込む
② 📋 抽出タブで候補者を抽出
③ 🤖 API生成タブで、送信種別を「再送」にして生成
④ 📤 送信タブで確認・送信
    └ 候補者リストに「再送」バッジが表示されていることを確認
```

## まとめ: 送信時の判断フローチャート

確認ポップアップが表示されたとき、以下の順番で判断してください：

```
赤色の警告が出ている？
  → YES → 必ず「NG - スキップ」
  → NO ↓

黄色の警告が出ている？
  → YES → 内容を確認。年齢や資格の問題なら原則スキップ
  → NO ↓

求人は正しい？（職種・雇用形態・施設が合っているか）
  → NO → 「NG - スキップ」
  → YES ↓

パーソナライズ文は自然か？（資格の取り違え・捏造・途中切れがないか）
  → NO → 「NG - スキップ」
  → YES ↓

3つのチェックボックスにチェック → 「OK - 送信する」
```

**迷ったらスキップが正解です。** 送れなかった候補者は後からでも対応できますが、不適切なスカウトを送ってしまうと取り消しができません。
