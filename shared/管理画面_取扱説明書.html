1. この管理画面について

Scout Admin は、スカウト文の自動生成に使われる設定（テンプレート、パターン、AIのプロンプト、送信条件など）を管理するWebアプリです。

Chrome拡張やAPIから送られるスカウト文は、すべてこの管理画面で設定した内容に基づいて生成されます。設定を変えれば、生成されるスカウト文の内容も変わります。

主な機能:

  • スカウト文のテンプレート（外枠）の編集
  • 送信対象を絞るバリデーションルールの設定
  • AIが文章を生成するときのプロンプト（指示文）の調整
  • 経歴情報が少ない候補者向けの定型パターン管理
  • 対象求人の登録・管理
  • 生成ログの閲覧
  • 新しい会社の設定をAIで一括生成（既存会社の設定を参考にして生成）
  • 送信データの分析とテンプレート改善 NEW
  • 会社プロフィール・検出キーワードの管理 NEW

画面モードは2種類あります:

モードどんなとき使う？
フロービュー生成の流れを全体像で把握したいとき。テスト実行もここから
テーブルビューテンプレートやパターンを具体的に追加・編集・削除したいとき

初期セットアップ（新会社追加・求人登録等）は管理者向けセットアップガイド（server/admin/SETUP.md）を参照してください。

2. アクセスとログイン

URLを開く

ブラウザで以下のURLにアクセスします:

https://scout-api-1080076995871.asia-northeast1.run.app/admin/

ログイン

初回アクセス時に「パスワードを入力してください」と表示されます。

  1. Chrome拡張のAPI設定と同じ パスワード を入力
  2. 「ログイン」 を押す

ログインに成功すると管理画面が表示されます。パスワードはブラウザに保存されるので、同じブラウザからは次回以降自動でログインできます。

「パスワードが違います」と表示される場合は、管理者にパスワードを確認してください。

パスワードを変更したい場合は、ログイン後にヘッダー右側の 「パスワード設定」 ボタンから変更できます。

3. 画面の見方（ヘッダー）

ログイン後、画面の一番上にヘッダーバー（緑色）が表示されます。ここで会社の切り替えやビューの切り替えを行います。

ヘッダーの各ボタン

要素説明
会社ドロップダウン（左側）操作対象の会社を切り替えます。選んだ会社のデータだけが表示されます
＋ 新規会社新しい会社を追加するウィザードを開きます（セットアップガイド参照）
フロー / テーブル画面の表示モードを切り替えます
キャッシュ更新（右側）設定変更後にこのボタンを押すと、変更がスカウト生成に反映されます
パスワード設定（右側）ログインパスワードを変更できます
重要: テンプレートやパターンを編集した後は、必ず 「キャッシュ更新」 を押してください。押さないと、Chrome拡張からのスカウト生成に変更が反映されません。

6. テンプレートにパーソナライズ文を挿入
↓
7. 求人情報とID情報を付与して完成

ノードの見方

各ステップはカード形式で表示されます。

  • クリック → 詳細が展開され、設定内容のサマリーが表示されます
  • 編集可能なノード（3, 4, 5a, 5b, 7）は「テーブルで編集」リンク付き
  • テスト結果の色:
    • 緑 = このステップを通過
    • 赤 = このステップで失敗/除外
    • 灰 = このステップを通過していない

パイプラインテスト

画面上部のテストパネルで、候補者プロフィールを入力してパイプラインの動作を確認できます。

入力項目:

項目説明
資格複数選択可（看護師、准看護師 等）。選択するとタグ表示され、×で解除
希望雇用複数選択可（パート、正社員 等）
年齢数値入力（例: 35）
経験年数数値入力（例: 5）
経歴タイプ訪問看護、病棟 等
就業状態ドロップダウンから選択（就業中、離職中 等）
職務経歴（空=型はめ）テキスト入力。空欄にすると型はめパスで処理されます
自己PRテキスト入力
AI生成をスキップ（モック）チェックを入れるとAI呼び出しをスキップ（高速テスト用）

手順:

  1. テスト用のプロフィール情報を入力
  2. 「テスト実行」 を押す
  3. 各ステップの通過/失敗がノードの色で可視化されます
  4. AI生成またはパターン型はめの結果テキストが表示されます

5. テーブルビュー

8つのタブでデータを管理します。

共通操作

操作方法
新規追加ツールバーの 「＋ 追加」 ボタン
編集行の 「編集」 ボタン → 編集ダイアログ → 「保存」
削除行の 「削除」 ボタン → 確認ダイアログ → OK
ロック/ロック解除ツールバーの 「ロック解除」 / 「ロック」 ボタン。ロック中は他社データや全社共通データの編集・削除が無効になります

長いテキストは先頭部分のみ表示されます。「もっと見る」 で全文を確認できます。

テンプレート

スカウト文の外枠（挨拶・会社紹介・締め等）を管理します。

4種類:

  • パート初回 / パート再送
  • 正社員初回 / 正社員再送

テーブルの列:

列名説明
会社会社ID（例: ark-visiting-nurse）
職種カテゴリ対象職種（例: nurse）
テンプレート種別パート_初回 / パート_再送 / 正社員_初回 / 正社員_再送
テンプレート本文スカウト文の外枠テキスト
操作「編集」「削除」ボタン
重要: テンプレート本文内の {ここに生成した文章を挿入} がAI生成文や型はめ文の挿入位置です。この記述を必ず含めてください。

テンプレートバージョニング NEW

テンプレートを編集・保存すると、以下の自動バージョニングが行われます:

  • バージョン番号の自動インクリメント: テンプレートを保存するたびに、バージョン番号が自動的に上がります（v1 → v2 → v3 ...）。テーブル上でも現在のバージョンが確認できます。
  • 変更履歴の自動保存: 変更内容は「テンプレート変更履歴」シート（Google Sheets）に自動記録されます。
  • 追跡可能な情報: いつ変更されたか、どのテンプレートが変更されたか、変更前後の内容が記録されるため、過去の状態をいつでも確認できます。
バージョニングは自動で行われるため、特別な操作は不要です。テンプレートを編集して「保存」を押すだけで、バージョンが自動的に管理されます。

年齢上限スカウト対象の最高年齢（例: 55）
資格ルール（JSON）求人ごとの必須資格・除外資格（下記参照）

編集ダイアログの設定項目

ルールベース（チェックボックス）:

ルール説明
資格不一致NG求人の必須資格を持たない候補者を除外
臨床経験なしNG臨床経験がない候補者を除外

スカウト上限設定（スカウト済みNGを有効にした場合に表示）:

項目説明
最大送信回数同じ候補者へのスカウト上限回数
再送間隔（日）再送までの最低日数

資格ルール（求人ごと）:

  • 必須資格: この資格のいずれかを持つ候補者のみ対象
  • 除外資格: この資格を持つ候補者を除外

AI判定条件（自由記述）: テキストで条件を書くとAIが判定します（例:「ブランク10年以上の場合は除外」）

AI生成設定

AI生成パス（経歴あり候補者）で使われるプロンプトの構成パーツを管理します。複数のセクションが表示順に結合されて、AIへの指示文（system prompt）になります。

テーブルの列:

列名説明
会社会社IDまたは 全社共通（青色リンク表示）
セクション種別このセクションの種類（後述）
職種カテゴリ対象職種（空欄＝全職種共通）
表示順結合される順番（1が最初）
内容プロンプト本文
操作「編集」「削除」ボタン

全社共通 と表示されている行は、全ての会社で共有されるセクションです。会社固有の設定と組み合わせて使われます。

セクション種別:

種別説明補足
role_definitionAIの役割定義通常は全社共通
station_features会社の特色（AI生成の参考情報）会社固有
education教育体制（AI生成の参考情報）会社固有
tone_and_manner文体・トーンのルール通常は全社共通
common_rules共通ルール（文字数等）通常は全社共通
ai_guideAI生成ガイド（生成の細かいルール）会社固有
ng_expressionsNG表現リスト通常は全社共通

職種カテゴリ列: 空欄の場合は全職種で使われます。特定の職種（例: nurse）を指定すると、その職種の候補者にのみ適用されます。

プロンプトプレビュー: 「全文プレビュー」 ボタンで、全セクションがどう組み立てられてAIに渡されるか確認できます。

AIで一括生成

パターンのテキストをAIで一括生成できます。

  1. ツールバーの 「AIで一括生成」 を押す
  2. 会社情報 テキストエリアに会社の特色・教育体制等を入力（profile.mdの内容を貼り付け）
  3. 「生成する」 を押す → AIがパターンA〜Gのテキストを生成
  4. プレビューを確認
  5. 「保存する」 を押す → 既存パターンが上書きされます

分析 NEW

送信データの集計・分析と、分析結果に基づくテンプレート改善を行うタブです。

クロス集計

送信済みスカウトデータを様々な切り口で集計・可視化できます。

  • 職種×テンプレートVer: どの職種でどのバージョンのテンプレートが使われたかを集計
  • 地域別: エリアごとの送信数を集計
  • AI生成 vs 型はめ: 生成パス（AI生成/型はめ）ごとの送信数・割合を比較
  • その他の軸: 雇用形態別、経験年数帯別など

Geminiによる自動分析

送信データをGemini（AI）が分析し、改善提案を自動生成します。

  1. 「分析を実行」 ボタンを押す
  2. Geminiが送信データの傾向やパターンを分析
  3. 分析結果と改善提案がテキストで表示されます

テンプレート改善

分析データに基づいて、AIがテンプレートの改善案を生成します。

  1. 「テンプレート改善」 ボタンを押す
  2. AIが現在のテンプレートと分析データを踏まえて改善案を作成
  3. 改善案が diff形式（変更前→変更後）で表示されます
  4. 内容を確認し、「承認して反映」 を押すとテンプレートに反映されます
テンプレート改善を承認すると、テンプレートのバージョンが自動的にインクリメントされ、変更履歴にも記録されます。反映後は 「キャッシュ更新」 を忘れずに。

全社横断分析

ヘッダーの会社ドロップダウンで「全社」を選択すると、全ての会社のデータを横断して分析できます。会社間の比較や全体の傾向把握に利用できます。

プロフィール管理 NEW

各会社のプロフィール情報（会社の特色・教育体制等）と、Chrome拡張の会社自動判定で使われる検出キーワードを管理します。

テーブルの列:

列名説明
会社会社ID
プロフィール内容会社の特色・教育体制・アピールポイントなど（profile.mdに相当する内容）
検出キーワードChrome拡張が求人ページから会社を自動判定するためのキーワード一覧

プロフィール内容

会社の特色や強みをテキストで管理します。この情報は以下の場面で参照されます:

  • AI生成時のプロンプトに会社情報として注入
  • 型はめパターンのAI一括生成時の参考情報
  • テンプレート改善提案時の会社コンテキスト

検出キーワード

Chrome拡張は、ジョブメドレーの求人ページ上のテキストからこのキーワードを照合し、どの会社のスカウト設定を使うかを自動判定します。

  • キーワードは会社名・施設名・法人名など、求人ページに表示される文字列を登録します
  • 複数キーワードを登録可能（いずれか1つがヒットすれば判定されます）
  • キーワードが正しく設定されていれば、オペレーターが手動で会社を選択する必要がなくなります

操作方法:

操作方法
プロフィール編集行の 「編集」 ボタン → プロフィール内容を修正 → 「保存」
キーワード追加行の 「編集」 ボタン → 検出キーワード欄にキーワードを追加 → 「保存」
新規会社プロフィール追加ツールバーの 「＋ 追加」 ボタン
検出キーワードを変更した場合は 「キャッシュ更新」 を押してください。Chrome拡張はAPIからキーワードを取得するため、キャッシュ更新後に反映されます。

6. よくある操作手順

テンプレートの文言を変更する

  1. テーブルビューで テンプレート タブを選択
  2. 対象行の 「編集」 を押す
  3. body フィールドでテキストを修正
  4. 「保存」 → 「キャッシュ更新」

バリデーションルールを追加・変更する

  1. テーブルビューで バリデーション タブを選択
  2. 対象行の 「編集」 を押す
  3. 年齢範囲やルールを変更
  4. 「保存」 → 「キャッシュ更新」

パターンをAIで一括生成する

  1. テーブルビューで 型はめ設定 タブを選択
  2. 「AIで一括生成」 を押す
  3. 会社情報を入力 → 「生成する」
  4. プレビュー確認 → 「保存する」 → 「キャッシュ更新」

パイプラインの動作を確認する

  1. フロービューに切り替え
  2. テストパネルに候補者情報を入力
  3. 「テスト実行」 を押す
  4. 各ステップの通過/失敗を確認

求人を追加する

推奨: Chrome拡張の⚙ 設定タブ →「求人を取得」→「選択した求人をAPIに登録」

手動: テーブルビューの 求人 タブ → 「＋ 追加」 でジョブメドレーの求人IDを入力

送信データを分析してテンプレートを改善する NEW

  1. テーブルビューで 分析 タブを選択
  2. クロス集計で送信データの傾向を確認
  3. 「分析を実行」 でAIによる分析と改善提案を取得
  4. 「テンプレート改善」 でdiff形式の改善案を確認
  5. 問題なければ 「承認して反映」 → 「キャッシュ更新」

会社プロフィール・検出キーワードを更新する NEW

  1. テーブルビューで プロフィール管理 タブを選択
  2. 対象会社の行で 「編集」 を押す
  3. プロフィール内容や検出キーワードを修正
  4. 「保存」 → 「キャッシュ更新」

7. キャッシュ更新

管理画面での設定変更は Google Sheets に書き込まれますが、APIサーバーはメモリ上のキャッシュを使用しています。

設定変更後は必ずヘッダーの「キャッシュ更新」を押してください。 これにより、APIサーバーがGoogle Sheetsから最新データを再読み込みします。

8. トラブルシューティング

症状対処法
