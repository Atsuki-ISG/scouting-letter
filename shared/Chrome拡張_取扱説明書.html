1. この拡張機能について

Scout Assistant は、ジョブメドレーでのスカウト送信作業をサポートするChrome拡張機能です。

📋 プロフィール抽出 → 🤖 スカウト文生成 → 📤 連続送信

主な機能:

  • ジョブメドレーの候補者一覧から、プロフィール情報を自動で読み取ります（最大50件）
  • 読み取った情報をもとに、AIが候補者ごとのスカウト文を自動生成します
  • 生成されたスカウト文を1件ずつ確認しながら、連続で送信できます

画面構成: サイドパネル（ブラウザの右側に表示）に5つのタブがあります。

タブ役割
📋 抽出候補者のプロフィールを読み取る
🤖 API生成スカウト文を一括生成する
📤 送信スカウト文を確認・編集・送信する
💬 返信候補者とのやりとりを取得・保存する
⚙ 設定APIの接続先やパスワードを設定する
初期設定（インストール・API接続・会社登録等）は管理者が行います。手順は管理者向けセットアップガイドを参照してください。

2. サイドパネルを開く

  1. Chromeツールバーの Scout Assistant アイコン をクリック
  2. ポップアップが表示されるので、「サイドパネルを開く」 ボタンを押す
  3. ブラウザの右側にサイドパネルが開きます
サイドパネルは一度開くと、別のタブに移動しても表示されたままです。

3. 会社を選ぶ

サイドパネル上部の 「会社」 ドロップダウンで、スカウトを送る会社を選択してください。

会社の自動判定 NEW

ジョブメドレーのページを開くと、ページ上の会社名・施設名から自動的に会社ドロップダウンが切り替わります。ページを移動するたびに自動判定が行われるため、通常は手動で会社を選ぶ必要はありません。

自動判定が正しくない場合は、手動でドロップダウンから正しい会社を選択してください。手動で選択した値が優先されます。

4. 基本操作: 抽出 → 生成 → 送信

Step 1: プロフィール抽出（📋 抽出タブ）

前提: ジョブメドレーの候補者検索ページを開いた状態で操作してください。

項目説明
開始会員番号途中から抽出したい場合に入力（空欄＝先頭から）
抽出件数1〜50件（デフォルト5件）

手順:

  1. 件数を設定
  2. 「抽出開始」 を押す
  3. 拡張機能が候補者カードを順番にクリックし、プロフィールを自動で読み取ります
  4. 進捗バーで「進捗: X/Y」が表示されます
  5. 完了後、候補者一覧が表示されます

途中停止: 「停止」 ボタンで中断できます。抽出済み分は保持されます。

CSVダウンロード: 抽出完了後に 「CSVダウンロード」 ボタンが表示されます。外部ツールでスカウト文を生成する場合に使用します。

Step 2: API生成（🤖 API生成タブ）

抽出したプロフィールからスカウト文をAIで一括生成します。

画面表示: 「抽出済み: N件」で対象件数を確認できます。

手順:

  1. 「一括生成」 を押す
  2. 生成設定モーダル が表示されます。必要に応じて設定を変更してください:
設定項目説明
会社サイドパネル上部と連動（変更可）
雇用形態自動判定 / パート / 正社員 から選択
職種カテゴリ全て / 看護師 / PT / ST / OT / 医療事務 から選択
送信タイプ初回 / 再送 から選択
前回の設定が保存されています。変更がなければそのまま生成できます。
  1. モーダル内の 「一括生成」 を押す
  2. 「生成中... X/Y」の進捗バーが表示されます
  3. 完了後、結果サマリーが表示されます:
    • 合計: 処理した総数
    • AI生成: 経歴情報からAIが文章を生成した件数
    • 型はめ: 定型パターンで生成した件数
    • 除外: バリデーションで対象外になった件数
  4. 生成結果は自動で 📤 送信 タブの候補者リストにセットされます

Step 3: 送信（📤 送信タブ）

生成されたスカウト文を確認し、ジョブメドレーから送信します。

候補者リストの見方

各候補者の行には以下が表示されます:

表示意味
🔵未送信（送信待ち）
✅送信済み
⏩スキップ
テンプレートバッジパート/正社員、初回/再送の種別
⭐ バッジ NEW候補者が求人に「気になる」を押している（お気に入り）
⚠ 警告バッジバリデーション警告（送信可能だが注意が必要）
❌ エラーバッジバリデーションエラー（対象外候補者）

「気になる」候補者の検出 NEW

候補者がジョブメドレーで求人に「気になる」を押している場合、候補者リストの名前の横に ⭐ バッジ が表示されます。

  • 抽出時に自動検出されるため、手動操作は不要です
  • 「気になる」候補者には、件名に感謝を含むお気に入り用テンプレートが自動選択されます
  • 通常のテンプレートに変更したい場合は、確認ポップアップ内で手動で切り替えられます

対象求人の自動選択

「求人自動選択」 がONの場合、候補者の職種カテゴリと雇用形態に応じて適切な求人が自動選択されます。

自動選択の仕組み:

  • ジョブメドレーの求人ドロップダウンを検索キーワードで開く
  • ドロップダウン内の求人テキストを、職種カテゴリ（看護師、理学療法士 等）＋雇用形態（パート、正職員 等）のキーワードで照合
  • 両方一致する求人が優先、なければ職種カテゴリのみで選択、1件のみの場合はそのまま選択
うまく選択されない場合は 「求人自動選択」 をOFFにして、手動で求人を選んでください。

手動送信（1件ずつ）

各候補者の行にあるボタンで操作します:

ボタン動作
コピースカウト文全文をクリップボードにコピー
本文セットジョブメドレーのスカウト画面にテキストを自動入力
求人+本文求人選択＋テキスト入力を同時に実行
送信済ステータスを「送信済み」に変更
スキップこの候補者をスキップ

「求人+本文」の流れ:

  1. ボタンを押すとジョブメドレーのスカウトフォームが開きます
  2. 求人が自動選択され、テキストエリアにスカウト文が自動入力されます
  3. 確認ポップアップが表示されます:
    • 候補者情報（資格、経験、希望雇用、エリア、自己PR、職歴）
    • 会員番号、対象求人、パーソナライズ文、全文プレビュー
  4. チェックリスト（会員番号・求人・パーソナライズ文）を確認してチェック
  5. OK（送信する） → ジョブメドレーの送信ボタンをクリックして送信
  6. NG（スキップ） → この候補者をスキップ

連続送信アシスト

複数の候補者を順番に処理する場合に使います。

  1. 「連続送信アシスト」 トグルをONにする → 「開始」 ボタンが表示される
  2. 「開始」 を押す
  3. 最初の未送信候補者から順番に確認ポップアップが表示されます
  4. 各候補者ごとに:
    • OK → 送信して次の候補者へ
    • NG → スキップして次の候補者へ
  5. 全件処理すると完了通知が表示されます

途中停止: 連続送信中は確認ポップアップ内に 「連続送信を停止」 ボタンが表示されます。サイドパネルの 「停止」 ボタンでも中断できます。

スキップ: サイドパネルの 「スキップ」 ボタンで現在の候補者を飛ばして次へ進みます。

文面の編集

候補者リストのパーソナライズ文プレビューをクリックすると、インライン編集モードになります。

  • テキストを修正して保存できます
  • 修正理由を入力するフィールドもあります（任意）
  • 修正すると修正記録が自動保存されます
  • 「修正履歴エクスポート」 ボタンで修正内容をMarkdown形式でエクスポートできます

確認ポップアップ内でも直接テキストを編集できます。

ドライランモード

「ドライランモード」 をONにすると、実際には送信せずに送信フローをシミュレートします。初回利用時の練習や動作確認に使用してください。

デバッグログ

「デバッグログ」 をONにすると、各ステップの詳細ログが表示されます。問題発生時のトラブルシューティングに使用します。

5. CSVインポートによる送信（代替フロー）

外部ツール（Claude.ai、ChatGPT等）で生成したスカウト文を送信する場合に使用します。

インポート方法

方法1: CSVファイル選択

  1. 📤 送信タブの 「CSVファイル」 ボタンを押す
  2. ファイル選択ダイアログでCSVファイルを選択

方法2: テキスト貼り付け

  1. 「貼り付け」 ボタンを押す → テキストエリアが表示される
  2. CSVテキストを貼り付け（コードブロック csv 付きでもOK）
  3. 「インポート」 ボタンを押す

CSVの必須列

列名説明
member_id会員番号
template_typeテンプレート種別（例: part_first, seishain_first）
personalized_textパーソナライズ文（AI生成部分）
full_scout_textスカウト文全文（テンプレート＋パーソナライズ文）

クリア: 「クリア」 ボタンで候補者リストを全て削除できます。

6. やりとり取得（💬 返信タブ）

候補者との返信やりとりを取得・保存し、分析用にエクスポートします。

自動取得（1件）

  1. ジョブメドレーのメッセージページで対象の会話を開く
  2. 「やりとりを取得」 を押す
  3. 現在表示中の会話が自動で読み取られます

自動取得（一括）

  1. ジョブメドレーのメッセージページを開く
  2. 「一括取得」 を押す
  3. サイドバーの会話リストを順番に自動取得します
  4. 「取得中... X/Y件」で進捗が表示されます

手動入力

  1. 「手動入力」 を押す → 入力フォームが表示される
  2. 会員番号 を入力
  3. やりとり内容 を以下のフォーマットで入力:
2026-03-01
事業所: スカウト文の内容...
求職者: 返信の内容...
事業所: 返答の内容...
  1. 「保存」 を押す

サーバーへの同期 NEW

取得したやりとりデータをAPIサーバーに送信し、分析用に蓄積できます。

  1. やりとりを取得した状態で、「サーバーに同期」 ボタンを押す
  2. 取得済みのやりとりデータがAPIサーバーに送信・保存されます
  3. 同期完了後、件数が表示されます
同期されたデータは管理画面の分析機能で活用されます。返信率やテンプレート効果の分析に使用するため、定期的に同期してください。

エクスポート

ボタン出力内容
全件YAMLエクスポート全会話をYAMLファイルとしてダウンロード
返信スカウト記録エクスポート分析用の構造化データとしてエクスポート

8. トラブルシューティング

症状対処法
「Content Scriptと通信できません」エラージョブメドレーのページをリロードしてください
API接続テスト失敗エンドポイントURLとパスワードが正しいか管理者に確認してください
一括生成ボタンが押せない先に📋 抽出タブでプロフィールを抽出してください
生成結果が全て「除外」管理者にバリデーション設定を確認してもらってください
求人自動選択が失敗する「求人自動選択」をOFFにして手動で求人を選択してください
連続送信が途中で止まるジョブメドレーのページがアクティブか確認してください
本文セットしてもテキストが入力されないページをリロードしてから再試行してください
抽出中にエラーが出る候補者検索結果ページを表示した状態で実行してください
拡張機能が動作しないchrome://extensions で拡張機能が有効か確認してください
会社が自動判定されない手動でドロップダウンから会社を選択してください
