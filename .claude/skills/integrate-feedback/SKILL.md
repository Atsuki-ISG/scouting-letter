# フィードバック統合スキル

修正記録（fixes）から学びを抽出し、ナレッジとスキルに反映します。

---

## Step 0: スコープ判定

入力内容を分析し、**個別修正**か**共通ルール**かを判定する。

| シグナル | スコープ | アクション |
|---------|---------|-----------|
| 特定候補者番号 + 特定会社の文脈 | **個別** | → 個別修正フロー（Step 1〜8） |
| 表現・トーン・構造に関するルールで、会社・職種を問わず適用可能 | **共通** | → 共通ルールフロー（Step C1〜C8） |
| 判断がつかない | **確認** | → ユーザーに確認してから判定 |

### 判定ヒント

**共通ルールの可能性が高い**:
- 「常に」「全社」「どの職種でも」等の汎用性を示す言葉
- 表現パターンの問題（何を言う/言わない）であって、会社固有の事実（教育制度名等）ではない
- 事実の誤り（eラーニング名の間違い等）ではなくコミュニケーション原則
- 1つの会社で発見されたが、他社にも同じパターンがある

**個別修正の可能性が高い**:
- 特定候補者番号が指定されている
- 会社固有の制度名・施設名の誤り
- 特定テンプレートの文言修正

## 個別修正フロー（Step 1〜8）

### Step 1. 修正内容の記録（fixes への追記）

- 新規の修正がある場合、該当会社の `companies/[会社名]/history/fixes/YYYY-MM.md` に記録
- ファイルが存在しない場合は新規作成し、月間記録としてまとめる

### Step 2. 修正記録の読み込みと学びの抽出

- 記録された内容、または既存の `fixes/YYYY-MM.md` から以下を抽出:
  - **問題点**: 修正前の何が問題だったか
  - **修正ポイント**: どう改善したか
  - **学び**: 今後に活かすパターン

### Step 3. 重複チェック

- `companies/[会社名]/recipes.md` および `knowledge/learnings.md` の既存パターンと照合
- 類似パターンがあれば統合、なければ新規追加

### Step 4. 会社別 recipes.md への反映（最優先）

- csv-scoutが直接参照するファイルのため、ここへの反映が生成品質に直結する
- 修正内容に応じて以下のセクションを更新:
  - **型はめパターンの表現修正**: 該当する型の本文・特色バリエーションを更新
  - **共通ルールの追加**: NGパターンや表現ルールを追加
  - **AI生成ガイドの更新**: 接点対応表やNGパターンに追加
  - **教育体制の誤用防止**: 他職種・他社の制度混入に関する注意を追加

### Step 5. knowledge/learnings.md への反映（全社横断の記録）

- `knowledge/learnings.md` の「改善履歴」セクションに日付と概要を記録
- 全社共通のパターンは「パターン別の接点例」にも追加
- ※ csv-scoutはlearnings.mdを直接読み込まないが、全社横断の記録・generate-scoutの参照用として維持

### Step 6. generate-scoutスキルへの反映（重要度が高い場合）

- `.claude/skills/generate-scout/SKILL.md` の該当セクションに追加
- 「絶対NG」「効果的」「パーソナライズの手法」など

### Step 7. 会社別 templates.md への反映（テンプレート本体・行動指針・フィルタリング条件の変更がある場合）

- `companies/[会社名]/templates.md` の該当セクションを更新

### Step 8. 完了報告

- 反映した内容のサマリーを出力
- 更新したファイル一覧を表示

## 共通ルールフロー（Step C1〜C8）

### Step C1. ルールの汎用化

- 特定会社・職種に閉じない汎用的なルール表現にする
- 複数職種の例を併記（例:「看護師求人で〜、リハビリ求人で〜」）
- 看護師以外の職種（リハビリ・医療事務等）、様々な事業形態（訪問看護・病院・施設等）にも適用できるか検証
- `knowledge/learnings.md` の「表現のルール」セクションに書く正規形を先に確定

### Step C2. 反映（全ファイルチェックリスト順）

以下のチェックリストを**上から順に**確認し、該当箇所があれば更新する。

**ナレッジ層（全社共通）**:
- [ ] `knowledge/learnings.md` — 「表現のルール」セクション + 「改善履歴」
- [ ] `knowledge/basics.md` — 基本ルールに該当する場合

**会社別ファイル（CLAUDE.md記載の全社分ループ）**:
- [ ] `companies/*/templates.md` — 「生成ルール」セクション
- [ ] `companies/*/recipes.md` — 共通ルールセクション + 型はめパターン本文 + AI生成ガイド

**スキル定義**:
- [ ] `.claude/skills/generate-scout/SKILL.md` — 核心ルール・NG表現リスト
- [ ] `.claude/skills/csv-scout/SKILL.md` — csv-scout固有の注意事項（該当時のみ）

**サーバー関連**:
- [ ] `server/seed-data/2_パターン.tsv` — 型はめパターンテキスト
- [ ] `server/seed-data/README.md` — seed-dataの説明
- [ ] `server/tests/test_pattern_matcher.py` — テストデータ内のパターンテキスト
- [ ] `test/api-prompt-test/system-prompt.md` — APIテスト用プロンプト

**バリデーションルール**:
- [ ] `test/rules/validation-rules.yml` — NG表現・必須表現・構造ルールを追加

### Step C3. ナレッジ全体スキャン（違反箇所の網羅検出）

新ルールに違反する箇所が反映先以外にも残っていないかを検出する。

- 新ルールから**違反パターンのgrepキーワード**を導出
- リポジトリ全体を検索し、ヒット箇所を一覧表示
- 各ヒットが実際に違反か（パターンテキスト）、許容か（資格修飾・例示等の文脈）を判定
- 違反箇所があれば追加修正

スキャン対象:
- `companies/*/recipes.md` — 全パターンテキスト + AI生成ガイドの例文
- `companies/*/templates.md` — テンプレート本文・行動指針の例示
- `companies/*/history/examples.md` — 過去の良い例
- `knowledge/` — 過去の改善履歴の例示
- `server/seed-data/` — 全パターン・資格修飾テキスト
- `server/tests/` — テストデータ
- `test/` — APIテスト用プロンプト・サンプルデータ
- `.claude/skills/*/SKILL.md` — スキル内の例示

### Step C4. 全社パターン横並びチェック

- 同じ型（F/G等）で会社間にルール違反が残っていないか
- テンプレートの生成ルール番号が全社で揃っているか
- recipes.md の共通ルールセクションが全社に入っているか

### Step C5. seed-data ↔ recipes.md 一致チェック

サーバー登録済みの会社について、seed-dataのパターンテキストとrecipes.mdが一致しているか確認。

### Step C6. テスト実行 + サンプル生成チェック

```bash
cd server && python3 -m pytest tests/ -q
```

テスト通過後、ダミープロフィール（`test/dummy-profiles/`）を使って一括生成し、全ルールに対して検証:

- `test/rules/validation-rules.yml` のNG表現が生成テキストに含まれていないか
- 必須表現（資格修飾等）が正しく含まれているか
- テンプレート接続時に構造ルール違反（重複表現等）がないか
- AI生成パスでもルールが効いているか（半自動、目視含む）

### Step C7. サーバー同期リマインダー

```
⚠️ サーバ同期が必要（デプロイ不要・Sheets更新のみ）:

パターンシート:
/sync-patterns [該当会社名]

プロンプトシート（AI生成パス）:
管理画面 > [各社] > プロンプト > 表現ルールセクションにルールを追加
```

### Step C8. 完了報告

以下を出力:
- 汎用化されたルール文
- 更新した全ファイル一覧
- 検証結果（スキャン結果 + テスト結果 + サンプル生成結果）
- サーバー同期リマインダー
- 確認用grepコマンド

## 判断基準

### スコープ判定基準

上記 Step 0 を参照。

### recipes.mdへの反映基準（最優先）

csv-scoutが直接参照するため、以下は必ず反映:
- 型はめパターンの表現修正（上から目線、不自然な表現等）
- 共通ルール・NGパターンの追加
- AI生成ガイドの接点対応表の更新
- 教育体制の誤用に関する注意の追加

### generate-scoutへの反映基準

以下に該当する場合は `generate-scout/SKILL.md` にも反映:
- 「絶対NG」に追加すべき表現パターン
- 繰り返し発生する修正パターン
- パーソナライズの新しい手法

## 注意事項

- 既存のパターンと重複する場合は追加せず、必要に応じて既存を更新
- **recipes.md が生成品質に直結する最重要ファイル**。会社固有の学びは必ずここに反映する
- templates.md は テンプレート本体・行動指針・フィルタリング条件の変更がある場合のみ更新
- 反映後は必ず差分を確認して報告
- 共通ルール適用時は `test/rules/validation-rules.yml` にもルールを追加し、今後の自動チェックに反映する
