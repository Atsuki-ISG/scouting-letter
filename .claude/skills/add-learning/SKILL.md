# 修正内容をlearnings.mdに追加するスキル

一般のAIチャット（ChatGPT、Claude.aiなど）で生成された修正内容を `knowledge/learnings.md` に追加します。

## 使い方

1. 一般のAIチャットでスカウト文を生成
2. 出力された「📝 learnings.md追加用（コピペ用）」のセクションをコピー
3. このスキルを実行: `/add-learning`
4. コピーした内容を貼り付け

## 実行手順

1. **引数または直前のメッセージから修正内容を取得**
   - `$ARGUMENTS` が空の場合、直前のユーザーメッセージから内容を取得
   - マークダウン形式の修正内容を期待

2. **内容の解析**
   以下の情報を抽出:
   - 日付（YYYY-MM-DD形式）
   - 候補者番号（会員番号）
   - 修正内容（「**候補者[番号]の修正から:**」以降の内容）

3. **learnings.mdの読み込み**
   - `knowledge/learnings.md` を読み込む
   - 「## 改善履歴」セクションを特定

4. **新しいエントリの追加**
   - 「## 改善履歴」セクションの直後（最新の位置）に追加
   - 同じ日付のエントリが既に存在する場合、その日付セクションに追加
   - 存在しない場合、新しい日付セクションを作成

5. **フォーマット確認**
   追加する内容が以下の形式になっているか確認:
   ```markdown
   ### YYYY-MM-DD

   **候補者[会員番号]の修正から:**
   [修正内容]
   ```

6. **ファイルへの書き込み**
   - Edit ツールを使用して learnings.md を更新
   - 追加完了のメッセージを表示

7. **確認**
   - 追加された内容を表示して確認を促す

## 入力形式の例

ユーザーが貼り付ける内容:

```markdown
### 2026-02-15

**候補者03090808の修正から:**
経験年数が非常に浅い場合（1年未満など）、わざわざ年数を強調しない。代わりに、教育体制や成長環境など会社の強みを前面に出す方が効果的。
```

または、AIチャットが出力した完全な形式:

```markdown
## 📝 learnings.md追加用（コピペ用）

以下をコピーして `knowledge/learnings.md` の「改善履歴」セクションに追加してください：

```markdown
### 2026-02-15

**候補者03090808の修正から:**
経験年数が非常に浅い場合（1年未満など）、わざわざ年数を強調しない。代わりに、教育体制や成長環境など会社の強みを前面に出す方が効果的。
```
```

## 出力フォーマット

```
✅ learnings.md に以下の内容を追加しました：

---
### YYYY-MM-DD

**候補者[会員番号]の修正から:**
[修正内容]
---

📍 追加場所: knowledge/learnings.md の「改善履歴」セクション
```

## エラーハンドリング

- 日付が見つからない場合: 今日の日付を使用
- 候補者番号が見つからない場合: そのまま追加（警告を表示）
- 「## 改善履歴」セクションが見つからない場合: ファイル末尾に追加
- 内容が空の場合: エラーメッセージを表示して中断

## 注意事項

- 同じ候補者の重複チェックは行わない（意図的に同じ候補者から複数の学びを追加できる）
- 既存の内容は変更しない（追加のみ）
- 日付は新しいものが上に来るように追加（最新が上）
- 共通ルール（全社横断で適用すべき表現ルール等）の場合は `/integrate-feedback` を使用すること。`/add-learning` は learnings.md のみに追加するため、recipes.md / templates.md / サーバーへの横展開は行わない
