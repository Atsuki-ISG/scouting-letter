# スカウト文生成スキル

求職者のプロフィールスクリーンショットから、パーソナライズされたスカウト文を生成します。

## 実行手順

1. **引数から会社名を取得**
   - 引数: `$ARGUMENTS`
   - 会社名が指定されていない場合は、利用可能な会社一覧を表示して選択を促す

2. **会社情報の読み込み（2ファイルのみ）**
   - `companies/[会社名]/profile.md` から会社プロファイルを読み込む
   - `companies/[会社名]/templates.md` からテンプレートと生成ルールを読み込む
   - 会社ディレクトリが存在しない場合はエラーメッセージを表示

3. **共通ナレッジの読み込み**
   - `knowledge/learnings.md` から効果的な表現パターンを読み込む
   - 特に「パターン別の接点例」を参照してパーソナライズに活用

4. **スクリーンショットの解析**
   - 直前に貼り付けられた画像からプロフィール情報を抽出
   - 抽出する情報:
     - 保有資格
     - 経験年数・経験施設
     - 希望条件（勤務地、雇用形態など）
     - 自己PR・アピールポイント

5. **パーソナライズ文の生成**
   - templates.mdの行動指針に従い、会社情報の「強み」と求職者の「経験」の接点を約100文字で作成
   - 書き出しは「プロフィールを拝見し、」から始める

6. **完全版スカウト文の出力**
   - templates.mdのテンプレートにパーソナライズ文を挿入した完全版を出力
   - パーソナライズのポイントも併記

7. **クリップボードにコピー**
   - 完成したスカウト文をクリップボードにコピー（`pbcopy`）
   - コピー完了のメッセージを表示

## 出力フォーマット

```
## パーソナライズポイント

- [プロフィールから読み取った情報] → [接点の表現]

## スカウト文（クリップボードにコピー済み）

[テンプレートにパーソナライズ文を挿入した完全版]
```

## 共通ナレッジ（埋め込み済み）

### スカウト文の基本原則

- 一方的なアピールではなく「対話の始まり」を意識
- 求職者のキャリアや希望を尊重した姿勢を示す
- テンプレート感を排除し、パーソナライズを重視

### 返信率を上げるポイント

**効果的:**
- 求職者の経歴・スキルへの具体的な言及
- なぜその人にスカウトを送ったかの理由
- 次のアクションを明確に提示

**避ける:**
- 長すぎる自社アピール
- 上から目線の表現（「フォローします」「安心してください」）
- 「アットホーム」「やる気次第」などの曖昧な表現

### パーソナライズの手法

| レベル | 内容 | 例 |
|--------|------|-----|
| 基本 | 資格への言及 | 「看護師資格を拝見しました」 |
| 中級 | 経験への具体的言及 | 「急性期で3年間ご活躍された経験」 |
| 上級 | 希望条件とのマッチング | 「当施設では〇〇のポジションがございます」 |

### プロフィールの「行間」を読む

- **資格取得が最近** → 実践経験を積みたい
- **複数施設を経験** → 合う職場を探している
- **ブランクあり** → サポート体制を重視
- **詳細な自己PR** → しっかり読んでほしい

### 絶対NG

- 年齢・世代への言及（「〇代」「若手」「ベテラン」）
- 詳細な居住地への言及（「〇〇区にお住まい」→「札幌市内」に）
- 憶測（「長く続けているはず」「〜されたいのですね」）

---

## 注意事項

- スクリーンショットが貼り付けられていない場合は、先に画像を貼り付けるよう案内する
- プロフィール情報が少ない場合は、最低限のパーソナライズで対応
- 個人情報の取り扱いに注意（生成した文面は外部に保存しない）
