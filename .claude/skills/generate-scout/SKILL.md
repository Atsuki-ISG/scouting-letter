# スカウト文生成スキル v4

求職者のプロフィールスクリーンショットから、パーソナライズされたスカウト文を生成します。

## 引数オプション
- `resend` → 再送スカウト（デフォルト: 初回）
- `seishain` → 正社員テンプレート使用（デフォルト: パートテンプレート）
- 組み合わせ例: `ark-visiting-nurse resend seishain`

## 実行手順

1. **引数から会社名・モード・テンプレート種別を取得**
   - 引数: `$ARGUMENTS`
   - 例: `ark-visiting-nurse` → 初回・パートテンプレート
   - 例: `ark-visiting-nurse resend` → 再送・パートテンプレート
   - 例: `ark-visiting-nurse seishain` → 初回・正社員テンプレート
   - 例: `ark-visiting-nurse resend seishain` → 再送・正社員テンプレート
   - 未指定の場合、`companies/` 配下のディレクトリ一覧を表示

   ### テンプレート自動判定
   引数に `seishain` がない場合でも、プロフィールの **希望勤務形態が「正職員」のみ**（パート・バイトを含まない）なら、自動的に正社員テンプレートを使用する。

2. **必要最小限のファイル読み込み**
   ```
   - companies/[会社名]/templates.md（テンプレート・行動指針）
   - knowledge/learnings.md（過去の学びのみ）
   ```
   ※ profile.md、basics.md、structure.md、personalization.mdは読み込まない（トークン削減）

3. **プロフィール解析**
   - 直前の画像から以下を抽出:
     - 保有資格
     - 経験年数・施設
     - 希望条件
     - 自己PR

4. **パーソナライズ文の生成**

   ### 初回モード（デフォルト）
   - 文字数目安: **約100文字**
   - 必ず「プロフィールを拝見し、」で開始
   - templates.mdの初回テンプレートを使用
   - templates.mdの行動指針に厳密に従う
   - learnings.mdの「改善履歴」を最優先で反映

   ### 再送モード（`resend` 引数あり）
   - 文字数目安: **120〜150文字**（熱量を感じさせるため少し長め）
   - 必ず「プロフィールを拝見し、」で開始
   - templates.mdの**再送テンプレート**（正社員/パートを判定して選択）を使用
   - **初回と異なる角度**でパーソナライズする（同じ接点を繰り返さない）
   - 「諦めきれず」という冒頭と整合した熱量を文中でも表現する
   - ただし「厚かましい」「押しつけ」にならないよう注意
   - 文体・トーン・ですます調は初回と同じを維持

5. **出力形式**

```
## パーソナライズ文

[生成した文章]（初回: 約100文字 / 再送: 120〜150文字）

## 完全版スカウト文（クリップボードにコピー済み）

```
[テンプレートに挿入した完全版]
```

## 自己評価チェック

- [ ] templates.mdの行動指針に準拠しているか
- [ ] learnings.mdの過去の修正事例を反映しているか
- [ ] 年齢・世代・詳細な居住地への言及がないか
- [ ] 上から目線・憶測・不要な前置きがないか
- [ ] 【再送時のみ】初回と異なる角度になっているか
- [ ] 【再送時のみ】熱量があるが厚かましくないか

**※ 上記チェックで懸念がある場合は修正候補を提示**
```

6. **クリップボードコピー**
   ```bash
   echo "[完全版]" | pbcopy
   ```

## 修正が必要と判断した場合の出力

スカウト文生成後、以下の条件で自動的に修正提案を出力：

**修正提案が必要な場合:**
- templates.mdの行動指針違反を検出した場合
- learnings.mdの過去の修正ルールに抵触する可能性がある場合

**出力フォーマット:**

```markdown
---

## 🔧 修正候補（learnings.md形式）

**候補者[会員番号]の修正提案:**
[修正理由の説明]

**修正前:**
[該当部分]

**修正後:**
[推奨する表現]

**ナレッジ化すべき学び:**
[今後に活かすルール]

---

※ この修正が適切であれば、`/integrate-feedback` で反映してください
```

## 核心ルール（必須遵守）

### learnings.mdから抽出した重要ルール

1. **経験年数が浅い場合**（1〜2年など）
   - 「〇年間」と明示しない
   - 研修制度などポジティブ面に焦点

2. **地理情報の重複回避**
   - 勤務地・訪問範囲はテンプレートに記載済み
   - パーソナライズ文では言及不要

3. **こだわり条件は参考程度**
   - 資格・経験で強い接点が作れる場合、こだわり条件への言及不要

4. **「感銘を受ける」の使用制限**
   - 経験そのものには使用禁止
   - 代替: 「心強く拝見」「注目しました」

5. **強力なキャリア（10年以上）**
   - 地理的接点より専門的接点を優先
   - 期待値をストレートに伝える

6. **情報が少ない場合**
   - 地理的要素のみに頼らない
   - 会社の強み（教育体制、成長環境）を前面に
   - 「ご一緒したい」「想いを強く感じ」などの**送り手感情主体**は不自然になりがち
   - 「成長をサポートできる環境がある、まずは一度お話でもいかがでしょうか」という**相手へのオファー主体**で締める

### templates.mdから抽出した絶対NG

- 年齢・世代への言及（「〇代」「若手」「ベテラン」）
- 詳細な居住地への言及（「〇〇区」→「札幌市内」に）
- 憶測（「長く続けているはず」「〜されたいのですね」）
- 上から目線（「フォローします」「安心してください」「基礎を固めてこられた」）
- 不要な前置き（「現在離職中とのこと」「新たな環境で〜とお考えのタイミング」）

---

## トークン最適化のポイント

- **不要なファイルは読み込まない**: basics.md、structure.md、personalization.mdの内容は本スキルに埋め込み済みのため読み込み不要
- **learnings.mdのみ動的読み込み**: 最新の学びを反映するため、learnings.mdのみ毎回読み込む
- **templates.mdは必須**: 会社別の固有情報とテンプレートのため必須

---

## 注意事項

- スクリーンショット未添付の場合: 画像添付を案内
- プロフィール情報が少ない場合: 会社の強みベースで最低限のパーソナライズ
- 会社ディレクトリ不在: エラーメッセージ表示
