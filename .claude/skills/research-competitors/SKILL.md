# 競合調査スキル

WebSearchで競合情報を調査し、対象会社の「隠れた強み」を発見する。

## ゴール

単なる情報比較ではなく、**対象会社が気づいていない自社の強み**を見つけ出すこと。
profile.mdに書いてあることをそのまま繰り返すのではなく、競合と比較して初めて見える優位性を提案する。

例:
- 「この規模で24時間オンコールは競合の半数以下。求職者にとって大きな安心材料」
- 「このエリアで精神科訪問看護は2施設しかない。専門性を前面に出すべき」
- 「給与は競合より低いが、スタッフあたり利用者数が少ない＝余裕のある働き方を訴求可能」

## 手順

### 1. 当社情報の把握

`companies/[会社ID]/profile.md` を読み、エリア・業態・職種・現在の訴求ポイントを把握する。

### 2. WebSearchで競合調査

以下の検索を実行（WebSearchツールを使用）:

**検索クエリ例**（エリア・業態・職種に応じて調整）:
- `[エリア] [業態] [職種] 求人 給与 待遇`
- `[エリア] [業態] 口コミ 評判 働きやすさ`
- `[エリア] [業態] 求人倍率 人材不足`
- `[業態] [職種] 転職 重視ポイント 2026`
- `ジョブメドレー [エリア] [職種]`

**調査する競合の条件**:
- 同エリア（同市区町村 or 隣接エリア）
- 同業態（訪問看護→訪問看護、病院→病院、有料老人ホーム→有料老人ホーム）
- 同職種（看護師→看護師）

### 3. 比較分析

#### 定量比較
| 項目 | 当社 | 競合A | 競合B | 競合C |
|------|------|-------|-------|-------|
| 月給/時給 | | | | |
| 勤務時間 | | | | |
| 夜勤/オンコール | | | | |
| 福利厚生 | | | | |
| 規模 | | | | |

#### 定性比較
- 教育・研修体制
- 文化・雰囲気（口コミベース）
- キャリアパス
- 特色・専門領域
- 働き方の柔軟性

### 4. 隠れた強みの発見（最重要）

競合データと当社profile.mdを突き合わせ、**当社が気づいていない可能性がある強み**を3-5個提案する。

各提案は以下の形式:

```
## 発見: [何が強みか（1行）]
根拠: [なぜそう言えるか — 競合との具体的な比較データ]
活用案: [スカウト文でどう訴求するか — 具体的な表現例]
確認事項: [クライアントに裏取りすべきこと]
```

**NGパターン**:
- profile.mdに書いてあることをそのまま繰り返す → 価値がない
- 「教育体制が充実しています」のような抽象的な提案 → 具体性がない
- 根拠なしの推測 → データで裏付ける

### 5. 出力の構成

以下のセクションでまとめる:

1. **調査概要**（対象会社、エリア、職種、調査日）
2. **競合施設一覧**（表形式）
3. **定量比較**
4. **定性比較**
5. **🔍 隠れた強み・差別化の種**（3-5個、各提案に根拠・活用案・確認事項）
6. **業界・エリア動向**（市場状況、求職者の重視ポイント）
7. **スカウト文フック提案**（隠れた強みを踏まえた具体表現5個）
8. **💡 ヒアリング提案**（仮説検証型: 「〇〇という仮説がありますが実態は？」形式）

### 6. 結果の保存

`workspace/competitor-research/YYYY-MM-DD_[会社ID].md` に保存する。
ディレクトリがなければ作成。
