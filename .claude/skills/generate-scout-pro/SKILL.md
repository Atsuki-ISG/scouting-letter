# スカウト文生成スキル pro 版（Phase 0 新原則）

## 概要

既存 `/generate-scout` の上位互換として、Phase 0 で確立した新原則を適用してスカウト文を生成する。

**新原則の核**:
1. **4ブロック統一**: opening / bridge / facility_intro / closing_cta（job_framing は原則統合）
2. **属性別 opening 修飾句マッピング**: 候補者属性ごとに勧誘構文を切り替える
3. **トーン別動詞マッピング**: business / casual / letter で語彙を変える
4. **読みやすさ最優先**: 体言止め禁止／読点ルール／経歴動詞列挙2つまで／組織語数字を opening に置かない
5. **対等パートナー感**: 上から（活かす場面ある）／下から（お力をお借り）／大げさ（響き合う）／機械的（重なる）を全て排除

参考サンプル: `workspace/phase0-full-refined.md`（4サンプル全文）

---

## 引数オプション

- `resend` → 再送スカウト（デフォルト: 初回）
- `seishain` → 正社員テンプレート使用（デフォルト: パートテンプレート）
- 未指定の場合、`companies/` 配下のディレクトリ一覧を表示

### テンプレート自動判定

引数に `seishain` がない場合でも、プロフィールの **希望勤務形態に「正職員」を含む**なら正社員テンプレートを使用。「パート・バイト」のみ（正職員を含まない）場合にパートテンプレートを使用。

## 実行手順

### 1. ファイル読み込み

```
- companies/[会社名]/profile.md
- companies/[会社名]/templates.md
- knowledge/learnings.md（参考）
- workspace/phase0-full-refined.md（4サンプル参照、判断に迷ったら）
```

### 2. 候補者プロフィール解析

スクリーンショット or テキスト入力から以下を抽出:
- 保有資格
- 経験職種・経験年数
- 就業状況（在職中／離職中／ブランク）
- 居住地・希望勤務地
- 希望雇用形態・希望入職時期
- 自己PR（全文・字数）
- 職務経歴概要

### 3. 候補者属性判定（3軸）

#### 3-1. 情報量判定

| 区分 | 条件 |
|---|---|
| **rich** | self_pr 100字以上 OR 職務経歴詳細あり |
| **moderate** | experience_type＋希望条件あり |
| **sparse** | 基本情報のみ／PR 50字未満／職歴半年未満 |

#### 3-2. 経験タイプ判定

| 区分 | 条件 | 例 |
|---|---|---|
| **veteran_with_specialty** | 経験10年以上＋専門資格（感染管理認定／DMAT／管理職経験 等） | 48歳師長×感染管理認定 |
| **pr_values_explicit** | PR に「傾聴」「協調性」「○○を大切に」「寄り添う」等の**姿勢が明記** | 47歳PT×傾聴・チームアプローチ／48歳離職中×協調性 |
| **young_sparse** | 経験1年未満 ＋ PR 50字未満 | 21歳看護1年未満×PR12字 |
| **blank_career** | 離職中 or ブランク3ヶ月以上 ＋ 多職歴 | 48歳離職9ヶ月×大学病院～有料老人ホーム |
| **general** | 上記いずれにも該当せず | デフォルト |

#### 3-3. トーン判定

| トーン | 該当条件 | 文体特徴 |
|---|---|---|
| **business** | 病院系・40代以上ベテラン・専門資格保有 | 敬語ベース・事実重視・「存じます」系 |
| **casual** | 訪看リハ・複合体制・若手・カジュアル雰囲気の会社 | 「！」OK・「カルチャー／マッチ」等カタカナ語OK |
| **letter** | 離職中・40代後半以上・多職歴・letter PR | お手紙調・「お迎えしたい」「ご縁」「幸い」等 |

会社のトーン傾向は `companies/[会社名]/profile.md` か `templates.md` の文体から推定可能。

#### 3-4. 対象フィルタ判定（templates.md のフィルタ階層に応じて分岐）

`companies/[会社名]/templates.md` の「## スカウト対象フィルタリング」セクションを読み、候補者を以下の3階層に分類:

| 区分 | 対応 |
|---|---|
| **対象（明確）** | 通常通り生成。警告なし |
| **対象（要警告）** | 生成するが、属性判定の「警告」フィールドに警告文を出力 |
| **除外条件** | 生成を中止し、対象外であること・除外理由を明示 |

**判定ロジック**:
1. 候補者の experience_type / qualifications / desired_area を読む
2. 「除外条件」のいずれかに該当 → 除外（生成中止）
3. 「対象（明確）」のいずれかに該当 → 明確対象として生成
4. 「対象（要警告）」のいずれかに該当 → 警告つき対象として生成
5. どこにも該当しない → 「対象不明」として警告つきで生成（オペレーター判断に委ねる）

**警告文の作り方**:
- なぜ要警告に分類したか（経験領域の違い等）
- どう接続したか（理念フィット・転用可能なスキル等）
- 例: 「障害福祉のみの経験。daiwa-house-ls の主対象（高齢者介護）からは外れますが、管理者経験/理念フィットでパーソナライズ可能と判断しました」

「除外条件」に該当する場合は、本ステップ以降をスキップして以下の出力で終了:
```
## 対象外
- 区分: 除外
- 理由: [具体的な除外条件と該当箇所]
- 推奨: スカウト送信を見送り、または別職種テンプレートを検討
```

#### 3-5. 給与訴求パターン判定（profile.md に「給与訴求マトリクス」がある会社のみ）

`companies/[会社名]/profile.md` に「## 給与訴求マトリクス」セクションがあれば発動。無ければスキップして Step 4 へ。

**抽出元**: 候補者プロフィールの `特記事項`（こだわり条件）または `special_conditions`

**抽出ルール**:
- 「年収500万円以上可能」「年収600万円以上可能」「年収700万円以上可能」等を含む → **500万以上希望**
- 「年収400万円以上可能」を含む（500万以上は含まない） → **400万以上希望**
- どちらも含まない／空欄／「-」「未入力」「なし」 → **希望なし**

**判定マトリクス参照**: `profile.md` の「給与訴求マトリクス」→「候補者の希望年収条件 → 訴求パターン判定」表で A / B / C を決定。

**判定結果の使い方**:
- **A: 即マッチ** → ヘッダーをパターンAに差し替え + 連絡理由フックAを bridge or facility_intro に1〜2文で挿入
- **B: 昇給で到達** → ヘッダーをパターンBに差し替え + 連絡理由フックBを挿入
- **C: 訴求しない** → デフォルトヘッダー（既存）使用、給与訴求フックは入れない

**訴求時のヘッジ表現**（A/Bともに必須）:
- ❌ 「〜になります」「確実に到達」（断定）
- ✅ 「〜のモデルケース」「お応えできる可能性」「実現可能な範囲」
- 候補者の希望額は「500万以上」のレンジ表記で引用、具体額の差分計算は避ける

### 4. 属性別 opening 修飾句マッピング

| 経験タイプ | opening 修飾句 | 結び動詞 |
|---|---|---|
| **veteran_with_specialty** | 〜ご経験と〜ご視点を拝見しました | お迎えしたい方だと感じ |
| **pr_values_explicit** | 〜を大切にされている方／〜を軸にされているご姿勢 | 来ていただきたいと思い／お迎えしたいと感じ |
| **young_sparse** | 構文外し「〜と拝見しました」→「ご検討いただけたら」 | ご検討いただけたらと思い |
| **blank_career** | 〜姿勢をお持ちの方 | お迎えしたいと感じ |
| **general** | 〜ご経験を拝見しました | お迎えしたい方だと感じ |

### 5. トーン別動詞マッピング

| トーン | 候補者経験の活用動詞（bridge 末尾） | 価値観合致表現（opening 中盤） |
|---|---|---|
| **casual** | 活かせます | カルチャーとマッチしており |
| **business** | 活かしていただけます | （明示せず opening の経歴展開で示す） |
| **letter** | 活かしていただけます | （opening で「姿勢をお持ちの方を〜お迎えしたい」） |

### 6. ブロック別ライティングガイド

#### opening (130-180字)

**役割**: 候補者の歩みを描写 → 属性別修飾句 → 結び動詞

**書き方**:
- 経歴動詞列挙は **2つまで**（「〜重ね、〜寄り添い、〜束ね、〜形にし」のような4連はNG）
- 追加属性（資格・視点）は「〜を拝見しました」で前半を閉じる
- 後半は属性別修飾句 → 結び動詞
- 数字・組織語（「172床増床を控える」「2026年4月に」等）は使わない、情景表現に（「新病棟づくりを進めている当院」「○○を整えている当院」）
- 結び末尾の読点削除（「思い、ご連絡」→「思いご連絡」）

**良い例**:
- 「救命救急から訪問看護へ、そして師長・ステーション立ち上げへと歩んでこられたご経験と、感染管理認定のご視点を拝見しました。新病棟づくりを進めている当院でお迎えしたい方だと感じご連絡いたしました。」

#### bridge (130-200字)

**役割**: 会社側の文脈（体制・進行中の動き）と、候補者の経験がどう活きるかの接続

**書き方**:
- 冒頭は **会社側の文脈**から入る（候補者経歴の事実反復で始めない）
- 具体数字（「172床」「70床」等）はここで自然に織り込む
- 候補者の経験 → 「活かせます／活かしていただけます」（トーン別）で締め
- opening と同じ動詞（重なる・響き合う等）を使わない
- bridge の中で「ご視点も〜活かせます」のように **候補者主体**で書く

**良い例（casual）**:
- 「当ステーションは看護師・PT・OT・STが同じ事業所に在籍する複合体制で、訪問先でのご本人・ご家族の暮らし全体を多職種で支えています。通所リハや病院で積み上げられたPTのご経験に加え、児童指導員や福祉住環境コーディネーターのご視点も、訪問リハの現場で活かせます。」

#### facility_intro (150-220字)

**役割**: 候補者の関心事に応える事実を3項目程度

**書き方**:
- 候補者の属性応答（OJT・希望エリア・通勤手段等）→ 勤務条件 の順で文章のまま
- **箇条書き不採用**（事務的・AI臭・手紙のトーン崩れ）
- 制度列挙は3項目程度に絞る
- 「〜運用」（事務語）→「〜流れ／〜ご用意しております」に置き換え

**良い例**:
- 「訪問リハが初めての場面でも、先輩リハビリ職との同行訪問とオンライン研修で現場に慣れていただける流れです。訪問先は練馬区・杉並区の半径3km圏内で、電動アシスト自転車・バイク・車の貸与もございます。勤務面は完全週休2日（土日祝）で年間休日120日以上、有給消化率もほぼ100%、資格取得支援はセミナー代から書籍代まで全額対応しております。」

#### closing_cta (60-100字)

**役割**: 短く軽い誘い

**書き方**:
- 会社強み（オンライン説明会・ビデオオフOK・応募書類不要 等）あれば織り込む
- 「見学のみ・カジュアル面談のみでも大歓迎です」等の低ハードル
- letter トーンの場合は「具体的な復職時期が決まっていない段階でも〜」等の配慮を加える

### 7. 禁止表現チェックリスト

#### 全廃
- [ ] 体言止め（「〜歩み。」「〜キャリア。」「〜ご経歴。」）
- [ ] ダッシュ（——）多用
- [ ] 「思い、ご連絡」「感じ、ご連絡」等の連用中止読点
- [ ] 経歴動詞列挙4つ以上
- [ ] 組織語の数字（「172床増床を控える」「○○床への増床を控える」）を opening に
- [ ] 「運用」（事務語）

#### 機械的・大げさ・上から
- [ ] 「重なる」「かみ合う」「響き合う」（機械的・大げさ）
- [ ] 「なじむ」（候補者姿勢が主語の時）
- [ ] 「〜場面があるかと思います」「〜活きるのではと考えております」（上から目線）
- [ ] 「探していた方向と近く」「求めていた方向と近く」（絞り出し感）
- [ ] 「今まさに」「考えていた」（わざとらしい）
- [ ] 「近いかと思います」（自信なし）

#### 下から目線
- [ ] 「お力をお借りしたい」「必要としています」「お力をお貸しください」

#### 熱烈すぎ
- [ ] 「ぜひ〜いただきたい」「〜こそ〜」
- [ ] 「〜のような方を探していました」（待ち望んでいた感が出過ぎ）

#### くどい
- [ ] 「ご資格まで併せて」「お持ちの方まで」等の連結
- [ ] 「感染管理認定のご視点もお持ちの方を、2026年4月に172床増床を控える当院でお迎えしたい」のようなもりもり1文

## 注意事項

- スクリーンショット未添付の場合: 画像添付を案内
- プロフィール情報が少ない場合（sparse）: opening の構文を「ご検討いただけたら」に切り替え
- 会社ディレクトリ不在: エラーメッセージ表示
- workspace/phase0-full-refined.md の4サンプル（S1〜S4）が「正解の温度感」の参照リファレンス
