# rate-scout

スカウト文を評価・分析し、`training/scout-examples.md` に蓄積するスキル。
自社・他社・他業界のスカウト文を問わず、良い例も悪い例も取り込む。

## 入力

ユーザーがスカウト文をペーストする。以下の情報を聞く:

1. **スカウト文本文**（必須）— ペーストされたテキスト
2. **ソース**（聞く）— 選択肢で提示:
   - `自社` — 自分たちが送ったスカウト
   - `他社・医療介護` — 医療介護系の他社から受け取ったスカウト
   - `他社・エンジニア` — エンジニア向けスカウト
   - `人材紹介` — 人材紹介会社から受け取ったスカウト
   - `人事向け` — 人事担当者向けのスカウト
   - `その他` — 上記以外
3. **ユーザー評価**（聞く）— S/A/B/C/D の5段階
4. **一言メモ**（任意）— 「ここが良かった」「ここがダメ」など

## 分析

`training/scout-fundamentals/good-scout-pro.md` の基準に基づいて自動分析する。

### 分析項目

1. **使われている心理テクニック**（第1章ベース）
   - 返報性、一貫性、社会的証明、好意、権威、希少性、損失回避
   - 該当するものをタグとして付与

2. **文章設計の評価**（第2章ベース）
   - 重心の置き方は適切か
   - 冒頭のフックは効いているか
   - CTAのハードルは低いか

3. **AI臭さチェック**（第3章ベース）
   - AI臭い表現が含まれているか
   - 人間らしさがあるか

4. **良い点**（最大3つ）
5. **改善点**（最大3つ）
6. **学び**（このスカウトから自社に取り入れられること）

## 出力

### 画面表示

分析結果をユーザーに表示する:

```
## 評価結果

**ソース:** 他社・エンジニア
**ユーザー評価:** A
**心理テクニック:** 返報性、希少性、自己関連効果

### 良い点
- GitHubリポジトリの具体的なコードに言及（自己関連効果が強い）
- CTAが「15分の雑談」で極めてハードルが低い
- 一文目で技術スタックを名指し（開封率に効く）

### 改善点
- 会社紹介が長い（中盤で離脱リスク）

### 学び
→ 冒頭で候補者の具体的な成果物に言及する手法は、医療介護でも「○○科での○年の経験」に転用可能
```

### ファイル保存

`training/scout-examples.md` に追記する。

フォーマット:

```markdown
---

### [連番]. [ソース] — 評価: [S/A/B/C/D]
**日付:** YYYY-MM-DD
**心理テクニック:** [タグ]
**メモ:** [ユーザーの一言メモ]

> [スカウト文本文（長い場合は冒頭300文字+省略）]

**良い点:** [箇条書き]
**改善点:** [箇条書き]
**学び:** [自社に取り入れられること]
```

## 注意事項

- 会社名・個人名が含まれている場合は `[会社名]` `[氏名]` にマスキングして保存
- スカウト文が長い場合は冒頭300文字 + 「（以下省略）」で保存。全文は不要
- ユーザーが評価を迷っている場合は `quality/scoring.md` の基準を簡潔に提示する
- 分析は簡潔に。長々と解説しない
