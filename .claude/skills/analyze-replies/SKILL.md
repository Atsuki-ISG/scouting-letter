# 返信分析スキル

蓄積されたやりとりデータ（YAML）と返信スカウト記録から、スカウト文改善のナレッジを抽出します。

## データソース

以下のファイルを読み込む:

1. **やりとりスレッド**: `companies/[会社名]/history/conversations/*.yml`
2. **返信スカウト記録**: `companies/[会社名]/history/replies/*.yml`
3. **修正履歴**: `companies/[会社名]/history/fixes/*.md`
4. **良い例**: `companies/[会社名]/history/examples.md`
5. **既存ナレッジ**: `knowledge/learnings.md`

## 実行手順

### 1. データ収集

```
companies/[会社名]/history/conversations/*.yml
companies/[会社名]/history/replies/*.yml
```

を全件読み込む。会社名が省略された場合は `companies/` 配下の全会社を対象にする。
ファイルが存在しない場合は、ユーザーにYAMLファイルを配置するよう案内する。

### 2. 基本統計の算出

以下を算出して表示:

- **総スカウト数**: やりとりスレッドの件数
- **返信あり件数**: 求職者メッセージが1通以上あるスレッド数
- **返信率**: 返信あり / 総スカウト数
- **平均応答日数**: スカウト送信日 → 最初の返信日の平均
- **テンプレート別返信率**: template_type ごとの返信率（replies データがある場合）

### 3. パーソナライズ分析

返信があったスカウトと返信がなかったスカウトのパーソナライズ文を比較:

- **返信ありのパーソナライズ文の特徴**:
  - 文字数の平均
  - よく使われるキーワード・フレーズ
  - 触れている要素（資格、経験年数、エリア、自己PR等）
- **返信なしとの差分**:
  - 具体的に何が違うか

### 4. 求職者プロフィール分析（replies データがある場合）

返信した求職者の傾向:
- 資格の分布
- 経験年数の分布
- 希望雇用形態（パート/正社員）
- 居住エリア

### 5. 返信内容のパターン分類

やりとりスレッドの求職者メッセージを分類:

| カテゴリ | 説明 | 例 |
|---------|------|-----|
| 前向き応募 | 面談・応募希望 | 「ぜひお話を伺いたく」 |
| 質問型 | 条件の確認 | 「勤務時間について教えて」 |
| 条件付き | 条件次第で前向き | 「週2日なら可能」 |
| 辞退 | お断り | 「今回は見送り」 |
| 情報収集 | まだ検討段階 | 「もう少し詳しく知りたい」 |

各カテゴリの件数と割合を算出する。

### 6. 会話の流れ分析

複数通のやりとりがあるスレッドから:
- **面談・採用に至ったケース**: 成功パターンの特定
- **途中で途絶えたケース**: 離脱ポイントの特定
- **企業側の返信内容**: 効果的だった返信とそうでない返信

### 7. ナレッジ生成

分析結果を以下の形式で出力:

#### (A) レポート出力

分析結果のサマリーを表示する。形式:

```
## スカウト分析レポート - [会社名] - YYYY-MM

### 基本統計
- 総スカウト: XX件
- 返信あり: XX件（XX%）
- 平均応答日数: X.X日

### テンプレート別効果
| テンプレート | 送信数 | 返信数 | 返信率 |
|------------|--------|--------|--------|
| パート初回   | XX     | XX     | XX%    |

### 発見した傾向
1. [具体的な発見]
2. [具体的な発見]

### 改善提案
1. [具体的なアクション]
2. [具体的なアクション]
```

#### (B) learnings.md への反映

新たに発見したルールがあれば、`knowledge/learnings.md` の「改善履歴」に追加:

```markdown
### [YYYY-MM-DD] 返信分析から得た知見
- 状況: [分析対象の概要]
- 発見: [具体的な発見]
- ルール: [今後のスカウト文に適用するルール]
```

#### (C) templates.md への反映提案

テンプレートの改善が必要な場合、具体的な修正案を提示する。
ただし **自動で書き換えず、ユーザーに確認を取ってから反映する**。

## データが少ない場合

件数が5件未満の場合:
- 基本統計のみ表示
- 「データが少ないため傾向分析は時期尚早です。10件以上蓄積されてから再度実行してください」と案内
- 現時点で気づいた点があれば定性的にコメント

## 分析なしでもできること

YAMLファイルがなくても、ユーザーがやりとり内容をテキストで直接貼り付ければ分析可能。
その場合は「やりとりテキストを貼り付けてください」と案内する。
